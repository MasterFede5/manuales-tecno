# 7.3 Forward pass y backpropagation (intuitivo, sin derivadas)

Una red neuronal aprende repitiendo un bucle de cuatro pasos clave.
Aquí te los explicamos en términos sencillos, sin cálculos de derivadas.

::concepto{titulo="Forward pass, loss, backward pass"}
* **Forward pass:** Los datos entran, cruzan las capas y generan una predicción.
* **Loss (función de pérdida):** Mide la distancia entre tu predicción y el valor real.
  * **MSE:** Para regresión (ej. predecir una nota de examen).
  * **BCE:** Para clasificación binaria (ej. aprueba o reprueba).
  * **CrossEntropy:** Para múltiples clases.
* **Backward pass (backpropagation):** Descubre qué pesos causaron el error y cuánto deben cambiar.
* **Optimizador (SGD, Adam):** Aplica esos cambios a los pesos.

```python
# Loop de entrenamiento
para cada batch de datos:
    1. pred = model(X)           # Forward pass
    2. loss = criterion(pred, y) # Calcular Error
    3. loss.backward()           # Backprop (PyTorch hace la mate)
    4. optimizer.step()          # Ajustar pesos
    5. optimizer.zero_grad()     # Limpiar para el siguiente batch
```

*Repite esto por muchos **epochs** hasta que la loss baje.*
::/concepto

::interioriza{titulo="El chef y el supervisor (Intuición)"}
Imagina un chef ciego (la red) preparando sopa.
* **Forward:** Sirve un plato y alguien lo prueba.
* **Loss:** El cliente dice "le falta sal" (mide el error).
* **Backward pass:** El supervisor deduce qué ingredientes causaron el mal sabor.
* **Optimizador:** Corrige la receta para el siguiente plato.

Backpropagation hizo posible el deep learning. Sin él, ajustar millones de parámetros a mano sería imposible.
::/interioriza

## Loss functions en código

```python
import torch.nn as nn

# Regresión (predecir cal_final)
criterion = nn.MSELoss()      
# Binaria (incluye sigmoid, más estable)
criterion = nn.BCEWithLogitsLoss()
# Multi-clase (incluye softmax)
criterion = nn.CrossEntropyLoss()
```

## Optimizadores

```python
import torch.optim as optim

# AdamW (con weight decay, mejor default moderno)
optim.AdamW(model.parameters(), lr=0.001, weight_decay=0.01)
```

El **learning rate (lr)** es el paso de aprendizaje:
* Muy alto: la pérdida explota.
* Muy bajo: tarda una eternidad.
* `0.001` con Adam/AdamW funciona casi siempre para datos tabulares.

## Lo que PyTorch hace por ti

```python
loss.backward()  # Magia pura en una sola línea
```

PyTorch usa **autograd** para calcular todas las derivadas automáticamente.
Guarda los gradientes correctos en `param.grad` listos para el optimizador.

## Conceptos clave que vas a oír

* **Epoch:** Una pasada completa por todo el dataset.
* **Batch:** Un pequeño subconjunto de datos (ej.: 32, 64) procesados juntos.
* **Iteration:** Una actualización de pesos tras procesar un batch.
* **Convergencia:** El punto donde la loss deja de bajar significativamente.

## Anti-patrones comunes

* **Olvidar `optimizer.zero_grad()`:** Los gradientes se acumulan y el ajuste falla.
* **No verificar la loss:** Asumir que aprendió sin revisar la curva de error.
* **Usar SGD básico:** Mejor usa AdamW a menos que reproduzcas un paper antiguo.
