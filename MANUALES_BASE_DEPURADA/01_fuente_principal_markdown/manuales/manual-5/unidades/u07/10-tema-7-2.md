# 7.2 Capas, activaciones (ReLU, Softmax)

Apilas neuronas en capas, y cada capa transforma los datos.
**La elección de capas y activaciones es la "arquitectura" de tu red.**

::concepto{titulo="Capas y activaciones"}
Una **red neuronal** organiza neuronas en **capas**:
- **Input layer**: tantas neuronas como features (sin activación).
- **Hidden layers**: capas intermedias con activación ReLU (estándar).
- **Output layer**: su activación depende de la tarea:
  - **Regresión**: 1 neurona, sin activación.
  - **Clasificación binaria**: 1 neurona, sigmoid (o nada con BCEWithLogits).
  - **Clasificación multi-clase**: K neuronas, softmax.

**MLP** (Multi-Layer Perceptron) = red densamente conectada con capas ocultas.
::/concepto

::interioriza{titulo="Analogía: El filtro del director"}
Imagina que la *input layer* son los datos del alumno (asistencias, tareas).
Las *hidden layers* son un equipo de profesores analizando el caso.
- ReLU es como decir "si es negativo, descártalo (0); si es positivo, pásalo tal cual".
La *output layer* es el director dando el veredicto final: "Aprobado" (Sigmoid: 0 a 1).
::/interioriza

## Activaciones más usadas

```python
import torch.nn.functional as F

# ReLU: max(0, x). Default para hidden layers.
F.relu(x)

# Sigmoid: 0 a 1. Para output binario.
torch.sigmoid(x)

# Softmax: probabilidades sumando 1. Para multi-clase.
F.softmax(x, dim=1)

# Tanh: -1 a 1. Menos usado modernamente.
torch.tanh(x)
```

## Arquitectura para el caso escolar

- **Predicción de cal_final** (regresión):
  `Input(4) → Linear(4→16) + ReLU → Linear(16→8) + ReLU → Linear(8→1)`
  *(97 + 136 + 9 = 242 parámetros)*
- **Clasificación binaria** (aprueba):
  `Input(4) → Linear(4→16) + ReLU → Linear(16→1) + Sigmoid`
- **Clasificación multi-clase** (Excelente/Bueno/Suficiente/Reprobado):
  `Input(4) → Linear(4→32) + ReLU → Linear(32→4) + Softmax`

## Reglas heurísticas

- **2-3 capas ocultas** suelen bastar para datos tabulares.
- **16-128 neuronas** por capa oculta para datasets pequeños.
- **Más capas/neuronas** para datos complejos (imágenes).
- **Más datos** permiten redes más grandes sin sobreajustar.

## Capas especiales y Anti-patrones

**Capas que mejoran tu red:**
- **BatchNorm**: normaliza en el mini-batch, acelerando el aprendizaje.
- **Dropout**: apaga aleatoriamente neuronas en *train* para evitar sobreajuste.

**Errores comunes que debes evitar:**
- **Sigmoid en hidden layers**: causa "vanishing gradient". Usa ReLU.
- **Activar mal el output**: si predices 0-10, sigmoid te limitará a 0-1 (¡error!).
- **Dropout en testing**: PyTorch lo apaga solo con `model.eval()`.
- **Demasiadas capas con poca data**: overfitting garantizado.
