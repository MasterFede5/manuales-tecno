# 7.4 PyTorch: estilo recetario (tensors, nn.Module, optimizer)

PyTorch domina la investigación y producción moderna en deep learning. 
Todo el framework se basa en tres componentes que interactúan: 
**Tensors** (los datos), **nn.Module** (el modelo) y **Optimizer** (el entrenamiento).

::concepto{titulo="PyTorch básico"}
PyTorch es la librería más intuitiva y "Pythónica" del ecosistema.
Sus 3 componentes clave son:

- **Tensor**: array multi-dimensional similar a NumPy (soporta GPU y auto-gradientes).
- **nn.Module**: clase base para definir la arquitectura de tu red neuronal.
- **Optimizer**: algoritmo que ajusta los pesos basándose en el cálculo de errores.

```python
import torch
import torch.nn as nn
import torch.optim as optim
```
::/concepto

::interioriza{titulo="La analogía de la cocina"}
Piensa en PyTorch como una cocina profesional para tus modelos:
- **Tensors**: Son tus **ingredientes** crudos, organizados y listos para cocinar.
- **nn.Module**: Es tu **horno/receta** que procesa y transforma los ingredientes.
- **Optimizer**: Es el **chef ejecutivo** que prueba el plato y mejora la receta (pesos).
::/interioriza

## Recetas Base

- **Receta 1: Ingredientes (Tensor)**
  Transformamos DataFrames de Pandas a formato nativo de PyTorch.
  `unsqueeze(1)` alinea las dimensiones para problemas de regresión (`[n] -> [n, 1]`).

```python
import pandas as pd
import torch

df = pd.read_csv("dataset.csv")
X = torch.tensor(df[["horas", "asistencia"]].values, dtype=torch.float32)
y = torch.tensor(df["cal_final"].values, dtype=torch.float32).unsqueeze(1)
```

- **Receta 2: El Horno (nn.Module / nn.Sequential)**
  Alineamos capas de neuronas secuencialmente. 
  Es ideal para arquitecturas directas que no requieren lógicas complejas.

```python
model = nn.Sequential(
    nn.Linear(2, 16),    # 2 entradas → 16 neuronas
    nn.ReLU(),
    nn.Linear(16, 1)     # 1 salida (regresión)
)
```

- **Receta 3: El Chef Cocinando (Loop de Entrenamiento)**
  El proceso se repite por varias *epochs* o iteraciones.
  Calculamos error (`loss`), reseteamos al chef (`zero_grad`), y aplicamos ajuste (`step`).

```python
criterion = nn.MSELoss()
optimizer = optim.Adam(model.parameters(), lr=0.001)

for epoch in range(100):
    pred = model(X)                # 1. Probar (Forward)
    loss = criterion(pred, y)      # 2. Evaluar error
    
    optimizer.zero_grad()          # 3. Limpiar memoria del chef
    loss.backward()                # 4. Calcular qué ajustar
    optimizer.step()               # 5. Aplicar ajuste
```

## Anti-patrones Comunes

- **Olvidar `optimizer.zero_grad()`**: acumula gradientes pasados, arruinando el aprendizaje.
- **Evaluar sin `model.eval()`**: capas como Dropout siguen activas, fallando predicciones.
- **Usar listas nativas de Python**: PyTorch solo procesa sus propios tensores optimizados.
- **Falta de DataLoader (Batches)**: causa *Out Of Memory* (OOM) en datasets grandes.
