# 2.1 NumPy: arrays y operaciones básicas

NumPy (*Numerical Python*) es la **base** del ecosistema de datos en Python. 
- Sustenta librerías clave (Pandas, scikit-learn, PyTorch).
- Dominarlo facilita aprender todo lo demás.

::concepto{titulo="NumPy y arrays"}
**NumPy** es la librería clave de cómputo numérico. 
Su pieza central es el **`ndarray`** (n-dimensional array), similar a una lista pero:
- **Homogénea**: Todos sus elementos son del mismo tipo.
- **Vectorizada**: Aplica operaciones a todos sin bucles explícitos.

```python
import numpy as np

calificaciones = np.array([8.5, 6.0, 9.5, 5.5, 7.8])
# Operación vectorizada
porcentaje = calificaciones * 10  # array([85., 60., 95., 55., 78.])
```

Ventajas principales:
- **10-100× más rápido** que las listas.
- **Sintaxis natural** para operar elemento a elemento.
- **Matemática masiva**: media, min, max directos.
- **Multidimensional**: desde 1D hasta tensores 3D.
::/concepto

::interioriza{titulo="Analiza la eficiencia"}
Piensa en una lista de Python como lavar platos a mano: lavas uno por uno (bucle `for`).
Usar NumPy es como usar un lavavajillas industrial: cargas todo y se lava en un solo ciclo (vectorización).
Esta diferencia reduce procesos de 100 segundos a solo 1 segundo.
::/interioriza

## Crear arrays

Para inicializar arrays puedes usar:
- Listas convencionales.
- Funciones generadoras (`arange`, `linspace`).
- Rellenos automáticos (`zeros`, `ones`, `full`).

```python
import numpy as np

# Desde lista
a = np.array([1, 2, 3, 4, 5])

# Rangos
b = np.arange(0, 10)         # [0,1,2,...,9]
c = np.linspace(0, 1, 5)     # [0, 0.25, 0.5, 0.75, 1.0]

# Con valor inicial
zeros = np.zeros(5)          # [0., 0., 0., 0., 0.]
ones = np.ones(5)            # [1., 1., 1., 1., 1.]
fives = np.full(5, 5.0)      # [5., 5., 5., 5., 5.]

# 2D (matriz)
matriz = np.array([[1, 2, 3], [4, 5, 6]])
print(matriz.shape)          # (2, 3) — 2 filas, 3 columnas
```

## Operaciones vectorizadas

Aplica matemática directamente sobre todo el conjunto de datos:
- Usa operadores estándar (`+`, `*`, `**`).
- Accede a métodos estadísticos integrados.

```python
calificaciones = np.array([8.5, 6.0, 9.5, 5.5, 7.8])

# Aritméticas
calificaciones + 1            # array([9.5, 7. , 10.5, 6.5, 8.8])
calificaciones * 0.5
calificaciones ** 2

# Estadísticas
calificaciones.mean()         # 7.46
calificaciones.std()          # 1.49
calificaciones.min()          # 5.5
calificaciones.max()          # 9.5
calificaciones.sum()          # 37.3
np.percentile(calificaciones, 50)  # 7.8 (mediana)
```

## Indexación y slicing

Extrae datos igual que en las listas, pero con mejoras:
- Acceso por índices y rangos.
- **Boolean indexing**: filtrado mediante máscaras de verdadero/falso.

```python
a = np.array([10, 20, 30, 40, 50])

a[0]           # 10
a[-1]          # 50
a[1:4]         # array([20, 30, 40])

# Boolean indexing (filtrado)
mask = a > 25
a[mask]        # array([30, 40, 50])

# Múltiples condiciones
a[(a > 15) & (a < 45)]  # array([20, 30, 40])
```

## Operaciones entre arrays

NumPy alinea arrays del mismo tamaño para operar:
- Ideal para cálculos paralelos entre variables.
- Permite lógica combinada con operadores como `&` o `|`.

```python
horas = np.array([5, 6, 4, 7, 5])
asistencia = np.array([0.9, 0.85, 0.95, 0.7, 0.92])

# Operaciones elemento a elemento
indice_esfuerzo = horas * asistencia
# array([4.5, 5.1, 3.8, 4.9, 4.6])

# Comparación elemento a elemento
sobresalientes = (horas >= 5) & (asistencia >= 0.85)
# array([True, True, False, False, True])
```

## Agregaciones por eje (en 2D)

En matrices, el parámetro `axis` dirige la operación:
- `axis=0`: Opera sobre columnas (ej. promedio por materia).
- `axis=1`: Opera sobre filas (ej. promedio por alumno).

```python
notas = np.array([
    [8, 7, 9],   # alumno 1: mate, fis, qui
    [6, 5, 7],   # alumno 2
    [9, 9, 10]   # alumno 3
])

# Promedio por alumno (filas, axis=1)
notas.mean(axis=1)   # array([8., 6., 9.33])

# Promedio por materia (columnas, axis=0)
notas.mean(axis=0)   # array([7.67, 7., 8.67])
```

## Reshape y transposición

Adapta la estructura de los datos sin alterar su contenido:
- `reshape()`: Cambia las dimensiones (ej. de 1D a 2D).
- El valor `-1` calcula automáticamente esa dimensión.
- `.T`: Intercambia filas por columnas (transpuesta).

```python
a = np.arange(12)              # 1D, 12 elementos
a.reshape(3, 4)                # 2D, 3x4
a.reshape(2, 2, 3)             # 3D
a.reshape(-1, 4)               # -1 = "calcula tú", da 3x4

# Transpuesta
m = np.array([[1, 2], [3, 4]])
m.T                            # array([[1, 3], [2, 4]])
```

## NaN (Not a Number) — datos faltantes

En la vida real los datos están incompletos:
- `np.nan` representa valores nulos o faltantes.
- Un solo `nan` contamina funciones normales (como `mean`).
- Usa funciones seguras como `nanmean` para ignorarlos.

```python
notas = np.array([8.5, np.nan, 7.0, np.nan, 9.0])

notas.mean()              # nan (un nan contamina)
np.nanmean(notas)         # 8.17 (ignora nans)
np.isnan(notas)           # array([False, True, False, True, False])
```

## Anti-patrones

- **Bucle manual**: Usar `for i in range(...)` desperdicia la vectorización de NumPy.
- **Mezclar tipos**: `np.array([1, "hola"])` convierte todo a texto sin avisar.
- **Operar con NaN**: Usar `.mean()` con faltantes da `nan`. Emplea `nanmean`.
- **Asumir dimensiones**: Bug típico de forma incorrecta. Imprime `.shape` al depurar.
- **Reinventar la rueda**: NumPy ya tiene `sort`, `unique`, `diff`, etc. Búscalas antes.
