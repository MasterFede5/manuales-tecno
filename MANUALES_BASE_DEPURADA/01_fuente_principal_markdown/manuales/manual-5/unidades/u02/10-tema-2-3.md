# 2.3 Lectura: CSV, Excel, JSON

- En la unidad previa leíste un CSV usando 10 líneas de Python.
- Pandas hace exactamente lo mismo en 1 sola línea.
- Esta lección cubre los 3 formatos más usados en datos.

::concepto{titulo="Lectura con pandas"}
- Pandas tiene funciones `read_*` para cada formato de archivo.
- Las más comunes son para CSV, Excel y JSON.

```python
import pandas as pd

# CSV
df = pd.read_csv("archivo.csv")

# Excel
df = pd.read_excel("archivo.xlsx", sheet_name="Hoja1")

# JSON
df = pd.read_json("archivo.json")
```

- Existen **decenas de parámetros** para ajustar la lectura (ej. separadores).
::/concepto

::interioriza{titulo="¿Por qué te importa? (El puerto universal)"}
- **Analogía:** Imagina que pandas es un puerto marítimo que recibe barcos de todo el mundo.
- Sin importar si traen cargamento en cajas (CSV), contenedores (Excel) o palets (JSON).
- Pandas descarga todo y lo acomoda en un único tipo de bodega: el **DataFrame**, listo para trabajar en un solo formato estándar.
::/interioriza

## CSV: la función estrella

```python
df = pd.read_csv(
    "dataset_escolar.csv",
    encoding="utf-8",          # maneja acentos
    sep=",",                    # puede ser ";" o "\t"
    decimal=".",                # puede ser ","
    parse_dates=["fecha"],      # detecta fechas
    index_col="id",             # usa columna como índice
    na_values=["N/A", "??"],   # marca nulos
    nrows=1000                  # lee solo 1000 filas
)
```

- El 99 % del tiempo usarás `pd.read_csv("archivo.csv")` sin parámetros extra.
- El otro 1 % requerirá banderas especiales, como separadores europeos o fechas.

## Excel

```python
# Cargar una sola hoja
df = pd.read_excel("calificaciones.xlsx", sheet_name="2026")

# Cargar todas las hojas
todas = pd.read_excel("calificaciones.xlsx", sheet_name=None)
# 'todas' es un diccionario: {nombre_hoja: DataFrame}
```

- *Nota:* Requiere tener instalada la librería `openpyxl`.

## JSON

```python
# JSON simple (lista de objetos)
df = pd.read_json("alumnos.json")

# JSON anidado
import json
with open("nested.json", "r") as f:
    raw = json.load(f)
df = pd.json_normalize(raw["alumnos"])
```

- JSON suele ser más complejo por su estructura variable.
- CSV y Excel son formatos mucho más planos y predecibles.

## Escribir (export)

```python
df.to_csv("salida.csv", index=False, encoding="utf-8")
df.to_excel("salida.xlsx", index=False, sheet_name="Resultado")
df.to_json("salida.json", orient="records", indent=2)
```

- Usa `index=False` para evitar que el índice se exporte como una columna extra.

## Casos especiales frecuentes

- **CSV europeo:** Usa coma para decimales y punto y coma para separar datos.
  `pd.read_csv("archivo.csv", sep=";", decimal=",")`
- **Encoding de Windows:** Si hay caracteres raros, prueba codificaciones antiguas.
  `pd.read_csv("archivo.csv", encoding="latin1")`
- **Múltiples archivos:** Combínalos al vuelo con `glob` y una lista de comprensión.
  `archivos = glob.glob("data/*.csv")`
  `df = pd.concat([pd.read_csv(f) for f in archivos])`

## Anti-patrones a evitar

- **Confiar a ciegas:** Siempre revisa `df.head()` y `df.dtypes` tras leer.
- **Exportar índices inútiles:** Olvidar `index=False` ensucia tu salida.
- **Asfixiar la memoria:** Cargar +1 GB completo. Usa `chunksize` para procesar por lotes.
- **Asumir que Pandas es adivino:** Si infiere mal un tipo, fuérzalo manualmente.
