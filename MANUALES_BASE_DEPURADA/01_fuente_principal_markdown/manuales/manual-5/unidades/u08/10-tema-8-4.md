# 8.4 Embeddings y búsqueda semántica básica

- Los embeddings convierten textos en listas de números.
- La búsqueda semántica encuentra similitud por significado.
- A diferencia de palabras exactas, capta el sentido.

::concepto{titulo="Embeddings y búsqueda semántica"}
**Embedding**: vector numérico (ej. 1024 dimensiones). 
- Representa el "significado" del texto.
- Textos similares producen vectores cercanos en el espacio.

**Búsqueda semántica**:
- Tomas una pregunta (query).
- Buscas los textos más cercanos matemáticamente.
- Comparas la distancia entre sus embeddings.

```python
# Pseudocódigo
embedding_query = embed("¿cómo apelar calificación?")
embedding_chunks = embed(reglamento_chunks)
top_3 = busqueda_similitud(embedding_query, embedding_chunks)
```

**Usos comunes:**
- RAG (Generación Aumentada por Recuperación).
- Sistemas de recomendación y deduplicación.
::/concepto

::interioriza{titulo="Analogía: El archivista intuitivo"}
Imagina un archivista de la escuela que no lee palabras exactas, sino que entiende ideas.
Si le pides "apelación de nota", no buscará esa palabra exacta. 
Te dará el documento de "impugnación de calificación" porque sabe que apuntan a lo mismo.
::/interioriza

## Generar embeddings con OpenAI

- Usaremos la API de OpenAI para transformar texto.
- El modelo `text-embedding-3-small` es rápido y barato.

```python
from openai import OpenAI
import os

client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))

def embed(texto: str) -> list:
    response = client.embeddings.create(
        model="text-embedding-3-small",
        input=texto
    )
    return response.data[0].embedding

vector = embed("¿cómo apelar calificación?")
print(len(vector))  # 1536 dimensiones
```

- Costo: aprox. $0.02 por millón de tokens.
- Puedes procesar reglamentos enteros por centavos.

## Búsqueda por similitud (coseno)

- La similitud coseno mide el ángulo entre dos vectores.
- 1 significa idénticos, 0 significa sin relación.

```python
import numpy as np

def similitud_coseno(v1, v2):
    return np.dot(v1, v2) / (np.linalg.norm(v1) * np.linalg.norm(v2))

# 1. Documentos
docs = ["Plazo para apelar: 5 días.", "Asistencia mínima: 80%."]

# 2. Embeddings de cada doc
emb_docs = [embed(d) for d in docs]

# 3. Query
query = "¿cuántos días para impugnar?"
emb_query = embed(query)

# 4. Buscar top
sims = [similitud_coseno(emb_query, ed) for ed in emb_docs]
mejor = np.argmax(sims)
print(f"Mejor: '{docs[mejor]}' (similitud={sims[mejor]:.3f})")
```

## Vector stores

- Con miles de documentos, calcular similitudes a mano es muy lento.
- Los **Vector stores** son bases de datos para embeddings.

**Opciones principales:**
- **ChromaDB**: Simple y local. Ideal para prototipos rápidos.
- **Pinecone / Weaviate**: En la nube, máxima escalabilidad.
- **pgvector**: Extensión para usar en bases PostgreSQL.

```python
import chromadb

client = chromadb.Client()
collection = client.create_collection("reglamento")

# Indexar fragmentos
collection.add(
    documents=["chunk 1", "chunk 2"],
    embeddings=[[0.1,...], [0.2,...]],
    ids=["id1", "id2"]
)

# Buscar los más relevantes
results = collection.query(
    query_embeddings=[emb_query],
    n_results=3
)
```

## Modelos de embeddings

- No todos los modelos sirven para todo.
- Elige balanceando dimensiones, idioma y costo.

| Modelo | Dimensiones | Idiomas | Costo /M |
|---|---|---|---|
| OpenAI `text-embedding-3-small` | 1536 | Múltiples | $0.02 |
| OpenAI `text-embedding-3-large` | 3072 | Múltiples | $0.13 |
| BGE-m3 (open) | 1024 | Excelente multilingüe | $0 (local) |

- **Tip:** Para textos académicos en español, BGE-m3 es excepcional.

## Anti-patrones

- **Mezclar proveedores:** Crear embeddings con OpenAI y buscar con otro modelo. Fallará.
- **Falta de chunking:** Convertir un PDF entero en un solo vector diluye detalles clave.
- **Sin base de datos:** Usar cálculos manuales en producción con miles de textos.
