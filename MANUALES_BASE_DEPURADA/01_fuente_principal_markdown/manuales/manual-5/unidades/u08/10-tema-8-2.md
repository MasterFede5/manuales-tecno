# 8.2 Tu primer chatbot en 30 líneas

Un chatbot real mantiene una conversación fluida porque recuerda lo dicho anteriormente.
Para lograr coherencia en las respuestas, la clave es dominar el **historial de mensajes**.

::concepto{titulo="Chatbot con historial"}
Un **chatbot conversacional** mantiene el contexto enviando el historial completo en cada llamada.

Su estructura básica utiliza una lista de mensajes:
```python
messages = [
    {"role": "system", "content": "Instrucciones generales..."},
    {"role": "user", "content": "Mensaje 1"},
    {"role": "assistant", "content": "Respuesta 1"},
    {"role": "user", "content": "Mensaje 2"},
    # ...
]
```

En cada turno, la secuencia es siempre la misma:
- Agregas el nuevo mensaje del usuario a la lista.
- Envías la lista completa al modelo de IA.
- Agregas la respuesta generada al historial para el futuro.
::/concepto

::interioriza{titulo="El pez Dory y el mesero"}
Piensa en una API sin historial como Dory (de Buscando a Nemo). Si le pides una hamburguesa y luego dices "sin cebolla", ya olvidó la hamburguesa.
El historial convierte a la API en un mesero experto que anota todo en su libreta para entender el contexto completo de tu pedido.
::/interioriza

## Chatbot básico (Claude)

Con este código puedes crear un chatbot en la terminal:

```python
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ.get("ANTHROPIC_API_KEY"))

messages = []
system_prompt = "Eres un asistente educativo conciso y amable."

print("Chatbot listo. Escribe 'salir' para terminar.\n")

while True:
    user_msg = input("Tú: ")
    if user_msg.lower() == "salir":
        break
    
    messages.append({"role": "user", "content": user_msg})
    
    response = client.messages.create(
        model="claude-haiku-4-5",
        system=system_prompt,
        max_tokens=500,
        messages=messages
    )
    
    assistant_msg = response.content[0].text
    messages.append({"role": "assistant", "content": assistant_msg})
    
    print(f"\nClaude: {assistant_msg}\n")
```

Son solo 30 líneas para tener un chatbot funcional con memoria activa.

## System prompt: el rol del asistente

El *system prompt* es persistente e influye en **todos** los turnos de la conversación.

```python
system_prompt = """Tu rol específico aquí.

Reglas:
- Sé conciso.
- Cita datos específicos cuando sea posible.
- No inventes información que no tengas.
- Si no sabes, dilo."""
```

Define las reglas de comportamiento y el formato de salida desde el principio.

## Limitaciones del historial

Cada llamada envía **todo** el historial, por lo que el costo crece linealmente.
Para evitar que sesiones largas agoten tu presupuesto o límite de tokens:

- **Límites fijos:** Quédate solo con los últimos 20 mensajes.
```python
messages = messages[-20:]
```

- **Resúmenes periódicos:** Condensa la conversación antigua.
```python
if len(messages) > 30:
    summary = client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=200,
        messages=[{"role": "user", "content": f"Resume esta conversación: {messages}"}]
    )
    messages = [{"role": "user", "content": f"Contexto previo: {summary}"}]
```

## Anti-patrones

Evita estas prácticas comunes al diseñar tu chatbot:
- **Omitir el system prompt:** El bot pierde su identidad y rol específico.
- **Permitir un historial infinito:** El costo se vuelve incontrolable.
- **Ignorar errores de API:** Un solo fallo de red matará todo el proceso.
