# 8.1 Configurar API de Claude / OpenAI

Para usar IA generativa programáticamente necesitas API key + SDK.
Esta lección te lleva del setup al primer "Hola" en 15 minutos.

::concepto{titulo="API de modelo generativo"}
Las **APIs de IA generativa** te permiten usar modelos desde código en lugar del chat web.
Ventajas clave:
- **Automatización**: procesar 1000 textos sin copiar y pegar.
- **Integración**: conectar la IA directo en tu app, web o workflow.
- **Control fino**: ajustar temperature, max_tokens y system prompt.

Setup canónico paso a paso:
1. Crear cuenta en proveedor.
2. Generar API key (tu llave de acceso).
3. Guardar key como variable de entorno (nunca en código).
4. Instalar SDK (librería de Python).
5. Primer call a la API.
::/concepto

::interioriza{titulo="Analogía: El empleado y la ventanilla"}
Usar el chat web es como ir al banco y hablar con el cajero: es manual y lento.
Usar la API es como darle una tarjeta corporativa a tu software.
Tu programa hace los trámites por detrás, miles de veces por segundo, sin que tú intervengas.
::/interioriza

## Anthropic Claude

Primero instalamos el SDK de Anthropic:
```bash
pip install anthropic
```

Código mínimo para hacer la petición:
```python
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ.get("ANTHROPIC_API_KEY"))

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "Hola, explícame en 2 frases qué es ML."}
    ]
)
print(response.content[0].text)
```

## OpenAI

El proceso con OpenAI es casi idéntico. Instalamos el SDK:
```bash
pip install openai
```

Y ejecutamos el código:
```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))

response = client.chat.completions.create(
    model="gpt-5",
    messages=[
        {"role": "user", "content": "Hola, explícame en 2 frases qué es ML."}
    ]
)
print(response.choices[0].message.content)
```

## Variables de entorno (seguridad)

**NUNCA** pegues API keys directamente en tu código.
Si lo subes a un repositorio público, un bot te robará la clave.
Tres formas correctas de manejar variables de entorno:

### En Colab
Usa el panel de secretos y cárgalo con la librería `userdata`:
```python
from google.colab import userdata
api_key = userdata.get('ANTHROPIC_API_KEY')
```

### En script local con .env
Crea un archivo llamado `.env` (agrega esto al `.gitignore`):
```bash
# .env (no subir a Git, agregar a .gitignore)
ANTHROPIC_API_KEY=sk-ant-xxx
```

Léelo en Python con la librería `python-dotenv`:
```python
from dotenv import load_dotenv
load_dotenv()
```

### En terminal antes
Exporta la variable directamente en la consola:
```bash
export ANTHROPIC_API_KEY="sk-ant-xxx"
python script.py
```

## Costos típicos (2025)

Las APIs se cobran por millón (M) de tokens procesados.

| Proveedor | Modelo | Input | Output |
|---|---|---|---|
| Anthropic | Sonnet 4.6 | $3/M tokens | $15/M tokens |
| Anthropic | Opus 4.7 | $15/M | $75/M |
| OpenAI | GPT-5 | $5/M | $15/M |
| OpenAI | GPT-5-mini | $0.50/M | $1.50/M |

- **Para experimentar:** Usa modelos ligeros (Haiku o GPT-5-mini).
- Costarán apenas unos centavos por cada sesión típica.

## Anti-patrones

- **API key en código fuente:** La filtras en GitHub, te llega factura de miles de dólares.
- **Sin límite de tokens:** No definir `max_tokens` descontrola los costos.
- **Probar con Opus/GPT-5:** Gastas dinero innecesario; usa mini o haiku para tests.
