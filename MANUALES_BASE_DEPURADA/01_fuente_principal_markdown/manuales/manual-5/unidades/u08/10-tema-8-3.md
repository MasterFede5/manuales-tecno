# 8.3 Streaming y tool use (introducción)

Dos *features* elevan tu chatbot al siguiente nivel:
- **Streaming:** El texto aparece progresivamente (mejor UX).
- **Tool use:** El modelo invoca tus propias funciones de Python.

::concepto{titulo="Streaming y tool use"}
- **Streaming:** Recibes tokens conforme se generan.
- Evita esperar la respuesta completa.
- Da una experiencia de usuario (UX) estilo ChatGPT.

- **Tool use:** Defines funciones Python.
- El modelo decide cuándo y con qué argumentos invocarlas.
- Conecta el bot con bases de datos, APIs o archivos.

Ambos son estándares en modelos modernos (2024-25).
::/concepto

::interioriza{titulo="Baja Curva: El Mesero y el Chef"}
Piensa en el **Streaming** como un mesero que trae las bebidas mientras esperas tu comida. Sientes que te atienden rápido, aunque el plato tarde lo mismo.
Piensa en el **Tool use** como darle al mesero un datáfono: ya no solo platica contigo, ahora tiene "herramientas" para actuar en el mundo real.
::/interioriza

## Streaming en código

```python
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ.get("ANTHROPIC_API_KEY"))

with client.messages.stream(
    model="claude-haiku-4-5",
    max_tokens=500,
    messages=[{"role": "user", "content": "Cuenta una historia corta"}]
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)

print()  # Newline al final
```

- Resultado: el texto aparece palabra por palabra.
- La percepción del usuario es de gran velocidad.
- La latencia total es la misma, pero la UX mejora drásticamente.

## Tool use básico

Defines funciones que el modelo puede invocar:

```python
import json

def get_grade_info(student_id: int) -> dict:
    """Consulta calificaciones del alumno desde BD interna."""
    # En real: query a BD
    return {"id": student_id, "cal_final": 7.2, "asistencia": 0.85}

tools = [{
    "name": "get_grade_info",
    "description": "Consulta info de calificaciones de un alumno por ID",
    "input_schema": {
        "type": "object",
        "properties": {
            "student_id": {"type": "integer", "description": "ID del alumno"}
        },
        "required": ["student_id"]
    }
}]

# Loop con tool use
messages = [{"role": "user", "content": "¿Cómo va el alumno 124?"}]
while True:
    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1000,
        tools=tools,
        messages=messages
    )
    
    if response.stop_reason == "tool_use":
        # Modelo quiere invocar función
        for block in response.content:
            if block.type == "tool_use":
                result = get_grade_info(**block.input)
                
                messages.append({"role": "assistant", "content": response.content})
                messages.append({
                    "role": "user",
                    "content": [{
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": json.dumps(result)
                    }]
                })
    else:
        # Respuesta final
        print(response.content[0].text)
        break
```

## Caso aplicado al Asistente Predictor

```python
def predecir_alumno(features: dict) -> dict:
    """Usa tu modelo de ML entrenado en U5/U7."""
    import joblib
    model = joblib.load("modelo_predictor_v1.pkl")
    pred = model.predict([list(features.values())])[0]
    return {"prediccion": float(pred), "riesgo": "alto" if pred < 6 else "bajo"}

# Tool definition
tools = [{
    "name": "predecir_alumno",
    "description": "Predice calificación final de alumno basándose en features",
    "input_schema": {...}
}]

# Usuario: "Predice para Andrea: estudia 4 horas, asiste 80%, cal anterior 6.5"
# Claude → invoca predecir_alumno({"horas_estudio": 4, ...})
# Recibe predicción → la explica en lenguaje natural
```

- Tu modelo predictor de ML se une al razonamiento de Claude.
- El resultado es un sistema capaz de predecir el riesgo y explicarlo.

## Anti-patrones

- **Tool use con permisos amplios:** Si tu función modifica BD, exige validación humana previa.
- **Sin streaming en interfaces lentas:** El usuario pensará que el bot se trabó.
- **Loop infinito sin límite:** El modelo podría invocar tools en bucle infinito. Usa `max_iterations`.
