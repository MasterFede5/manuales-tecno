# 8.7 Buenas prácticas: costo, manejo de errores, caching

App demo + buenas prácticas = app productiva.
Esta lección cubre las que no debes saltarte.

::concepto{titulo="Buenas prácticas en apps con APIs de IA"}
Cuatro categorías preventivas:
- **Costo**: caching, elección de modelo, límite de tokens, batching.
- **Manejo de errores**: try/except, retry con backoff, fallbacks.
- **Seguridad**: protección de API keys y validación estricta de inputs.
- **Monitoreo**: observabilidad mediante logs y métricas de uso.

Previenen problemas predecibles que matan apps en producción.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Piensa en una casa. Sin cimientos (buenas prácticas), se ve bien un día, pero se cae con el viento.

Demos sin estas prácticas funcionan un día en tu computadora.
Apps con ellas funcionan por años. La diferencia entre proyecto y producto está aquí.
::/interioriza

## Costo: 4 técnicas clave

### 1. Streamlit caching

```python
@st.cache_resource
def cargar_modelo():
    return joblib.load("modelo.pkl")

@st.cache_data(ttl=3600)  # cache 1 hora
def consultar_api(prompt):
    return claude.messages.create(...)
```

`cache_resource` para modelos (no se serializa). `cache_data` para resultados.

### 2. Elegir modelo apropiado

```python
# Tareas simples (clasificación, extracción)
modelo = "claude-haiku-4-5"  # $0.80 / $4 por M tokens

# Razonamiento complejo
modelo = "claude-sonnet-4-6"  # $3 / $15

# Solo lo crítico
modelo = "claude-opus-4-7"  # $15 / $75
```

### 3. Limitar tokens

```python
response = claude.messages.create(
    max_tokens=300,  # no 4000 si esperas 200
    ...
)
```

### 4. Prompt caching (Anthropic)

```python
response = claude.messages.create(
    model="claude-sonnet-4-6",
    system=[
        {"type": "text", "text": system_prompt_largo, "cache_control": {"type": "ephemeral"}}
    ],
    messages=...
)
```

Reduce costo del system prompt repetido a 10%.

## Manejo de errores

```python
import time
from anthropic import APIError, RateLimitError

def llamar_api_con_retry(prompt, max_retries=3):
    for intento in range(max_retries):
        try:
            response = claude.messages.create(
                model="claude-sonnet-4-6",
                max_tokens=500,
                messages=[{"role": "user", "content": prompt}]
            )
            return response.content[0].text
        except RateLimitError:
            wait = 2 ** intento  # exponential backoff
            time.sleep(wait)
        except APIError as e:
            print(f"Error API: {e}")
            return "Lo siento, no pude procesar tu solicitud."
    
    return "Servicio no disponible. Intenta luego."
```

## Monitoreo

```python
import logging
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

def llamar_api(prompt, user_id):
    start = time.time()
    try:
        result = claude.messages.create(...)
        duration = time.time() - start
        
        logger.info({
            "user": user_id,
            "tokens_in": result.usage.input_tokens,
            "tokens_out": result.usage.output_tokens,
            "duration": duration,
            "model": "claude-sonnet-4-6"
        })
        return result
    except Exception as e:
        logger.error({"user": user_id, "error": str(e)})
        raise
```

## Checklist pre-deploy

- [ ] API keys en secrets/env, NO en código.
- [ ] Caching configurado para modelo y consultas frecuentes.
- [ ] Try/except en cada llamada a API externa.
- [ ] Retry con backoff para rate limits.
- [ ] Fallback graceful si API cae.
- [ ] max_tokens límite razonable.
- [ ] Logs de uso (tokens, errores).
- [ ] Validación de inputs del usuario.
- [ ] README con setup claro.
- [ ] Test con 5 usuarios reales antes de promover.

## Anti-patrones

Evita estos errores comunes que arruinan apps:
- **Sin caching**: Cada interacción cuesta completo, elevando gastos exponencialmente.
- **Sin manejo de errores**: Si la API falla, toda la app colapsa.
- **API keys en repo público**: Bots las usan y vacían tu saldo en minutos.
- **Sin logs**: Imposible debugear o entender qué falló en producción.
- **Demasiados tokens**: Facturas sorpresa por no limitar salidas.
