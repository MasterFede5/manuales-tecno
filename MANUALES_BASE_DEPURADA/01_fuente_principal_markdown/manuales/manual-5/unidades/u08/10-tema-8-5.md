# 8.5 Mini-RAG sobre 3 documentos

- Combinas embeddings y LLM para responder usando tus propios documentos.
- Esta lección construye un pipeline RAG completo en código.

::concepto{titulo="Mini-RAG"}
**RAG** (Retrieval-Augmented Generation) consta de:
- **Indexación**: cortas el texto (chunks), sacas su embedding y guardas.
- **Consulta**: sacas el embedding de la pregunta y buscas los top-k chunks.
- **Generación**: pasas los chunks al LLM para responder citando.

Versión "mini": todo en menos de 100 líneas.
::/concepto

::interioriza{titulo="Baja Curva: El bibliotecario eficiente"}
- Imagina que el LLM es un orador brillante pero desmemoriado. 
- El RAG actúa como su bibliotecario personal. 
- Al preguntar sobre el "reglamento escolar" (nuestro caso), el bibliotecario busca la página exacta y se la da al orador para que no invente.
::/interioriza

## Mini-RAG completo (~50 líneas)

- Este es un ejemplo funcional usando OpenAI y Anthropic.
- Primero cargamos y partimos el texto (chunking).

```python
import os
import numpy as np
from anthropic import Anthropic
from openai import OpenAI

claude = Anthropic(api_key=os.environ.get("ANTHROPIC_API_KEY"))
openai_client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))

# 1. Cargar y chunkear
def cargar_y_chunkear(path, chunk_size=400):
    with open(path, "r", encoding="utf-8") as f:
        texto = f.read()
    
    palabras = texto.split()
    chunks = []
    for i in range(0, len(palabras), chunk_size):
        chunks.append({
            "texto": " ".join(palabras[i:i+chunk_size]),
            "fuente": path,
            "indice": i // chunk_size
        })
    return chunks

# 2. Embedder
def embed(texto):
    return openai_client.embeddings.create(
        model="text-embedding-3-small",
        input=texto
    ).data[0].embedding

# 3. Indexar
def indexar(rutas):
    todos = []
    for r in rutas:
        chunks = cargar_y_chunkear(r)
        for c in chunks:
            c["embedding"] = embed(c["texto"])
            todos.append(c)
    return todos

# 4. Buscar top-k
def buscar(query, indice, k=3):
    e_q = embed(query)
    sims = [
        np.dot(e_q, c["embedding"]) / (np.linalg.norm(e_q) * np.linalg.norm(c["embedding"]))
        for c in indice
    ]
    top_idx = np.argsort(sims)[::-1][:k]
    return [indice[i] for i in top_idx]

# 5. Generar respuesta
def responder(query, contexto):
    prompt = f"""Responde la pregunta usando exclusivamente los siguientes fragmentos.
Si la información no está, di "no la encontré en los documentos".
Cita la fuente al final entre [corchetes].

Fragmentos:
{chr(10).join([f"[{c['fuente']} chunk {c['indice']}]" + chr(10) + c['texto'] for c in contexto])}

Pregunta: {query}"""
    
    response = claude.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=500,
        messages=[{"role": "user", "content": prompt}]
    )
    return response.content[0].text

# === Uso ===
indice = indexar(["reglamento.txt", "manual_docente.txt", "politica.txt"])

query = "¿Cuántos días tengo para apelar una calificación?"
contexto = buscar(query, indice, k=3)
respuesta = responder(query, contexto)
print(respuesta)
```

Output ejemplo:
```text
Tienes 5 días hábiles después de la publicación de la calificación 
para apelar [reglamento.txt chunk 7]. La solicitud debe entregarse 
por escrito a coordinación académica.
```

## Mejoras siguientes

- **Persistir vector store** (ChromaDB) para no recalcular embeddings.
- **Re-ranking** (Cohere) para mejorar los resultados más relevantes.
- **Hybrid search**: combinar similitud vectorial y palabras clave (BM25).
- **Citas estructuradas** usando metadatos limpios.

## Aplicado al caso escolar completo

- Así procesaríamos múltiples documentos institucionales:

```python
# Setup
docs = ["reglamento.txt", "manual_docente.txt", "politica_admisiones.txt"]
indice = indexar(docs)

# Query del coordinador
query = "Alumno con 50% asistencia, ¿qué dice el reglamento?"
contexto = buscar(query, indice, k=3)
respuesta = responder(query, contexto)
```

## Anti-patrones comunes

- **Sin chunking**: pasar todo el documento excede la ventana de contexto.
- **Chunks diminutos**: si usas 50 palabras, pierdes el hilo lógico.
- **Top-k excesivo**: retornar 20 chunks solo satura al modelo y lo confunde.
- **Sin regla de no inventar**: si no aclaras qué hacer sin datos, el modelo alucina.
