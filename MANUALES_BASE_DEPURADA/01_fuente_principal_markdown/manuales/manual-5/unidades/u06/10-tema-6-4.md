# 6.4 PCA básico (intuición + uso)

- Tienes 4 features: horas, asistencia, calificación anterior, calificación final.
- No es posible graficar en 4D para entender los datos visualmente.
- **PCA** proyecta estos datos a 2D o 3D, preservando la mayor variabilidad posible.

::concepto{titulo="PCA — Principal Component Analysis"}
- **PCA** es una técnica clave de **reducción de dimensionalidad**.
- Busca las direcciones de máxima varianza (componentes principales) en tus datos.
- Proyecta tu dataset original (N features) a un espacio menor (K componentes, K<N).
- El objetivo central es **perder la menor cantidad de información posible**.

```python
from sklearn.decomposition import PCA
pca = PCA(n_components=2)
X_2d = pca.fit_transform(X_scaled)

print(f"Varianza explicada: {pca.explained_variance_ratio_}")
# [0.55, 0.28] → componente 1 explica 55%, componente 2 explica 28%, total 83%
```
::/concepto

::interioriza{titulo="¿Por qué te importa? (Analiza la sombra)"}
- Imagina una tetera tridimensional iluminada por una linterna.
- Su sombra proyectada en la pared es una imagen en 2D.
- Dependiendo del ángulo de la linterna, la sombra revela más o menos la forma real de la tetera.
- PCA encuentra el ángulo perfecto para que la "sombra" muestre la mayor cantidad de detalles posible.
- **Usos prácticos clave:**
  - **Visualizar** datasets gigantes proyectados en 2D o 3D.
  - **Acelerar modelos** reduciendo drásticamente el número de columnas procesadas.
::/interioriza

## Caso uso 2: Acelerar modelos con muchos features

- Puedes decirle a PCA exactamente qué porcentaje de varianza quieres conservar.
- El algoritmo calculará matemáticamente cuántas componentes necesita para lograrlo.

```python
# De 100 features a un número que preserve el 95% de varianza
pca = PCA(n_components=0.95)
X_reducido = pca.fit_transform(X_scaled)
print(f"Features originales: {X_scaled.shape[1]} → Reducidas: {X_reducido.shape[1]}")
```

- Los modelos de Machine Learning entrenan muchísimo más rápido al procesar menos columnas.

## Cuándo usar PCA

| Escenario | PCA útil |
|---|---|
| Visualizar dataset con >3 features | ✓ Sí |
| Acelerar modelos pesados con >100 features | ✓ Sí |
| Reducir ruido removiendo componentes pequeños | ✓ Sí |
| Tienes apenas 3-5 features en total | ✗ No (innecesario) |
| Necesitas interpretar cada feature original | ✗ No (PCA mezcla las features) |

## Componentes principales: ¿qué son?

- Cada componente resultante es una **combinación lineal** de los features originales.
- Ya no son variables aisladas, sino "recetas" que mezclan los ingredientes originales.

```text
Componente 1 = 0.4*horas + 0.5*asistencia + 0.5*cal_ant + 0.5*cal_final
Componente 2 = 0.6*horas - 0.3*asistencia - 0.4*cal_ant + 0.5*cal_final
```

- El Componente 1 podría representar un "factor general de éxito" de un estudiante.
- El Componente 2 podría medir el "esfuerzo reciente vs historial pasado".

```python
print(pca.components_)
```

## Limitaciones de la técnica

- **Linealidad:** PCA solo captura relaciones lineales, perdiendo patrones curvos o complejos.
- **Interpretabilidad:** Los componentes generados pierden el significado humano directo.
- **Escalado obligatorio:** Es absolutamente vital escalar los datos antes de aplicar PCA.
- Para relaciones no lineales, se usan otras técnicas como t-SNE o UMAP.

## Anti-patrones a evitar

- **Olvidar escalar los datos:** La feature con mayor rango numérico dominará y arruinará el PCA.
- **Usar datos categóricos crudos:** PCA exige números, convierte tus categorías antes.
- **Fijar `n_components = N`:** Extraer la misma cantidad de features originales no reduce nada.
- **Interpretación literal:** Intentar explicar un componente como si fuera una feature cruda.
