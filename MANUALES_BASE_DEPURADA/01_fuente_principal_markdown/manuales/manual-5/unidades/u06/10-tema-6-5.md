# 6.5 Detección de anomalías simple

Las anomalías son datos que **no encajan** en tus clusters habituales.  
En el entorno escolar, representan alumnos con perfiles únicos.  
Estos casos extremos requieren tu atención porque indican genialidad o crisis.

::concepto{titulo="Detección de anomalías"}
Identifica puntos atípicos (outliers) en un dataset.  
Casos de uso comunes incluyen:
- Fraude financiero o transacciones raras.
- Fallas técnicas de sensores fuera de rango.
- Errores de captura de datos manual.

Principales métodos para detectarlos:
- **Estadístico:** IQR o z-score (vistos en la U4).
- **Clustering:** Distancia a los centroides (puntos muy lejanos).
- **Isolation Forest:** Aísla anomalías cortando los datos.
- **LOF (Local Outlier Factor):** Mide la densidad local.
::/concepto

::interioriza{titulo="Baja Curva: El sistema de alarmas"}
Imagina las anomalías como el detector de humo de la escuela.  
El detector no apaga el incendio, solo te avisa de que algo no es normal.  
En tus datos, un alumno con 100% de asistencia y notas bajas es el "humo".  
Te alerta de que hay que ir a revisar la situación, no asume la solución.
::/interioriza

## 1. Distancia al centroide (Clustering)

Este método asume que las anomalías están lejos de los grupos normales.

- **Mecánica:** Mides la distancia de cada punto a su centroide.
- **Filtro:** Te quedas con el 5% (o el umbral que decidas) de puntos más alejados.

```python
from sklearn.cluster import KMeans
import numpy as np

kmeans = KMeans(n_clusters=4, random_state=42, n_init=10)
kmeans.fit(X_scaled)

distancias = np.min(kmeans.transform(X_scaled), axis=1)
threshold = np.percentile(distancias, 95)
anomalias = df[distancias > threshold]
```

## 2. Isolation Forest

Algoritmo especializado y altamente eficiente para anomalías.

- **Mecánica:** Aísla puntos mediante divisiones aleatorias del espacio.
- **Lógica:** Las anomalías necesitan menos divisiones para quedar solas.
- **Parámetro clave:** `contamination` indica la proporción esperada de outliers.

```python
from sklearn.ensemble import IsolationForest

iso = IsolationForest(contamination=0.05, random_state=42)
df["es_anomalia"] = iso.fit_predict(X_scaled) == -1
```

## 3. Local Outlier Factor (LOF)

LOF evalúa si un punto está en una zona "vacía" comparado con sus vecinos.

- **Mecánica:** Compara la densidad local de un punto con la de sus vecinos.
- **Lógica:** Puntos en áreas dispersas son considerados anómalos.

```python
from sklearn.neighbors import LocalOutlierFactor

lof = LocalOutlierFactor(n_neighbors=20, contamination=0.05)
df["es_anomalia_lof"] = lof.fit_predict(X_scaled) == -1
```

## El factor humano en las anomalías

Identificar la anomalía matemáticamente es solo el primer paso.  
El valor real está en la interpretación que el coordinador haga de ese caso.  
Mira cómo el código expone perfiles atípicos en el caso escolar:

```python
iso = IsolationForest(contamination=0.05, random_state=42)
anomalias_mask = iso.fit_predict(X_scaled) == -1

print(df[anomalias_mask][["horas_estudio", "asistencia", "cal_final"]])
```
*Resultados típicos y su posible interpretación:*
- **Estudio: 0.5 | Asistencia: 1.00 | Calificación: 9.8** (Genio que no necesita estudiar)
- **Estudio: 8.0 | Asistencia: 0.40 | Calificación: 7.5** (Falta a clase pero logra aprobar)
- **Estudio: 5.0 | Asistencia: 0.95 | Calificación: 4.0** (Se esfuerza pero no asimila)

## ¿Qué hacer tras la detección?

Una anomalía siempre exige una decisión operativa.
- **Errores de captura:** Corrige el dato (ej. nota de 99 en lugar de 9) y reentrena.
- **Casos reales atípicos:** Sepáralos del dataset principal pero estúdialos (ej. el genio).
- **Crisis detectada:** Activa protocolos de intervención humana inmediata.

## Anti-patrones comunes

- **Eliminar todas las anomalías:** Pierdes información valiosa y sesgas tu modelo.
- **Contaminación arbitraria:** Poner `0.05` sin validar con el experto del dominio.
- **Ignorar el contexto:** Ver el dato anómalo y no investigar qué persona hay detrás.
