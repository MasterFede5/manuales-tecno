# 1.6 Lectura y escritura de archivos (CSV, JSON)

Aquí cumples el Episodio 1: tu programa **lee el CSV de coordinación** y muestra los datos.
Esta lección te enseña los dos formatos más comunes en datos: CSV y JSON.

::concepto{titulo="Archivos CSV y JSON"}
**CSV** (*Comma-Separated Values*) es un formato de texto plano:
- Cada línea es una fila.
- Los valores están separados por comas.
- Es el rey de los datos tabulares.

```
id,nombre,calificacion
124,Andrea,8.5
125,Carlos,7.0
```

**JSON** (*JavaScript Object Notation*) representa estructuras anidadas:
- Se parece a un `dict` de Python.
- Ideal para datos complejos.

```json
{
  "alumnos": [
    {"id": 124, "nombre": "Andrea", "calificacion": 8.5},
    {"id": 125, "nombre": "Carlos", "calificacion": 7.0}
  ]
}
```

Python usa los módulos estándar `csv` y `json` para procesarlos.
::/concepto

::interioriza{titulo="¿Por qué te importa? (Analogía del Archivero)"}
Tus datos NO viven en variables Python, igual que tus documentos no flotan en el aire.
Viven en archivos (el archivero físico).
Excel, bases de datos o sensores exportan a CSV/JSON.
Saber leerlos es tener las llaves del archivero en el mundo real.
::/interioriza

## Leer CSV con módulo `csv`

```python
import csv

# Asume archivo "calificaciones.csv" con header
with open("calificaciones.csv", "r", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for fila in reader:
        print(fila)
```

**Notas clave:**
- `with open()` abre y **cierra automáticamente** el archivo.
- `encoding="utf-8"` evita que los acentos se rompan.
- `DictReader` lee cada fila como diccionario (la primera línea da las claves).
- Los valores leídos son **siempre strings**; debes convertirlos.

## Procesando el CSV

```python
import csv

calificaciones = []
with open("calificaciones.csv", "r", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for fila in reader:
        cal = float(fila["calificacion"])
        calificaciones.append(cal)

promedio = sum(calificaciones) / len(calificaciones)
print(f"Promedio general: {promedio:.2f}")
```

## Escribir en CSV

```python
import csv

datos = [{"id": 124, "promedio": 8.5}, {"id": 125, "promedio": 7.0}]

with open("salida.csv", "w", encoding="utf-8", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=["id", "promedio"])
    writer.writeheader()
    writer.writerows(datos)
```

- `newline=""` evita líneas en blanco adicionales al escribir (común en Windows).

## Leer y escribir JSON

```python
import json

# Leer
with open("alumnos.json", "r", encoding="utf-8") as f:
    datos = json.load(f)

# Escribir
with open("salida.json", "w", encoding="utf-8") as f:
    json.dump(datos, f, indent=2, ensure_ascii=False)
```

- `indent=2` formatea el JSON para que sea legible.
- `ensure_ascii=False` mantiene los acentos intactos.

## Caso aplicado: CSV escolar (Episodio 1)

```python
import csv

estudiantes = []
with open("dataset_escolar.csv", "r", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for fila in reader:
        estudiantes.append({
            "id": int(fila["id"]),
            "cal_final": float(fila["cal_final"])
        })

aprobados = [e for e in estudiantes if e["cal_final"] >= 6]
print(f"Aprobados: {len(aprobados)} alumnos.")
```

- ¡Episodio 1 superado! Entender el flujo manual es clave.
- Más adelante usarás Pandas para hacer esto en 3 líneas.

## Anti-patrones (¡Evítalos!)

- **Olvidar `with`:** Dejar archivos abiertos puede corromperlos.
- **Ignorar `encoding`:** Genera textos rotos (ej. "NiÃ±o"). Usa UTF-8.
- **Rutas absolutas:** `"C:/User/.../archivo.csv"` falla en otras PCs.
- **Falta de validación:** Abrir archivos sin verificar `os.path.exists()`.
- **Todo en RAM:** Leer archivos gigantes de golpe colapsa la memoria.
