# 1.7 Errores y excepciones básicas

Imagina cargar 480 filas de datos.
Si la fila 327 tiene un error, un programa sin manejo de errores muere ahí.
Con manejo de errores, procesa las 479 buenas y reporta la mala.

::concepto{titulo="Excepciones y manejo de errores"}
- **Excepción:** Evento de error en tiempo de ejecución.
- Interrumpe el flujo normal del programa si no se maneja.
- Ejemplos en Python: `FileNotFoundError`, `ValueError`, `KeyError`.

El bloque `try/except` captura excepciones:

```python
try:
    # Código que puede fallar
    cal = float(input_usuario)
except ValueError:
    # Qué hacer si falla
    print("Valor inválido")
```

- Sin este patrón, el programa es frágil.
- Con este patrón, se vuelve robusto.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Piensa en un cajero automático. 
Si no lee tu tarjeta, no se incendia ni se apaga de golpe. 
Te avisa: "Tarjeta no reconocida".
Tu código debe hacer lo mismo con los datos.
::/interioriza

## Las 6 excepciones más comunes

| Excepción | Cuándo ocurre | Ejemplo |
|---|---|---|
| `FileNotFoundError` | Archivo no existe | `open("xx.csv")` |
| `ValueError` | Tipo correcto, valor inválido | `float("ABC")` |
| `KeyError` | Clave de dict no existe | `d["clave_inexistente"]` |
| `TypeError` | Operación con tipo incorrecto | `5 + "hola"` |
| `ZeroDivisionError` | División entre 0 | `5 / 0` |
| `IndexError` | Índice fuera de rango | `lista[100]` con 5 items |

## Anatomía de try/except

```python
try:
    cal = float(input("Calificación: "))
except ValueError as e:
    print(f"Error: {e}")
else:
    print("Calificación válida:", cal)
finally:
    print("Programa termina")
```

**Estructura de bloques:**
- `try`: Código bajo vigilancia.
- `except`: Acción en caso de emergencia.
- `else`: (Opcional) Ejecuta si todo salió bien.
- `finally`: (Opcional) Ejecuta siempre, ideal para cerrar archivos.

## Aplicado al caso escolar

```python
import csv

def cargar_calificaciones(path):
    estudiantes, errores = [], []
    try:
        with open(path, "r", encoding="utf-8") as f:
            reader = csv.DictReader(f)
            for i, fila in enumerate(reader, start=2):
                try:
                    estudiantes.append({
                        "id": int(fila["id"]),
                        "calificacion": float(fila["calificacion"])
                    })
                except (ValueError, KeyError) as e:
                    errores.append((i, str(e), fila))
    except FileNotFoundError:
        print("Archivo no encontrado.")
    return estudiantes, errores
```

- Este programa carga lo bueno y reporta lo malo.
- Nunca se detiene por un error de formato.

## Reglas de oro

1. **Sé específico:** No captures `Exception` genérico sin razón.
2. **No silencies:** Evita `except: pass`, oculta los bugs.
3. **Reporta siempre:** Un error sin reportar es un problema futuro.
4. **Limpia siempre:** Usa `finally` para cerrar archivos y conexiones.
5. **Crea tus reglas:** Usa `raise` para levantar tus propios errores.

## Anti-patrones a evitar

- **Bloques `try` gigantes:** Difícil saber qué línea falló exactamente.
- **Try/except fantasma:** Si no haces nada con el error, mejor no lo captures.
- **Uso para control de flujo:** Las excepciones no reemplazan los `if`.
