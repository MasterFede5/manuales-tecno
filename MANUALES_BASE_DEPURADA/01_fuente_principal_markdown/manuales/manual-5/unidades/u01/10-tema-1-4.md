# 1.4 Funciones y argumentos

Si necesitas clasificar 480 calificaciones, no escribes 480 veces el mismo `if`. 
Lo escribes UNA vez como **función** y la llamas 480 veces.
Así se construye software verdaderamente reutilizable y limpio.

::concepto{titulo="Función y argumentos"}
Una **función** es un bloque de código con nombre que:
- Recibe **argumentos** (datos de entrada).
- Ejecuta una lógica específica.
- Opcionalmente devuelve un **resultado** (salida con `return`).

```python
def nombre_funcion(parametro1, parametro2):
    # cuerpo indentado
    resultado = ...
    return resultado
```

**Beneficios clave:**
- **Reusabilidad** — escribes una vez, usas N veces.
- **Claridad** — el nombre documenta la intención.
- **Testabilidad** — pruebas piezas pequeñas de forma aislada.
::/concepto

::interioriza{titulo="Baja Curva: La receta de cocina"}
Imagina una receta escolar: "Evaluación Final".
- Tienes el nombre ("Evaluar Alumno").
- Los ingredientes o entradas ("Asistencia", "Examen").
- Los pasos (calcular promedio, verificar faltas).
- El platillo final o salida ("Aprobado" o "Reprobado").
Una función es exactamente lo mismo: empaquetas los pasos para no reinventarlos en cada ciclo escolar.
::/interioriza

## Anatomía básica

Observa cómo empaquetamos la lógica para evaluar una calificación:

```python
def clasificar_calificacion(cal):
    if cal >= 9:
        return "Excelente"
    elif cal >= 8:
        return "Muy bien"
    elif cal >= 6:
        return "Aprobó"
    else:
        return "Reprobó"

# Uso simple
resultado = clasificar_calificacion(7.5)
print(resultado)  # "Aprobó"
```

## Argumentos y Parámetros

Hay distintos tipos y formas de enviar datos a una función:

- **Parámetros:** variables en la definición (`notas`, `pesos`).
- **Argumentos:** los valores reales pasados (`[8.5, 9]`, `[0.3, 0.7]`).
- **Por defecto:** asumen un valor si no se envía (`asistencia=1.0`).
- **Por nombre (keywords):** indicas a qué parámetro va el valor (`materia="Física"`).

```python
def calcular_indice(horas_estudio, asistencia=1.0):
    return horas_estudio * asistencia

# Uso de argumento por defecto
print(calcular_indice(5))         # 5.0
print(calcular_indice(5, 0.8))    # 4.0
```

## Tuplas y Retornos Múltiples

Python permite devolver varios valores simultáneamente.
El lenguaje los agrupa en una tupla y luego los "desempaquetas" en variables.

```python
def estadisticas(notas):
    promedio = sum(notas) / len(notas)
    return promedio, min(notas), max(notas)

prom, mn, mx = estadisticas([8.5, 6.0, 9.5, 5.5])
print("Promedio:", prom, "| Min:", mn, "| Max:", mx)
```

## Anti-patrones comunes

Evita estas prácticas que dificultan el mantenimiento:
- **Función gigante.** Si no cabe en tu pantalla, pártela en sub-funciones.
- **Sin `return`.** Si olvidas el `return`, la función devuelve `None`.
- **Modificar parámetros.** No alteres las listas de entrada directamente.
- **Funciones con efectos secundarios.** Modificar variables globales causa errores difíciles de rastrear. Prefiere **funciones puras** (misma entrada = misma salida).
