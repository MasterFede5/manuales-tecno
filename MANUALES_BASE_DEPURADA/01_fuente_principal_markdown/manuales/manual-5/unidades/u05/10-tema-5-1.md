# 5.1 ¿Qué es ML? Supervisado vs no supervisado

- **ML no es magia.**
- Es estadística aplicada iterativamente con muchos datos.
- Su objetivo central es encontrar patrones predictivos invisibles a simple vista.

::concepto{titulo="Machine Learning (ML)"}
Es construir programas que **aprenden de datos** en lugar de seguir reglas rígidas pre-programadas. 

Existen tres familias principales:
- **Supervisado**: Aprendes con ejemplos y sus respuestas correctas.
  - **Regresión**: Predice un número continuo (ej. `cal_final`).
  - **Clasificación**: Predice una categoría (ej. `aprobado` / `reprobado`).
- **No supervisado**: Analizas datos sin respuestas correctas buscando estructuras ocultas (ej. clusters).
- **Refuerzo**: Un agente aprende a tomar decisiones interactuando con un entorno (no cubierto aquí).
::/concepto

::interioriza{titulo="Analógica de la Baja Curva: Aprendiendo a reconocer"}
Imagina enseñarle a un niño a reconocer un gato.
- **Programación tradicional:** Le das una lista de reglas (4 patas, bigotes, cola, maúlla).
- **Machine Learning:** Le muestras 100 fotos de gatos y 100 de perros, y su cerebro ajusta los patrones.

Así mismo, en vez de programar "si asiste > 80% aprueba", le damos al modelo miles de historiales de alumnos.
El modelo encuentra por sí solo el patrón que conecta asistencia y calificaciones.
::/interioriza

## Supervisado: el patrón canónico

- Recolectas **datos históricos** con características (features) y resultados (labels).
- **Entrenas** el modelo para que deduzca la relación matemática.
- **Predices** el resultado para datos nuevos y nunca antes vistos.

## Tipos de tareas

| Tarea | Ejemplo case escolar | Modelo típico |
|---|---|---|
| Regresión | Predecir cal_final exacta | Linear, Random Forest |
| Clasificación binaria | ¿Aprobará? (Sí/No) | Logistic, Decision Tree |
| Clasificación multi-clase | ¿Excelente/Bueno/Suficiente/Reprobado? | Decision Tree, MLP |
| Clustering | Agrupar perfiles de estudio similares | K-Means (U6) |
| Detección anomalías | Detectar comportamientos atípicos | Isolation Forest |

## Vocabulario fundamental

- **Feature** (`X`): Variable de entrada que ayuda a predecir (ej. horas, asistencia).
- **Label / Target** (`y`): El valor objetivo que quieres predecir (ej. cal_final).
- **Sample / Instancia**: Un caso individual o fila (ej. un alumno).
- **Dataset**: La tabla completa de datos.
- **Train**: El proceso de enseñar al modelo usando el dataset.
- **Predict**: Usar el modelo entrenado para generar nuevas estimaciones.

## Anti-patrones

- **Llamar "IA" a reglas duras**: Si es un simple `if/else`, no es Machine Learning.
- **Usar ML innecesariamente**: Si una regla matemática simple resuelve tu problema al 100%, úsala.
- **Ignorar el contexto**: Un modelo entrenado con alumnos de preparatoria fallará prediciendo alumnos de posgrado.
