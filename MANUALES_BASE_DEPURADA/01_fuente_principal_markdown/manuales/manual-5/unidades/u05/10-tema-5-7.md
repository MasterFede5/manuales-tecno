# 5.7 Árboles de decisión

- Un árbol funciona como una entrevista de respuestas "sí o no".
- Va dividiendo a los alumnos hasta tomar una decisión final (aprobar o reprobar).
- Es el algoritmo que más se parece a la lógica humana.

::concepto{titulo="Árbol de decisión"}
Un **árbol de decisión** realiza una serie de **preguntas binarias** sobre las *features*.
- En cada nivel busca la pregunta que mejor separe a las clases.
- Al final, llega a una conclusión (clasificación o regresión).

```python
from sklearn.tree import DecisionTreeClassifier
from sklearn.tree import plot_tree

clf = DecisionTreeClassifier(max_depth=4, random_state=42)
clf.fit(X_train, y_train)

# Visualización visual del árbol
import matplotlib.pyplot as plt
plt.figure(figsize=(20, 10))
plot_tree(clf, feature_names=X.columns, class_names=["Reprobado", "Aprobado"], filled=True)
plt.show()
```

- Aplica para clasificación (`DecisionTreeClassifier`) y regresión (`DecisionTreeRegressor`).
::/concepto

::interioriza{titulo="¿Por qué te importa? (El juego de 'Adivina Quién')"}
Imagina el juego "Adivina Quién". Para ganar rápido, no preguntas "¿Tiene un lunar en la nariz?", sino "¿Tiene pelo rubio?", porque divide mejor a los personajes.
El algoritmo hace lo mismo. Un árbol pequeño es **100 % interpretable**.
Puedes justificar a los padres exactamente por qué el modelo marcó a su hijo en riesgo.
::/interioriza

## Cómo funciona el algoritmo

El proceso interno sigue estos pasos sencillos:
- **Paso 1:** Evalúa los datos buscando la mejor división.
- **Paso 2:** Hace una pregunta binaria clave (ej. "¿asistencia < 0.7?").
- **Paso 3:** Divide el grupo y repite el proceso en cada rama.
- **Paso 4:** Se detiene si llega a `max_depth` o todos son de una clase.

## Aplicado al case escolar

```python
from sklearn.tree import DecisionTreeRegressor

X = df[["horas_estudio", "asistencia", "cal_anterior"]]
y = df["cal_final"]

reg = DecisionTreeRegressor(max_depth=4, random_state=42)
reg.fit(X_train, y_train)

# Mostrar la importancia de cada feature
for col, imp in zip(X.columns, reg.feature_importances_):
    print(f"{col}: {imp:.3f}")
```

Output típico:
```
horas_estudio: 0.18
asistencia:    0.32
cal_anterior:  0.50
```

- En este ejemplo, `cal_anterior` define el 50 % de las decisiones.

## Hiperparámetros clave

Controlar el crecimiento del árbol es vital:

| Parámetro | Función principal |
|---|---|
| `max_depth` | Límite de profundidad (sin límite = sobreajuste seguro). |
| `min_samples_split` | Mínimo de alumnos para permitir otra división. |
| `min_samples_leaf` | Mínimo de alumnos en el resultado final (hoja). |
| `max_features` | Límite de features a evaluar en cada pregunta. |

- Un `max_depth` entre 3 y 5 ofrece el mejor balance de interpretabilidad.

## Traducción a reglas de negocio

- Un gran poder del árbol es que se convierte en código ordinario:

```
SI cal_anterior >= 6.5:
    SI asistencia >= 0.85:
        cal_final ≈ 8.5
    ELSE:
        cal_final ≈ 7.0
ELSE:
    SI horas_estudio >= 4:
        cal_final ≈ 6.5
    ELSE:
        cal_final ≈ 5.0
```

- Puedes exportar estas reglas usando `export_text()`.

## Mención especial: Random Forest

- Un árbol solitario sufre de sobreajuste y poca generalización.
- **Random Forest** entrena cientos de árboles diferentes y promedia el resultado.

```python
from sklearn.ensemble import RandomForestRegressor
rf = RandomForestRegressor(n_estimators=100, max_depth=10, random_state=42)
```

- Gana mucha precisión pero pierde interpretabilidad visual directa.

## Anti-patrones comunes

- **Falta de poda**: Dejar `max_depth` en infinito hará que el árbol memorice.
- **Ocultar el modelo**: No usar `plot_tree` desperdicia su principal ventaja.
- **Cambiar features**: Si agregas nuevas columnas, debes re-entrenar de cero.
- **Confusión causal**: La importancia de feature dice qué usa el árbol, no si es la causa.
