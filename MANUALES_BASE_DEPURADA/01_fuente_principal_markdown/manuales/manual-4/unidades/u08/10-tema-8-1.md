# 8.1 Model Context Protocol (qué resuelve)

- Antes de Nov 2024, conectar un LLM a 4 servicios exigía **4 integraciones distintas**.
- **MCP** convirtió este caos en un estándar universal.
- Descubre por qué MCP cambió las reglas del juego.

::concepto{titulo="Model Context Protocol (MCP)"}
- **Estándar abierto** (Anthropic, Nov 2024).
- Define la comunicación entre **clientes de IA** y sistemas.

**Arquitectura:**
- **Cliente MCP:** La app del LLM (Claude, Cursor).
- **Servidor MCP:** Proceso que expone:
  - **Tools:** Funciones ejecutables (`query_db`).
  - **Resources:** Datos lectura (`file://`).
  - **Prompts:** Plantillas de instrucciones.
::/concepto

::interioriza{titulo="El enchufe universal (Analogía)"}
- **El caos:** Antes, cada electrodoméstico usaba un enchufe distinto.
- **La solución:** MCP es el **enchufe estándar**. 
- Construyes un Servidor MCP para tu CRM una vez.
- **Cualquier** LLM moderno se conecta al instante. Magia pura.
::/interioriza

## Problema vs Solución

**El Pasado (Q3 2024): Caos**
- Cada IA tenía su propio mecanismo (plugins, extensions).
- Integrabas tu sistema repitiendo código para Claude, Cursor, etc.
- La fragmentación bloqueaba el progreso.

**El Presente (Q4 2024+): Estandarización**
- Creas **un solo** Servidor MCP para tu sistema.
- Todos los clientes de IA se conectan a él sin esfuerzo.
- Escribes una vez, lo usan todos.

## Adopción y Anatomía

- **Clientes listos:** Claude Desktop, Cursor, Windsurf, Zed, etc.
- **Ecosistema vibrante:** Cientos de servidores open-source.
- **Funcionamiento dinámico:** El LLM explora el servidor, descubre tools/resources y actúa.

::visual{tipo="diagrama-flujo" descripcion="Diagrama comparativo. Izquierda 'Pre-MCP': 5 flechas caóticas a 5 LLMs. Derecha 'Post-MCP': 1 flecha estandarizada a todos. 'Escribes 1 vez, conectas con todos'." paginas=1}

## Las 3 Armas de MCP

| Capacidad | Función | Ejemplos |
|---|---|---|
| **Tools** | Ejecutar acciones (escribir) | `commit_git`, `query_db` |
| **Resources** | Leer datos (estado) | `file://`, `notion://` |
| **Prompts** | Guiar respuestas | "Resumir resource" |

- El LLM es quien **toma la decisión** de cuándo usarlas.

## Transportes y Anti-patrones

**Conexiones:**
- **stdio:** Subproceso local. Ultra-privado y común.
- **HTTP/SSE:** Ideal para servidores en red.
- **WebSocket:** Conexiones en tiempo real.

**Evita estos errores:**
- **Reinventar la rueda:** Busca servidores MCP ya hechos.
- **Cero Auth:** Protege siempre tus puertos.
- **Tools vagos:** Describe tus funciones con precisión láser.
