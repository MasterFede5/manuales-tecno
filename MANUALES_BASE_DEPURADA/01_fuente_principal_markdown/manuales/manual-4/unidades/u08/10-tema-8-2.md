# 8.2 Servidores MCP populares (filesystem, GitHub, Slack, Notion)

Cientos de servidores MCP existen. 
Esta lección cubre los **12 más útiles** que probablemente vas a usar desde el primer día.

::concepto{titulo="Catálogo de servidores MCP"}
Los servidores MCP se distribuyen como paquetes ejecutables (npm, pip, binarios).
Anthropic mantiene un repo oficial con servidores curados.
La comunidad ha publicado cientos más en Hugging Face y Awesome MCP.

Categorías:
- **Filesystem y archivos** (lectura/escritura local).
- **Versionado** (Git, GitHub, GitLab).
- **Comunicación** (Slack, Discord, email).
- **Productividad** (Notion, Airtable, Google Workspace).
- **Bases de datos** (Postgres, SQLite, MongoDB).
- **Web y búsqueda** (fetch, brave-search, browser).
- **Utilidades** (time, calculator, memory).
::/concepto

::interioriza{titulo="El catálogo de herramientas"}
Imagina que construyes un robot de cocina. Antes tenías que inventar cada brazo mecánico.
Hoy, vas a la tienda MCP y compras el brazo de "batir", el de "cortar" y el de "medir".
Tu única tarea es decidir qué brazos conectar para preparar el platillo deseado.
::/interioriza

## Top 12 servidores MCP imprescindibles

- **1. filesystem:** Comando: `npx @modelcontextprotocol/server-filesystem /allowed/path`. Tools: `read_file`, `write_file`. Caso: leer documentos locales.
- **2. github:** Comando: `npx @modelcontextprotocol/server-github`. Tools: `search_repositories`. Caso: abrir issues institucionales.
- **3. slack:** Comando: `npx @modelcontextprotocol/server-slack`. Tools: `slack_post_message`. Caso: notificar canal #admisiones.
- **4. notion:** Comando: `mcp-notion-server`. Tools: `create_page`, `query_database`. Caso: crear tickets de soporte.
- **5. postgres:** Comando: `npx @modelcontextprotocol/server-postgres`. Tools: `query`. Caso: consultar calificaciones.
- **6. sqlite:** Comando: `npx @modelcontextprotocol/server-sqlite`. Tools: `query`. Caso: seguimiento ligero.
- **7. brave-search:** Comando: `npx @modelcontextprotocol/server-brave-search`. Tools: `brave_web_search`. Caso: búsqueda web.
- **8. fetch:** Comando: `npx @modelcontextprotocol/server-fetch`. Tools: `fetch`. Caso: leer páginas web públicas.
- **9. memory:** Comando: `npx @modelcontextprotocol/server-memory`. Tools: `create_entities`. Caso: memoria persistente.
- **10. time:** Comando: `npx @modelcontextprotocol/server-time`. Tools: `get_current_time`. Caso: calcular fechas.
- **11. google-drive:** Comando: múltiples. Tools: `gdrive_search`. Caso: buscar en el Drive académico.
- **12. gmail:** Comando: múltiples. Tools: `gmail_send`. Caso: redactar correos.

## Configurar un servidor en Claude Desktop

Edita el archivo de configuración `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/me/AlbatrosDocs"]
    }
  }
}
```

Reinicias Claude Desktop.
Los servidores aparecen disponibles.
Todo sin escribir una sola línea de código.

## Servidores custom

Si tu sistema no tiene servidor MCP, puedes construirlo:
- **TypeScript SDK**: `@modelcontextprotocol/sdk` (oficial).
- **Python SDK**: `mcp` (PyPI).
- **Plantillas**: repo oficial con templates listos.

Construir uno toma 1-2 horas con conocimientos básicos.
Luego, todos los clientes MCP lo pueden consumir.

## Estrategia de adopción

- **Mes 1**: filesystem + memory + time.
- **Mes 2**: GitHub + slack.
- **Mes 3**: notion + postgres / sqlite.
- **Mes 6**: servidor custom para tu sistema interno.

## Anti-patrones

- **Conectar todo:** Iniciar con 12 servidores es un error. Usa 1 o 2.
- **Tokens overpowered:** Limita el acceso; cuidado con el prompt injection.
- **No actualizar:** Mantén versiones para evitar bugs de seguridad.
- **No probar permisos:** Valida que escriban solo donde deben.
