# 8.3 Claude Desktop con MCP

Claude Desktop es el cliente MCP **referencia**.
Anthropic creó el estándar y su app lo integra nativamente.
Aprenderás a configurarlo y operarlo en producción.

::concepto{titulo="Claude Desktop"}
Es la aplicación oficial para macOS y Windows.
Ofrece una interfaz similar a la versión web, pero con **integración nativa MCP**.
Cada servidor configurado aparece como una herramienta disponible.

Características:
- **Nativo**: Soporte MCP desde noviembre 2024.
- **Multi-server**: Permite N servidores simultáneos.
- **Indicadores**: UI muestra servidores activos.
- **Aprobaciones**: Pide permiso para acciones críticas.
- **Logs**: Registra tool calls auditables.
- **Aislamiento**: Servidores corren en sandbox.
::/concepto

::interioriza{titulo="¿Por qué te importa? La llave maestra"}
Usar la web es como tener un asistente brillante pero encerrado en otra oficina.
Claude Desktop con MCP es darle a ese asistente un pase VIP a tus archivos y sistemas.
Puede leer, buscar y trabajar directamente en tu entorno institucional.
::/interioriza

## Instalación y configuración

### macOS / Windows

- **Descarga**: Obtén la app en `claude.ai/download`.
- **Instala**: Proceso estándar del sistema operativo.
- **Login**: Usa tu cuenta (Pro recomendado).
- **Configura**: Edita el archivo `claude_desktop_config.json`.
  - macOS: `~/Library/Application Support/Claude/`
  - Windows: `%APPDATA%\Claude\`

### Configuración inicial

```json
{
  "mcpServers": {
    "filesystem-albatros": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/me/AlbatrosDocs"]
    },
    "memory": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"]
    }
  }
}
```

Al reiniciar, verás el icono de "2 MCP servers connected".

## Tu primer prompt

Conversación de ejemplo:

- **Tú**: ¿Qué archivos hay de "becas"?
- **Claude**: (invoca `search_files`) "Encontré reglamento_2026.pdf".
- **Tú**: Resume el reglamento.
- **Claude**: (invoca `read_file`) "El resumen es...".
- **Tú**: Los hermanos tienen 15% extra.
- **Claude**: (invoca `create_entity`) "Anotado en memoria".

## Configuración institucional

```json
{
  "mcpServers": {
    "github-asistente": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {"GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxxxx"}
    },
    "postgres-readonly": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres", "postgresql://user:pass@db/asistente"]
    }
  }
}
```

Así tu Claude Desktop accede a repositorios y bases de datos.

## Approval flow y Seguridad

Claude pregunta antes de acciones de escritura:

- **Lectura**: Se auto-aprueba (e.g., leer un archivo).
- **Escritura**: Requiere tu confirmación explícita.
- **Destructivas**: Siempre protegidas por el flujo de aprobación visual.

## Distribución al equipo

Para implementarlo con tu staff:

- **Script de setup**: Automatiza instalación y config JSON.
- **Credenciales**: Usa gestores de secretos, no tokens expuestos.
- **Capacitación**: Enseña cómo pedir acciones concretas por chat.

## Anti-patrones

- **Tokens en config**: Peligroso si se comparte. Usa variables de entorno.
- **Path raíz (`/`)**: Da acceso a todo el disco. Restringe carpetas.
- **Exceso de servidores**: Más de 10 confunden al modelo.
- **Falta de logs**: Dificulta auditorías y depuración.
