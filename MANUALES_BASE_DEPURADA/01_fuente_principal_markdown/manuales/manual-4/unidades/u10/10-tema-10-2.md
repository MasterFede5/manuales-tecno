# 10.2 Costos por tokens y optimización

- Tu Asistente puede costar $300/mes en API.
- Con técnicas correctas, lo reduces a $80/mes sin perder calidad.

::concepto{titulo="Costos por tokens y optimización"}
Los **tokens** son las unidades de cobro por inferencia.
Cada modelo tarifa por 1M de tokens (input/output).

Optimizar implica combinar:
- **Modelo ideal:** Haiku (clasificación), Opus (razonamiento).
- **Prompt caching:** Reusar contexto extenso sin doble cobro.
- **Batch API:** Procesamiento offline al 50 % de costo.
- **Reducción de tokens:** Prompts breves y `max_tokens` fijo.
- **RAG eficiente:** Fragmentos justos, no bases de datos enteras.
- **Ejecución local:** Ollama para tareas gratuitas.
::/concepto

::interioriza{titulo="Baja Curva: La economía de los envíos"}
Piensa en los tokens como el peso de un paquete postal:
- **Opus:** Envío urgente en avión (caro, para paquetes delicados o complejos).
- **Haiku/Flash:** Envío terrestre estándar (barato, para cotidianos).
- **Prompt caching:** Almacén local en lugar de pedir desde China cada vez.
La disciplina de optimización no es solo abaratar, es usar el transporte correcto.
::/interioriza

## Las 6 técnicas de optimización

### 1. Modelo correcto por tarea
- **Clasificación/extracción:** Usa Haiku / Flash / mini.
- **Redacción/generación:** Usa Sonnet / GPT-5 / Pro.
- **Razonamiento crítico:** Reserva para Opus / o3.
- Si usas Opus para todo, gastarás 18× más de lo necesario.

### 2. Prompt caching
- Si envías un system prompt de 10k tokens, lo pagas cada vez.
- **Con caching:** El primer envío cuesta 1.25×, los siguientes 0.10×.
- Ahorras ~85 % en partes fijas (Anthropic, OpenAI, Google).

### 3. Batch API
- Procesamientos masivos no urgentes (ej. clasificar 1000 correos).
- Anthropic y OpenAI ofrecen 50 % de descuento por latencia 24h.
- Ideal para reportes asíncronos o procesos nocturnos.

### 4. Reducción de tokens
- **Prompts concisos:** Edita sin piedad; cada palabra cuesta.
- **Limita `max_tokens`:** Si esperas 200, no reserves 4000.
- **Few-shot mínimo:** 3 ejemplos claros superan a 10 redundantes.

### 5. RAG eficiente
- Usa **Re-ranker** para reducir de top-50 a top-5 antes del LLM.
- Fragmenta en **chunks** de 400-600 tokens (no 2000).
- Cachea las consultas (queries) más frecuentes.

### 6. Híbrido cloud-local
- **Tareas masivas/rutinarias:** Modelos locales gratuitos (Ollama).
- **Tareas críticas:** Modelos cloud (si no hay datos sensibles).

## Caso real: optimización del Asistente Institucional

**Antes de optimizar:**
- 80 inscripciones/mes × Sonnet ($3+$15) × 5k tokens.
- Total: ~$80/mes solo en clasificación.

**Después:**
- Cambio a Haiku ($0.80+$4) × 5k tokens.
- Caching de system prompt (90 % de reducción).
- Total: ~$15/mes.

- **Ahorro:** $65/mes = $780/año, sin pérdida de calidad.

## El cálculo del costo total (TCO)

- **Costos API y suscripciones** (ChatGPT Team, Claude Pro).
- **Infraestructura:** Hardware local amortizado y electricidad.
- **Operación:** Mantenimiento TI, plataformas no-code (n8n, Dify).
- **Servicios extra:** Vector DBs cloud, observabilidad (LangFuse).

**Comparado con beneficios (ROI):**
- ROI = (Tiempo ahorrado) + (Casos resueltos) + (Reducción errores) - TCO.
- Documenta estas métricas en el plan a 12 meses.

## Anti-patrones de optimización
- **No medir:** Sin tracking, no sabes qué optimizar.
- **Ignorar la calidad:** Ahorrar perdiendo precisión no es negocio.
- **Caching de datos sensibles:** El caché retiene PII de forma indeseada.
- **Batch para urgencias:** Latencia de 24h arruina un chat en vivo.
- **Modelo barato para lógica:** Sale más caro corregir sus alucinaciones.
