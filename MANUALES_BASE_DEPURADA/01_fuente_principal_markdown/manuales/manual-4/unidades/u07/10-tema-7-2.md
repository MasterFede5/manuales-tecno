# 7.2 Ollama: descarga y ejecución

Ollama convierte "correr un LLM local" de un proyecto complejo a un comando de 60 segundos.
- Es la herramienta estándar para probar IA local.
- Simplifica el proceso para que cualquier perfil técnico lo utilice.

::concepto{titulo="Ollama"}
**Ollama** es un runtime ligero que descarga, ejecuta y sirve modelos de lenguaje abiertos en tu máquina.
Características principales:
- **CLI simple** (`ollama pull`, `ollama run`, `ollama list`).
- **API REST local** compatible con OpenAI API (`localhost:11434`).
- **Librería curada** (Llama, Mistral, Qwen, DeepSeek, etc.).
- **Soporte multi-OS** (macOS, Linux, Windows).
- **GPU automática** si está disponible (CUDA/Metal/ROCm).
Es el líder indiscutible en 2025 para uso local.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Antes, correr un LLM era como **armar un auto desde cero** (configurar Python, PyTorch, CUDA, inferencia = 3-5 horas).
Hoy, con Ollama, es como **comprar un auto automático y encenderlo**:
- Escribes `ollama run llama3.1`.
- En 5 minutos tienes IA funcionando.
Tu equipo de TI, sin ser doctores en Machine Learning, puede operarlo de inmediato.
::/interioriza

## Instalación (Windows / macOS / Linux)

- **Linux / macOS:** 
  `curl -fsSL https://ollama.ai/install.sh | sh`
- **Windows:** 
  Descargar desde [ollama.com/download](https://ollama.com/download).
- **Docker (servidor):**
  ```bash
  docker run -d --gpus all -v ollama:/root/.ollama -p 11434:11434 --name ollama ollama/ollama
  ```

El servicio correrá siempre en `localhost:11434`.

## Comandos esenciales

Operaciones clave en la terminal:
- `ollama list`: Muestra modelos descargados.
- `ollama pull llama3.1`: Descarga modelo (~4.7 GB).
- `ollama run llama3.1`: Abre chat interactivo.
- `ollama run llama3.1 "Hola"`: Ejecuta prompt directo.
- `cat doc.md | ollama run llama3.1 "Resume"`: Procesa archivos.
- `ollama rm llama3.1`: Borra el modelo para liberar espacio.

## API REST (compatible OpenAI)

Ollama expone una API idéntica a OpenAI en `localhost:11434`:
- Usa el SDK estándar de Python (`openai`).
- Cambia la `base_url` a `http://localhost:11434/v1`.
- Usa cualquier string en `api_key`.

**Ventaja clave:** Tu código existente para OpenAI funciona localmente con **cero migración**.

## Modelfile: customización

Crea variantes personalizadas usando un `Modelfile` (como un Dockerfile):
- Define el modelo base (`FROM llama3.1:8b`).
- Inyecta un prompt maestro (`SYSTEM "Eres el asistente..."`).
- Ajusta parámetros técnicos (`PARAMETER temperature 0.3`).

Para construirlo y usarlo:
```bash
ollama create asistente -f Modelfile.asistente
ollama run asistente
```
El nuevo modelo ya incluye sus reglas de fábrica.

## Cómo se compara con cloud (en práctica)

Velocidad y calidad al procesar 5000 palabras:
- **Llama 3.1 8B (CPU):** 25s, calidad 7/10.
- **Qwen 2.5 72B (GPU):** 9s, calidad 9/10 (gran español).
- **Claude Sonnet 4.6 (Cloud):** 3s, calidad 9.5/10.

**Conclusión:** Los modelos locales grandes (70B+) bastan para el 80% de tareas institucionales. Cloud sigue dominando en razonamiento profundo.

## Anti-patrones

- **Modelos gigantes en CPU:** Un 70B sin GPU tardará minutos en responder.
- **Ignorar la RAM:** Necesitas al menos 1.2× el peso del modelo en RAM libre.
- **Síndrome de Diógenes de IA:** No borrar modelos viejos llenará tu disco duro.
- **Implementar a ciegas:** Siempre valida con casos de uso reales antes de producción.
- **Asumir vigencia:** La IA abierta avanza rápido; actualiza modelos cada 3-6 meses.
