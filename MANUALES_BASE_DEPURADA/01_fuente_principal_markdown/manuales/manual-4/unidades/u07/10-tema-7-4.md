# 7.4 LM Studio y Jan

Ollama con Open WebUI es excelente para equipos e instituciones.
Para usuarios individuales, destacan **LM Studio** y **Jan**.
Ambas son aplicaciones de escritorio nativas que simplifican la IA local.

::concepto{titulo="LM Studio y Jan"}
**LM Studio** y **Jan** son aplicaciones de escritorio todo-en-uno:
- Combinan motor, interfaz gráfica y descarga de modelos en un clic.
- No requieren Docker ni configuraciones de servidor.
- Están diseñadas para uso individual (no multi-usuario).
- LM Studio es "freemium", mientras que Jan es 100% open source.
::/concepto

::interioriza{titulo="Analogía: El autobús vs. Tu coche propio"}
Ollama con Open WebUI es como un autobús escolar: requiere un chófer (Docker) y transporta a 50 personas a la vez (multi-user).
LM Studio o Jan son como tu coche automático: giras la llave y conduces tú solo.
Son ideales para tu coordinadora académica, que solo quiere usar IA sin aprender "mecánica" de servidores.
::/interioriza

## LM Studio

Es la opción más pulida para usuarios de Mac (Apple Silicon) y Windows.
**Web:** lmstudio.ai · **Plataformas:** macOS, Windows, Linux.

- **Ventajas:** Instalación en 1 clic y explorador de modelos integrado.
- **Desarrollo:** Servidor local (puerto 1234) compatible con API de OpenAI.
- **Control:** Interfaz visual para ajustar parámetros como temperatura.
- **Desventajas:** Código cerrado y carece de base de conocimientos nativa.

## Jan

Es la alternativa transparente y enfocada en la privacidad absoluta.
**Web:** jan.ai · **Plataformas:** macOS, Windows, Linux.

- **Ventajas:** Código abierto real (licencia AGPL-3.0) y diseño minimalista.
- **Privacidad:** Cero telemetría por diseño, ideal para datos sensibles.
- **API:** También ofrece API local compatible con OpenAI.
- **Desventajas:** Menos modelos pre-configurados y sin multi-usuario.

## Puntos en Común y Anti-patrones

Ambas herramientas comparten características esenciales:
- Procesamiento 100% local y compatibilidad nativa con la API de OpenAI.
- Soporte para archivos GGUF y modelos cuantizados.

**Anti-patrones a evitar:**
- **Coleccionismo:** Usar Ollama, LM Studio y Jan a la vez causa confusión.
- **Ignorar el hardware:** Dejar la app corriendo al 100% sobrecalienta el equipo.
- **Exposición de red:** Abrir el puerto local a la red sin autenticación.

::pausa{titulo="💭 Pausa para pensar"}
1. Si un investigador maneja datos de pacientes muy confidenciales, ¿qué instalador le enviarías y por qué?

3. ¿Qué riesgo asumes si activas la "API local" de Jan en una red wifi pública sin seguridad?
