# 1.1 Estructura XML y delimitadores

¿Pides un comunicado y el modelo responde en tonos distintos cada vez?
Esto ocurre porque tu prompt carece de un contrato técnico.
Aprenderemos a estructurarlo para evitar resultados impredecibles.

::concepto{titulo="Estructura XML aplicada a prompts"}
Consiste en envolver componentes con etiquetas semánticas:
`<role>`, `<context>`, `<task>`, `<constraints>`, `<examples>`, `<output_format>`.

Modelos como Claude y Gemini 2 reconocen esta estructura:
- La tratan como **secciones independientes** durante su razonamiento.

**Ventajas clave:**
- **Parsing fiable**: Separa datos de instrucciones y rol.
- **Edición quirúrgica**: Modificas una sección sin romper el resto.
- **Versionado claro**: Se gestiona en Git como cualquier código.

XML es la estructura más robusta para prompts largos.
También es esencial cuando hay riesgo de inyección de código.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Imagina pedirle a un recepcionista escolar: "Llama a la madre de Juan castigado por notas hablar con director urgente".
Adivinará la intención el 60 % de las veces.

Pero si usas un formulario con campos claros:
- **[Alumno]**: Juan
- **[Motivo]**: Notas
- **[Acción]**: Cita urgente con director

Aciertas el 100 %. XML es ese formulario estructurado para el modelo.
::/interioriza

## Anatomía de un prompt XML profesional

El esqueleto para el 90 % de los casos del Asistente Institucional:

```xml
<role>
Eres el Asistente Institucional Albatros.
Tono profesional, cálido y respetuoso.
</role>

<context>
Un padre pregunta sobre fechas de exámenes finales.
Fecha actual: 28 de abril.
</context>

<task>
Redacta un comunicado de máx. 200 palabras.
Incluye: fechas, lugar, qué llevar y contacto.
</task>

<constraints>
- No inventes fechas fuera del contexto.
- Usa tratamiento de "usted".
- No uses emojis.
</constraints>
```

## Reglas de oro

- **Etiquetas semánticas:** `<task>` para acciones, `<context>` para datos.
- **Anidamiento:** Útil en `<examples><example_good>...`.
- **Cierre obligatorio:** Evita confundir al parser.
- **Sin atributos:** No uses `<task priority="high">`.
- **CDATA:** Úsalo para datos que contengan `<` o `>`.

## Alternativas al XML

| Caso | Delimitador | Razón |
|---|---|---|
| Cortos (<50 palabras) | Markdown (## Tarea) | XML pesa demasiado |
| Código fuente | Triple backticks (\```) | Preserva indentación |
| Texto de usuario | `<user_input>` | Anti-inyección |
