# 1.3 CoT avanzado y ReAct

- Sin razonar, el Asistente falla operaciones simples.
- Exígele tiempo para pensar antes de generar respuestas.
- Darle estructura mejora la precisión en lógicas complejas.

::concepto{titulo="Chain-of-Thought (CoT) y ReAct"}
- **CoT:** Exige al modelo escribir sus pasos lógicos primero.
- **ReAct** (*Reasoning + Acting*): Extiende CoT con herramientas externas.
- **Ciclo ReAct:** *Thought* (piensa) → *Action* (usa herramienta) → *Observation* (recibe dato).
- CoT es interno; ReAct conecta el modelo al mundo real.
::/concepto

::interioriza{titulo="El Asistente como Analista Financiero"}
- Un analista no da un presupuesto sin ver su hoja de cálculo.
- *Thought:* Necesito el saldo actual.
- *Action:* Abre el software contable.
- *Observation:* Ve 5,000 MXN y luego responde con certeza.
::/interioriza

## CoT Simple vs Variantes

- El truco base es instruir: **"Piensa paso a paso"**.
- Puedes estructurar su razonamiento con etiquetas `<scratchpad>`.
- Obligar esta estructura facilita depurar y auditar errores.

- **4 Variantes clave:**
  - **Zero-shot CoT:** Agrega la frase mágica "Piensa paso a paso".
  - **Few-shot CoT:** Dale 2-3 ejemplos del análisis deseado.
  - **Self-ask:** El modelo se hace subpreguntas para tareas complejas.
  - **Plan-and-solve:** Divide el problema en un plan antes de actuar.

## ReAct en Acción

- **Thought:** "Necesito saber cupos de 2026."
- **Action:** `query_db("SELECT cupos FROM...")`
- **Observation:** `47`
- El ciclo se repite hasta poder dar una **Final Answer**.
- Limita los ciclos de ReAct (ej. máx 8) para evitar *loops* infinitos.

## Tres Reglas de Oro

- Usa **CoT** por defecto para becas, plazos y cálculos.
- Usa **ReAct** si debes cruzar datos externos o validar reglas.
- Guarda siempre el `<scratchpad>` en los logs para trazabilidad.

::pausa{titulo="💭 Pausa para deducción"}
- Si el Asistente se queda en un ciclo infinito de consultas, ¿qué falló en su prompt?
- ¿Por qué ocultar el `<scratchpad>` al usuario final del sistema?
- ¿Qué ventaja da Few-shot CoT frente a Zero-shot en consultas institucionales?
::/pausa

> **Siguiente paso.** Si CoT falla en una cadena, debes generar varias y combinarlas. Eso es Self-consistency. Subtema 1.4.
