# 1.2 Meta-prompting

¿Quién está mejor calificado para escribir un prompt para Claude que el propio Claude? 
Ese es el chiste —y la herramienta operativa— del meta-prompting.

::concepto{titulo="Meta-prompting"}
El **meta-prompting** consiste en usar un modelo de IA para diseñar, refinar o evaluar otros prompts. 
En lugar de escribir tú mismo el prompt definitivo, le pides al modelo que lo haga.

Existen tres modalidades principales:
- **Prompt-author**: el modelo te entrega un prompt listo para usar.
- **Prompt-critic**: el modelo evalúa un prompt existente y sugiere mejoras.
- **Prompt-rewriter**: el modelo reescribe iterativamente hasta cumplir criterios.

Esto funciona porque el modelo conoce sus propios mecanismos de atención. 
Sabe qué estructura responde mejor y qué orden reduce alucinaciones.
::/concepto

::interioriza{titulo="El chef que escribe su propia receta"}
Imagina un restaurante donde el chef (el LLM) conoce perfectamente sus ingredientes. 
En lugar de adivinar la receta, le dices al chef qué plato quieres.
Él diseña su propia receta perfecta y la cocina. 
Diez minutos de meta-prompting ahorran horas de ensayo y error.
::/interioriza

## El meta-prompt patrón (úsalo tal cual)

```text
Eres un ingeniero de prompts experto. Voy a darte una tarea y necesito
que diseñes el prompt óptimo para resolverla con un LLM.

<objetivo>
[describe en 1-2 frases qué quieres lograr]
</objetivo>

<audiencia_destino_del_output>
[quién va a leer/usar el resultado del prompt]
</audiencia_destino_del_output>

<criterios_de_éxito>
[3-5 criterios verificables: longitud, tono, formato, etc.]
</criterios_de_éxito>

<restricciones>
[qué NO debe hacer el output]
</restricciones>

Tarea:
1. Diseña un prompt en estructura XML con secciones <role>, <context>,
   <task>, <constraints>, <examples> y <output_format>.
2. Justifica cada decisión de diseño en 1 línea.
3. Propón 3 ejemplos de input que probarían bordes del prompt.
```

Copia este meta-prompt y úsalo con los datos de tu tarea en tu LLM favorito. 
Obtendrás un prompt estructurado y su justificación en segundos.

## Bucle de refinamiento iterativo

Un solo round rara vez basta. El bucle profesional sigue estos pasos:

1. **Genera v1** usando el meta-prompt patrón.
2. **Ejecuta v1** contra 5 inputs reales.
3. **Pide crítica:** "¿Dónde falla con estos outputs? ¿Qué sobra?".
4. **Pide reescritura:** "Corrige los problemas graves. Marca con `// CHANGE:`".
5. **Repite** hasta no encontrar mejoras sustantivas.

Suelen bastar 2–4 vueltas para lograr un prompt de calidad de producción.

## Anti-patrones del meta-prompting

Evita estos errores comunes al aplicar meta-prompting:
- **Meta-promptear sin criterios:** Define cómo medir el éxito antes de empezar.
- **Aceptar el primer output:** El valor real está en el refinamiento iterativo.
- **Confundir meta-prompt con prompt final:** El meta-prompt es la fábrica, no el producto.
- **Perder versiones:** Guarda y numera cada iteración (ver subtema 1.8).

## Caso aplicado: dudas de inscripción

Para que un Asistente Institucional responda preguntas frecuentes:

- **Meta-prompt:** objetivo "responder dudas de padres sobre inscripciones".
- **v1 generada:** Claude crea 220 palabras, secciones XML y 2 ejemplos.
- **Pruebas:** ejecutas con 5 preguntas reales de admisiones.
- **Crítica:** notas que v1 no maneja bien los tonos molestos.
- **v2:** añades la restricción `<de_escalation>` con frases empáticas.
- **Versionado:** `prompt-admisiones-v2.0.xml` queda en el repositorio.
