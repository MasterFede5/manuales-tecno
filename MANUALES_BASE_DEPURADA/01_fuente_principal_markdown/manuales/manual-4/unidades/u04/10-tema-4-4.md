# 4.4 Plataformas no-code: Dify, Flowise, Stack AI

Custom GPT, Claude Projects y NotebookLM resuelven el 80 % de los casos. 
¿Pero qué pasa con el 20 % restante? 

Bienvenido al mundo no-code técnico. Aquí podrás:
- Elegir el embedder a tu medida.
- Ajustar el chunking con precisión.
- Conectar con Notion y otras bases.
- Exponer una API propia para tu app.

::concepto{titulo="Plataformas no-code para LLM apps"}
**Dify, Flowise y Stack AI** permiten construir aplicaciones LLM (RAG, agentes).
Todo funciona **arrastrando bloques visuales** en un canvas.
Evitas programar código desde cero, pero mantienes control absoluto.

Con estas herramientas puedes:
- Elegir embedder, vector store y modelo de forma independiente.
- Definir pipelines de chunking y re-ranking.
- Conectar fuentes como Notion, Drive, o APIs.
- Versionar y monitorear desde un dashboard central.

Es el equilibrio perfecto entre Custom GPT (cerrado) y LangChain (código puro).
::/concepto

::interioriza{titulo="El traje a la medida"}
Imagina que Custom GPT es un traje de talla única: te saca del apuro, pero puede quedar holgado. 
Programar en código puro es tejer la tela desde cero: requiere muchísimo tiempo y habilidad.
Las plataformas no-code son como ir a un sastre: eliges las piezas prefabricadas y las ajustas al milímetro.
Tienes un "traje a la medida" para tu RAG sin necesidad de ser un costurero experto.
::/interioriza

## Dify — la opción más madura para producción

**Web:** dify.ai · **Hosting:** cloud (SaaS) o self-hosted.

**Fortalezas:**
- UI extremadamente pulida y amigable.
- Soporta workflows complejos y multi-modelo.
- Self-hosted gratuito para pymes.
- Marketplace con plantillas listas para usar.
- Genera una API REST por cada app creada.

**Cuándo elegir:**
- Buscas llevar tu app a producción.
- Tienes requisitos de privacidad (self-hosting).
- Tienes un técnico para manejar Docker.

**Cuándo NO:**
- Solo requieres un chatbot simple.
- No tienes ninguna capacidad de DevOps.

## Flowise — open-source, ultra-flexible

**Web:** flowiseai.com · **Hosting:** self-hosted (npm install).

**Fortalezas:**
- Open source puro (Apache 2.0).
- Construido sobre LangChain.js (todos sus nodos).
- Gran comunidad activa en GitHub.
- Permite customizar código JavaScript en cada nodo.

**Cuándo elegir:**
- Exiges una solución 100% open source.
- Tu equipo técnico domina Node.js.
- Requieres integraciones muy específicas de LangChain.

**Cuándo NO:**
- Prefieres no mantener servidores propios.
- Buscas una interfaz más limpia (Dify es superior ahí).

## Stack AI — enterprise con énfasis en cumplimiento

**Web:** stack-ai.com · **Hosting:** cloud / self-hosted para Enterprise.

**Fortalezas:**
- Diseñado para compliance, auditoría y roles.
- Integraciones nativas empresariales (Salesforce, Snowflake).
- Builders especializados (agentes, automatización).
- Planes Enterprise incluyen implementación.

**Cuándo elegir:**
- Eres una empresa grande con altas regulaciones.
- Tienes sistemas legacy corporativos.
- Hay presupuesto para soporte profesional.

**Cuándo NO:**
- Eres una startup o pyme.
- Solo estás haciendo pruebas iniciales.

## Patrón típico de pipeline RAG

Los tres exponen un flujo similar mediante bloques visuales:

1. **Input usuario**: Recibe la consulta.
2. **Pre-processing**: Limpia texto o detecta idioma.
3. **Vector store query**: Extrae los top-k chunks.
4. **Re-ranking** (Opcional): Reordena resultados.
5. **Prompt construction**: Arma el contexto.
6. **LLM generation**: Crea la respuesta.
7. **Post-processing**: Formatea y añade citas.

Arrastras nodos, configuras el panel lateral y despliegas con un clic.

## Receta concreta: setup en Dify (45 min)

- **Minuto 1-5**: Instala Dify con Docker o regístrate en SaaS.
- **Minuto 5-10**: Configura API keys (OpenAI, Claude, etc).
- **Minuto 10-20**: Crea Knowledge Base (sube PDFs, ajusta chunking).
- **Minuto 20-30**: Crea la App tipo "Chatbot" y vincula tu Knowledge Base.
- **Minuto 30-35**: Prueba en el debugger integrado.
- **Minuto 35-40**: Publica la app (widget web o API).
- **Minuto 40-45**: Documenta tu proceso.

## Anti-patrones

- **Ignorar Dify por Stack AI**: Este último suele estar sobredimensionado.
- **Self-host sin soporte**: Eventualmente los servidores fallan.
- **Probar todas a la vez**: Pierdes el foco. Elige una y profundiza.
- **Olvidar la evaluación**: Necesitas siempre tu golden set para validar resultados.
