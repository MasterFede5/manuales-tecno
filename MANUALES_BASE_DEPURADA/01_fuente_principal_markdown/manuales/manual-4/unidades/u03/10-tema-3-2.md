# 3.2 Claude Artifacts (creación, iteración, publicación)

Abres claude.ai y pides: "Construye un dashboard para mi Asistente con 4 métricas". 
Treinta segundos después, tienes un panel funcional en pantalla. 
Pides un botón naranja, el modelo edita al instante y lo ves en vivo. Esto es Claude Artifacts.

::concepto{titulo="Claude Artifacts"}
**Claude Artifacts** es el sistema de Anthropic para crear, iterar y publicar piezas autocontenidas.
Se abre en un panel paralelo al chat dentro de `claude.ai`.
Soporta nativamente:

- **React** con Tailwind y librerías (`recharts`, `lucide-react`, `framer-motion`).
- **HTML/CSS/JS** estático, **SVG**, **Mermaid** y **Markdown**.
- **Código** en varios lenguajes (formateado y descargable, aunque no ejecutable).

Se consolidó como la referencia para "vibe coding" y prototipado rápido.
::/concepto

::interioriza{titulo="Baja Curva: La cocina abierta"}
Piensa en Artifacts como un restaurante de cocina abierta. 
Antes, hacías tu pedido y esperabas horas en tu mesa sin ver nada (ciclo tradicional de deploy). 
Ahora, te sientas frente al chef. Si no te gusta la salsa, se lo dices y la cambia al instante. 
Esta fricción cero cambia por completo qué te atreves a "pedir" o probar.
::/interioriza

## Anatomía de la interfaz

```text
┌─────────────────────────┬───────────────────────────────┐
│                         │  ┌─[Code][Preview]─[v3]─[⋯]┐ │
│  CHAT                   │  │                          │ │
│  ─────                  │  │       ARTIFACT           │ │
│  > "haz un dashboard"   │  │      (vista preview)     │ │
│  Claude: aquí está...   │  │                          │ │
│                         │  └──────────────────────────┘ │
│  [campo de texto]       │  [Publish] [Copy] [Download]  │
└─────────────────────────┴───────────────────────────────┘
```

- **Izquierda**: Chat y conversación.
- **Derecha**: Panel del artifact.
- **Vistas**: Pestañas *Code* y *Preview*.
- **Controles**: Navegación de versiones (v1, v2...) y acciones (Publish, Copy).

## Patrón de creación: el "primer prompt"

El primer prompt define la calidad del artifact resultante.
Ahorrarás decenas de iteraciones si usas una estructura sólida.
Usa este formato recomendado:

- **PROPÓSITO**: Una frase clara del objetivo.
- **COMPONENTES**: Lista numerada de elementos visuales.
- **DATOS DE EJEMPLO**: JSON o tabla para evitar estados vacíos.
- **ESTILO**: Paleta, tipografía y *mood*.
- **INTERACCIONES**: Qué ocurre al hacer click o hover.
- **RESTRICCIONES**: Por ejemplo, mobile-first o librerías permitidas.

Ejemplo aplicado a tu Asistente:

```text
Crea un Artifact en React que muestre el dashboard del Asistente Institucional.

PROPÓSITO: Una página que un stakeholder lea en 60 segundos.
COMPONENTES:
1. Header con logo y status global.
2. 4 cards: solicitudes, escalación, latencia, costo.
3. Tabla de módulos con badge de estado.
DATOS: JSON realista embebido.
ESTILO: Tailwind, azul Albatros, fuente Inter.
INTERACCIONES: tabs sin recargar, sparklines en cards.
RESTRICCIONES: mobile-first, solo recharts y lucide-react.
```

## Iteración fluida (el corazón del valor)

Las iteraciones se logran conversando en lenguaje natural:

- *"Cambia el badge a rojo cuando latencia > 3s."*
- *"Agrega un input para buscar."*
- *"Tira esto y empieza de cero a dos columnas."*

El modelo edita el artifact en tiempo real en la misma pantalla.
Si el cambio empeora las cosas, simplemente **vuelves a la versión previa** usando el selector.

## ¿Cuándo decide Claude crear un Artifact?

Claude sigue reglas internas para abrir el panel:

- **Genera un Artifact si**: Hay >15 líneas de código, >800 palabras de texto, o contenido interactivo.
- **No genera un Artifact si**: La pregunta es exploratoria, el contenido cambia mucho cada turno, o es privado.
- **Control manual**: Puedes forzarlo diciendo *"créalo como Artifact"* o *"dámelo inline"*.

## Publicación y Remix

Al hacer clic en **Publish**, generas una URL pública en `claude.site`.

- **Ventajas**: Cualquiera con el enlace puede probar tu demo rápidamente.
- **Limitaciones**: No hay autenticación, ni persistencia de estado entre visitas.
- **Producción**: Para usos reales, debes descargar el código y desplegarlo tú mismo.

Con **Remix**, puedes "clonar" un Artifact público en tu propio chat.
Es como hacer *fork* en GitHub, ideal para construir sobre ideas que ya funcionan.

## Prácticas de excelencia

1. **Primer prompt completo**: Ahorra muchísimas correcciones.
2. **Datos plausibles**: Evita *lorem ipsum*, usa valores realistas.
3. **Cambios pequeños**: Pide una sola cosa por turno para evitar roturas.
4. **Retrocede si es necesario**: Usa el historial de versiones en lugar de parchear.
5. **Cierre de calidad**: Pide una pasada final de accesibilidad y responsividad.

## Errores frecuentes

- **Todo de una vez**: Es mejor hacer layout primero, luego datos, luego interacciones.
- **Ignorar el diseño**: Si no defines paleta o tipografía, se verá muy genérico.
- **Fetch a APIs**: El entorno seguro no tiene acceso a red externa; usa datos estáticos.

## Caso aplicado: tu dashboard semanal

Cada lunes actualizas un JSON con métricas semanales.
Lo subes al chat junto al prompt base de tu dashboard.
Claude actualiza el Artifact en segundos y publicas el link al equipo.
