# 3.4 Mini-apps con Artifacts (React, HTML, p5.js)

Tu institución no necesita "el siguiente Spotify".
Necesita **20 mini-apps cotidianas** que hoy se hacen a mano.
Hablamos de cuestionarios, calculadoras, simuladores, formularios y dashboards. 

Aprenderás a producir cada una en menos de 30 minutos, sin tocar `npm install`.

::concepto{titulo="Mini-app como Artifact"}
Una **mini-app como Artifact** es software autocontenido (≤200 KB) que vive en un Artifact.
Ejecuta lógica interactiva directamente en el navegador del usuario.
Se construye conversacionalmente y soporta tres familias técnicas:

- **HTML/CSS/JS estático:** Para formularios, landings y cuestionarios sin estado complejo.
- **React + Tailwind + libs curadas:** Ideal para dashboards y herramientas con estado.
- **p5.js:** Perfecto para simulaciones físicas, arte generativo y demos educativas.

La gran diferencia: **se generan, iteran y publican sin salir del chat**.
::/concepto

::interioriza{titulo="Baja Curva: La navaja suiza institucional"}
Piensa en los Artifacts como tener a un conserje con una navaja suiza.
Antes, pedir una calculadora de cupos era como contratar una constructora para hacer una rampa.
Había planos, permisos (setup, deploy) y semanas de espera.

Ahora, con Artifacts, se lo pides a tu Asistente y lo "construye" en 25 minutos.
Te permite crear herramientas que antes descartabas por "no valer el esfuerzo".
::/interioriza

## Las 6 Mini-apps Esenciales

### 1. Dashboard React (la más común)
- **Stack:** React + Tailwind + recharts + lucide-react.
- **Estructura:** Header con periodo, 4 métric-cards y tabla filtrable.
- **Producción:** ~25 minutos con 3-5 iteraciones.
- *Tip:* Pide datos de ejemplo realistas desde el principio.

### 2. Calculadora con lógica de negocio
- **Stack:** HTML + CSS + JS plano.
- **Caso de uso:** Calculadora de cupos disponibles.
- **Flujo:** Inputs (totales, retención) → Cálculo → Output (semáforo).
- Es más simple, pero ofrece el mismo poder analítico.

### 3. Simulador p5.js
- **Stack:** p5.js (nativa en Artifacts).
- **Caso de uso:** Simular la propagación de un comunicado a padres.
- **Visual:** Nodos animados que muestran retrasos en recepción.
- Ideal para visualizaciones educativas, no para productividad.

### 4. Formulario inteligente
- **Stack:** React (usando `useState` para control).
- **Caso de uso:** Pre-formulario de inscripción multi-paso.
- **UX:** Barra de progreso, validación en cliente, y resumen PDF.

### 5. Cuestionario / Quiz educativo
- **Stack:** React.
- **Caso de uso:** Examen de autoevaluación.
- **Lógica:** Feedback inmediato, score final y preguntas aleatorias.

### 6. Generador de comunicados (Template Engine)
- **Stack:** React.
- **Caso de uso:** Generar avisos por suspensión o eventos.
- **Output:** Texto editable y exportación rápida.

## Patrón general: Descomposición progresiva

No pidas la mini-app entera de un golpe. Descompón el trabajo:

- **1. Layout vacío (5m):** Solo estructura visual.
- **2. Datos de ejemplo (5m):** Agrega contenido realista.
- **3. Lógica básica (10m):** Interacciones principales.
- **4. Estados de borde (5m):** Manejo de errores y carga.
- **5. Polish final (5m):** Accesibilidad y microinteracciones.

Pídela por partes para garantizar una base sólida en cada iteración.

## Cuándo migrar a una app real

Tu mini-app fue un éxito. Migra cuando:
- Requieras **autenticación real** o datos persistentes.
- Superes los **20 usuarios concurrentes**.
- Necesites integrar **APIs externas** (bloqueadas por CORS en sandbox).

La migración toma una tarde: descargas el código, lo pones en Vite/Next y añades BD.
