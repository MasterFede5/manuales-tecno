# 3.1 ¿Qué son los Artifacts? Tipos

Le pides al modelo "explícame regresión lineal" y te entrega 600 palabras. Útil. 
Pero si pides "muéstrame regresión lineal con un slider", te entrega una **mini-app interactiva** al lado.
Esa diferencia entre texto y un panel interactivo se llama **Artifact**.

::concepto{titulo="Artifact / Canvas (concepto)"}
Un **Artifact** (Claude) o **Canvas** (ChatGPT, Gemini) es un **panel separado del chat** generado por la IA.
Aloja contenido **manipulable, ejecutable o iterable**: código, mini-apps, diagramas o SVGs.
El chat lo crea y edita; el artifact lo muestra, ejecuta y conserva.

Diferencias clave:
- **Respuesta normal:** texto en el chat, efímero, difícil de iterar.
- **Artifact:** panel persistente, editable a pedido, exportable y embebible.

Los Artifacts convierten al chatbot de un simple oráculo a una **superficie de creación**.
::/concepto

::interioriza{titulo="El Asistente como Mecánico"}
Imagina un mecánico reparando tu coche. 
Sin Artifacts, solo te describe por teléfono cómo cambiar la pieza.
Con Artifacts, te fabrica la pieza exacta y te la entrega lista para instalar.
La conversación es la consultoría; el Artifact es el entregable final.
::/interioriza

## Los 8 tipos canónicos

| Tipo | Qué es | Ejemplo en tu Asistente | Mejor proveedor |
|---|---|---|---|
| **Texto largo** | Documento >800 palabras editable | PRD, política de uso, propuesta a dirección | Claude Artifacts, ChatGPT Canvas |
| **Código** | Bloques de código ejecutables o listos para copiar | Script `evals/runner.py`, esquema JSON | Cualquiera |
| **HTML/CSS** | Página estática lista para ver | Mockup landing del Asistente | Claude Artifacts (renderiza en vivo) |
| **React** | Componente interactivo con estado | Dashboard con sliders, formularios, gráficas | Claude Artifacts (rendering nativo) |
| **Mermaid** | Diagramas declarativos | Arquitectura del Asistente, flujos n8n | Claude Artifacts, Notion AI |
| **SVG** | Gráfico vectorial editable | Logo experimental, infografías | Claude Artifacts, ChatGPT Canvas |
| **Markdown formateado** | Texto con renderizado rico | Manuales, READMEs, posts | Cualquiera |
| **JSON / YAML** | Datos estructurados | Schema de output, config de prompt | Cualquiera |

## Ciclo de vida de un Artifact

1. **Creación:** pides algo complejo y el modelo abre el panel.
2. **Iteración:** solicitas cambios en el chat y el modelo edita el documento.
3. **Versionado:** cada edición genera una versión navegable (v1, v2...).
4. **Compartir:** generas una URL pública con un botón.
5. **Embeber o exportar:** descargas el código o lo integras en tu sitio.

## Cuándo SÍ usar Artifact

- Texto >15 líneas que vas a editar en pasadas.
- Código que el usuario quiere copiar o ejecutar.
- Cualquier cosa con HTML/SVG/diagrama.
- Mini-app interactiva (slider, formulario, gráfica).
- Pieza que pretendes compartir con alguien que no está en el chat.

## Cuándo NO usar Artifact

- Respuesta corta (<3 párrafos).
- Conversación exploratoria sin entregable.
- Información que cambiará 30 veces sin estabilizarse.
- Datos sensibles que no deberían tener URL pública.

A veces el modelo se equivoca y crea un Artifact innecesario.
Si eso pasa, dile: *"no uses Artifact, respóndeme inline"*.
Él obedecerá y cerrará el panel.

## Limitaciones reales (lo que no te dicen los marketing)

- **Sin acceso a internet:** El React corre en un sandbox sin peticiones externas.
- **Bibliotecas limitadas:** Soportan React, Tailwind y librerías pre-curadas. No hay `npm install`.
- **Estado efímero:** El panel no persiste datos al recargar la página.
- **Tamaño máximo:** Tienen un límite de ~150KB. Si lo pasas, el código se corta.
- **Sin backend nativo:** Para bases de datos, necesitas workflows o agentes.

## Caso aplicado: dashboard del Asistente

El Episodio 3 requiere un dashboard. Así lo estructurarías:

- **Resumen de métricas:** Componente React con gráficas `recharts`.
- **Estado de módulos:** Tabla HTML en vivo.
- **Bitácora (ADRs):** Documento Markdown formateado.
- **Arquitectura:** Diagrama visual de Mermaid.

El modelo empaquetará esto en **un solo Artifact React** con pestañas.
Tendrás una app funcional sin configurar servidores locales.
