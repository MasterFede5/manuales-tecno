# 3.5 Compartir y embeber

Tu Artifact funciona perfecto en tu pantalla. ¿Cómo lo ven los demás?
Tienes opciones: un link, un iframe, un download o un deploy real.
Cada vía tiene sus propias reglas y niveles de seguridad.

::concepto{titulo="Distribución de un Artifact"}
**Distribución** es hacer que un Artifact salga de tu chat y llegue a terceros.
Existen cuatro vías principales:
- **Link público** (publish nativo).
- **Embed con iframe** (incrustar).
- **Download** (descargar HTML/JSX).
- **Deploy real** (Vercel/Netlify).

Cada vía resuelve un problema distinto con riesgos específicos.
::/concepto

::interioriza{titulo="Analogía: El Platillo en la Cocina"}
Construir es cocinar; distribuir es servir el plato en la mesa correcta.
Si publicas mal, tu obra maestra se queda enfriando en la cocina.
Si publicas bien, tu director presume tu dashboard en plena junta directiva.
::/interioriza

## Vía 1: Link público (la comida rápida)

- **Cómo:** Botón **Publish** o **Share** en tu IA.
- **Resultado:** URL pública tipo `claude.site/abc123`.
- **Pros:** Cero fricción, cero costo y cero configuración.
- **Contras:** Sin autenticación, control ni métricas. El proveedor puede borrarlo.
- **Cuándo usar:** Demos rápidas, prototipos o contenido sin datos sensibles.

## Vía 2: Embed con iframe (el servicio a domicilio)

- **Cómo:** Copias un código `<iframe>` y lo pegas en Notion o intranet.
- **Pros:** El Artifact vive dentro de tu página. Parece tuyo.
- **Contras:** La URL de fondo sigue siendo pública. Tienes limitaciones técnicas.
- **Cuándo usar:** Dashboards internos en herramientas como Notion o sitios estáticos.

## Vía 3: Download (la receta para llevar)

- **Cómo:** Botón **Download** o copiar el código fuente generado.
- **Pros:** Código 100% tuyo y auditable. Sin dependencia de la IA.
- **Contras:** Necesitas un servidor para alojarlo. No hay link mágico.
- **Cuándo usar:** Buscas independencia total o preparar el código para producción.

## Vía 4: Deploy real (tu propio restaurante)

- **Cómo:** Vite local + código Artifact + GitHub + Vercel.
- **Pros:** Dominio propio, autenticación real, bases de datos y analíticas.
- **Contras:** Requiere tiempo de configuración y mantenimiento continuo.
- **Cuándo usar:** Aplicaciones en producción con tráfico real y datos protegidos.

## Riesgos y mejores prácticas

- **Datos expuestos:** Anonimiza siempre antes de hacer Publish.
- **URL efímeras:** Descarga el código apenas funcione. No asumas que vivirá siempre.
- **Fugas en embeds:** Si un iframe público muestra datos internos, tus datos son públicos.

## Patrón y Caso Aplicado

- **Demo a director:** Usa link público.
- **Dashboard interno:** Embebe en el Notion del equipo.
- **Producción:** Despliega en Vercel con autenticación.

**Flujo real:** Actualizas en Claude -> Publish a Slack -> Download al repo -> Deploy a Vercel.
