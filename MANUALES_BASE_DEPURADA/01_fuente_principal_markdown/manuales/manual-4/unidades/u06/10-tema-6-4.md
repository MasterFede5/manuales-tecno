# 6.4 Plataformas: ChatGPT Agents, Claude Computer Use, Manus, Devin, Lindy

El espacio de plataformas de agentes explotó en 2024-2025.
Aquí tienes el mapa de las 6 más relevantes.
Aprenderás a elegir la adecuada para tu Asistente Institucional.

::concepto{titulo="Plataformas de agentes (2025)"}
A diferencia de los workflows, las **plataformas de agentes** siguen evolucionando rápido.
Cada una tiene un enfoque distinto:

- **ChatGPT Agents**: Integrados al chat de OpenAI. Útiles para conocimiento y acciones vía Zapier.
- **Claude Computer Use**: Controla un sistema operativo (mouse y teclado) como un humano.
- **Manus**: Agente generalista y muy autónomo (popular en 2025).
- **Devin**: Creador de software autónomo, especializado en programar.
- **Lindy**: Construcción de agentes de negocio sin código.
- **AutoGen / CrewAI**: Frameworks open source para agentes a medida.

No existe una "mejor" plataforma universal.
La elección depende de tu caso, riesgo y presupuesto.
::/concepto

::interioriza{titulo="Baja Curva: La metáfora del estacionamiento"}
Elegir plataforma de agentes es como elegir vehículos para una empresa:
- **Lindy** es la motoneta fácil para repartos rápidos.
- **Devin** es la excavadora para la obra (software).
- **Claude Computer Use** es el todoterreno para caminos de terracería (interfaces antiguas sin API).
Si envías una motoneta a la terracería, fallará. Conoce tu flota.
::/interioriza

## ChatGPT Agents (OpenAI)

**Acceso:** ChatGPT Plus / Team / Enterprise.
- **Tasks:** Agenda acciones recurrentes.
- **Operator:** Agente que controla un navegador en la nube.

**Pros y Contras:**
- 👍 Integración nativa con OpenAI y tools listos.
- 👍 Operator es un genuino agente de navegación web.
- 👎 Operator solo existe en la nube, no en tu PC.
- 👎 Menos personalizable que el open source.

**Caso ideal:** Investigación web rutinaria y manejo de apps populares.

## Claude Computer Use (Anthropic)

**Acceso:** API Anthropic (Claude 4.7 Opus / Sonnet).
- El modelo analiza **capturas de pantalla**.
- Ejecuta clics, scroll y tipeo hasta cumplir el objetivo.

**Pros y Contras:**
- 👍 Funciona en cualquier app del sistema (no solo web).
- 👍 Ideal para UIs sin API disponible.
- 👎 Alta latencia y costo por llamar repetidamente al modelo.
- 👎 Riesgo de seguridad sin entorno aislado (sandbox).

**Caso ideal:** Automatizar sistemas legacy (antiguos) que solo tienen interfaz gráfica.

## Manus

**Acceso:** Cloud mediante manus.im.
- Agente generalista.
- Descompone tareas complejas en sub-pasos.

**Pros y Contras:**
- 👍 Muy autónomo (funciona hasta 30 mins sin intervención).
- 👍 Excelente en research y reportes largos.
- 👎 Tus datos pasan por infraestructura en China.
- 👎 Baja privacidad y poca personalización.

**Caso ideal:** Investigaciones de mercado profundas (one-shot).

## Devin y Lindy

**Devin (Cognition Labs):**
- Especialista en crear software, resolver issues y hacer Pull Requests.
- 👍 Excelente en su nicho (programación).
- 👎 Muy costoso y limitado solo a desarrollo.

**Lindy:**
- Plataforma no-code para crear agentes de negocio.
- 👍 Muy accesible, con plantillas y 200+ integraciones.
- 👎 Solo en la nube y personalización moderada.

**Casos ideales:** Devin para equipos TI. Lindy para tu primer agente rápido.

## Frameworks Open Source

**Opciones:** AutoGen, CrewAI, LangGraph.
- Bibliotecas para construir agentes usando código local.
- Soporte para multi-agentes y grafos complejos.

**Pros y Contras:**
- 👍 Control total, máxima privacidad y sin costo de licencia.
- 👎 Requiere conocimientos de programación y mantenimiento propio.

**Caso ideal:** Instituciones maduras que exigen privacidad total y tienen programadores.

## Fases y Anti-patrones

**Fases recomendadas:**
1. **Inicial:** Lindy o n8n AI Agent (rápido y no-code).
2. **Media:** AutoGen/LangGraph cuando exijas privacidad total.
3. **Avanzada:** Lindy (rutinas) + AutoGen (casos complejos) + Claude (UI legacy).

**Anti-patrones (Evítalos):**
- **AutoGen sin programador:** Inviable.
- **Claude sin Sandbox:** Peligro de borrar archivos reales.
- **Generalistas para tareas críticas:** Usa agentes especializados.
- **Casarse temprano:** Diseña prompts portables, la plataforma cambia rápido.
