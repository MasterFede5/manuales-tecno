# 6.2 Anatomía: percepción, planificación, herramientas, memoria

Un agente que parece "inteligente" tiene siempre los mismos 5 órganos. 
Si falta uno, el resto no compensa. 
Esta lección los disecciona uno por uno con ejemplos del Asistente Institucional.

::concepto{titulo="Anatomía de un agente"}
Un agente robusto tiene 5 componentes integrados:

- **Percepción:** recibe el input del mundo (texto, imagen, audio, eventos).
- **Planificación:** descompone el objetivo en sub-tareas.
- **Herramientas (tools):** funciones invocables disponibles.
- **Memoria:** recuerda datos entre pasos y sesiones.
- **Ejecución:** coordina todo, controla el loop y aplica safeguards.

Ningún componente actúa aislado. 
La percepción alimenta la planificación, que elige tools.
Los tools modifican memoria y la ejecución gobierna el ritmo.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Piensa en contratar a un piloto para un auto escolar.
No basta con el motor: necesita ojos (percepción), navegador (planificación) y volante (herramientas).
También requiere bitácora (memoria) y ser el conductor (ejecución).
Tu agente es un sistema completo, no solo un LLM conectado a APIs.
::/interioriza

## Componente 1 — Percepción
**Cómo recibe el agente la información del mundo.**

- **Texto:** chat, email, ticket, documento.
- **Imagen:** análisis multimodal (GPT-4o, Gemini).
- **Audio:** transcripciones.
- **Eventos:** webhooks, schedules.
- **Estado de UI:** computer use (ver la pantalla).

*Práctica:* El Asistente recibe texto del padre, capturas de pantalla y eventos del sistema (ej. correos de admisiones).

## Componente 2 — Planificación
**Cómo descompone el objetivo en pasos.**

- **Reactivo:** decide cada paso según el último estado.
- **Plan-and-execute:** genera plan completo y luego ejecuta.
- **ReAct:** alterna Thought-Action-Observation.
- **Reflexion:** añade autoevaluación tras cada paso.
- **Tree-of-Thoughts:** explora múltiples planes.

*Práctica:* Para el Asistente, "plan-and-execute" es más estable. En tareas creativas, usa "ReAct" con reflexión.

## Componente 3 — Herramientas (tools)
**Lo que el agente puede hacer en el mundo.**

- **Información:** buscar en web, RAG, queries a DB.
- **Acción:** crear registros, mandar emails, agendar.
- **Cómputo:** correr código, cálculos.
- **Comunicación:** mandar Slack, llamar humanos.
- **Observación:** leer estado de sistemas.

Cada tool necesita un **schema** claro, **descripción** precisa y **permisos**.

```json
{
  "name": "create_notion_page",
  "description": "Crea página en Notion cuando llega solicitud nueva",
  "input_schema": {
    "type": "object",
    "properties": {
      "nombre": {"type": "string"},
      "email": {"type": "string"}
    }
  }
}
```

*Nota:* Inicia con 6-12 tools bien definidas. Más opciones generan ruido.

## Componente 4 — Memoria
**Qué persiste entre pasos, conversaciones y sesiones.**

- **Working memory:** contexto inmediato (context window).
- **Episodic memory:** historial de la charla actual.
- **Semantic memory:** conocimiento estructurado (RAG).
- **Procedural memory:** patrones aprendidos.
- **Long-term memory:** base de datos permanente del usuario.

*Práctica:* Para el 90% de casos, basta el context window + RAG + perfil en DB.

## Componente 5 — Ejecución (orquestador)
**El motor que conduce todo el loop.**

Responsabilidades:
- Conecta percepción con planificación.
- Invoca el tool correcto según decisión del LLM.
- Controla límites: max_iterations, max_tokens.
- Maneja errores (retry) y loops de aprobación humana.
- Registra logs para observabilidad.

## Patrón de implementación

```python
agent = create_agent(
    model="claude-opus",
    tools=[search_kb, create_notion, send_email],
    memory=working_mem + rag_mem,
    max_iterations=10,
    on_error=alert_human
)
```

## Errores frecuentes
- **Falta un componente:** sin memoria olvida, sin límites gasta.
- **Tools ambiguos:** el agente se confunde al elegirlos.
- **Exceso de tools:** más de 30 paraliza al modelo.
- **Memoria sin limpiar:** contexto lleno de basura.
- **Sin logs:** imposible debuggear los pasos internos.
