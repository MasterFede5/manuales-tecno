# 1.4 IA, ML, DL y LLMs: jerarquía conceptual

- Términos como "IA", "machine learning", "deep learning" y "LLM" no son sinónimos.
- Son **subconjuntos** jerárquicos: cada uno representa una técnica distinta.
- Saber distinguirlos te permite elegir la herramienta correcta para cada problema.

::concepto{titulo="Jerarquía IA → ML → DL → LLMs"}
- **IA:** campo amplio de sistemas con apariencia de inteligencia (reglas, lógica o aprendizaje).
- **ML (Machine Learning):** subconjunto de IA que **aprende de datos** sin reglas explícitas.
- **DL (Deep Learning):** subconjunto de ML que usa **redes neuronales profundas**.
- **LLMs (Large Language Models):** subconjunto de DL de redes gigantes entrenadas en texto.
$$\text{IA} \supset \text{ML} \supset \text{DL} \supset \text{LLMs}$$
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
- Confundir los términos lleva a errores costosos. 
- "Necesito ML para mi web" — quizá un simple filtro de reglas basta.
- Diferenciarlos te convierte en alguien capaz de tomar decisiones técnicas serias.
::/interioriza

## IA: el campo más amplio

- Incluye IA simbólica clásica (sistemas expertos, reglas IF-THEN).
- Incluye algoritmos de búsqueda y planificación (como Deep Blue o la NASA).
- **Punto clave:** Un sistema basado en reglas es IA, pero **no** aprende de datos.

## ML (Machine Learning): IA que aprende de datos

Aquí el programador no escribe reglas directas:
1. Reúne ejemplos etiquetados y define una arquitectura.
2. Entrena el modelo para que ajuste sus parámetros minimizando errores.
3. Evalúa con datos nuevos para ver si generaliza bien.

| Tipo | Qué aprende | Ejemplo |
|---|---|---|
| **Supervisado** | Datos con etiqueta | Clasificador de spam |
| **No supervisado** | Datos sin etiqueta | Detección de anomalías |
| **Por refuerzo** | Política por recompensas | AlphaGo, robótica |

**Algoritmos clásicos de ML (sin redes neuronales):**
- Regresión lineal/logística, Árboles de decisión (Random Forest), SVM, K-means.
- **Punto clave:** Mucho ML del mundo real **NO es deep learning** (para Excel un Random Forest suele ganar).

## DL (Deep Learning): redes neuronales profundas

- Una red neuronal conecta capas de "neuronas artificiales" (pesos y funciones de activación).
- Al ser "profundas" (muchas capas), **aprenden las features automáticamente** de datos crudos.

**Arquitecturas más usadas:**

| Arquitectura | Para qué | Ejemplo |
|---|---|---|
| **CNN** | Imágenes y video | ResNet, YOLO |
| **RNN / LSTM** | Secuencias temporales | Audio, series |
| **Transformer**| Texto, secuencias | GPT, Claude |
| **Diffusion** | Generar imágenes/video | DALL·E 3, Sora |

- **Costo:** El DL requiere muchos datos y GPUs, se usa cuando la tarea lo justifica.

## LLMs: el subconjunto que generó el boom

- Los LLMs son Transformers masivos entrenados en texto para predecir el siguiente token.
- **Características:** De millones a billones de parámetros (GPT-4 tiene ~1.7T).
- **Capacidad emergente:** Resuelven tareas sin reentrenamiento específico, solo con ejemplos en el prompt (few-shot).

**Familias actuales:**
- **GPT (OpenAI):** GPT-4, GPT-4o, o1, o3.
- **Claude (Anthropic):** Claude 3.5, Claude 4.
- **Gemini (Google):** Gemini 1.5, Gemini 2.0.
- **Llama (Meta):** LLM abierto (open source).
