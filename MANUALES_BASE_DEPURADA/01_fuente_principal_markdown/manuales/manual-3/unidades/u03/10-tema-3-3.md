# 3.3 Role prompting y output structuring

Dos técnicas pequeñas transforman radicalmente tu uso de la IA: 
* **Role prompting:** Darle un rol específico (ej. "tutor paciente").
* **Structured output:** Forzar el formato de salida (ej. JSON).

::concepto{titulo="Role prompting y structured output"}
* **Role prompting:** Asignar una **personalidad específica**.
* Modifica el tono, vocabulario, profundidad y prioridades del modelo.
* **Structured output:** Exigir un **formato exacto y parseable**.
* Usa formatos como JSON, XML, YAML o tablas para facilitar el procesamiento.
::/concepto

::interioriza{titulo="Analogía: El Actor y el Formulario"}
Piensa en el LLM como un actor de teatro. Si no le das un **Role**, será un oficinista genérico y aburrido. 
Si le dices "eres un tutor inspirador", se pone el disfraz y adapta su guion al perfil de los estudiantes.
El **Structured output** es darle un formulario oficial, forzándolo a entregar datos organizados en casillas listas para tu sistema.
::/interioriza

## Role prompting: el modelo es quien le digas

Compara estos enfoques:

**Sin role:**
```text
Explícame el principio de Bernoulli.
```

**Con role:**
```text
Eres un tutor de física para bachillerato. Tu estilo es de un coach paciente:
detectas malentendidos, das ejemplos cotidianos y confirmas comprensión.
Antes de explicar, pregunta qué sabe el estudiante.
Explícame el principio de Bernoulli.
```

El segundo prompt cambia **completamente** la respuesta. 
Pasas de una definición enciclopédica a un diálogo interactivo y personalizado.

Cinco respuestas radicalmente distintas para el mismo modelo." paginas=1}

### Roles potentes para estudio

* `Eres mi tutor paciente. Pregúntame qué sé antes de responder.`
* `Eres mi abogado del diablo. Critica mi argumento sin suavizar.`
* `Eres un editor severo. Marca redundancias y jerga vaga.`
* `Eres profesor universitario riguroso. Señala errores en mi pregunta.`

### Persona-style con tone y voice

Refina aún más para tareas creativas:

```text
Eres copywriter senior con 10 años en agencias mexicanas.
Estilo: directo, conversacional, con humor seco.
Voice: tú-tú al lector. Activa, no pasiva.
Tone: persuasivo pero sin clichés de "transforma tu vida".
```

## Structured output: el modelo te devuelve datos

Cuando vas a **procesar** la respuesta, necesitas un formato exacto.
Esto permite conectar el LLM con código, hojas de cálculo u otros modelos.

### JSON

```json
Devuelve la respuesta en JSON con esta estructura exacta:
{
  "concepto": "string",
  "preguntas_comprension": ["string"]
}
NO incluyas texto fuera del JSON.
```

> **Buena práctica:** Muestra el esquema explícito o usa few-shot con un JSON completo.

### Otros formatos

* **Tabla markdown:** `Lista 5 LLMs en formato tabla markdown con columnas...`
* **XML:** Útil para sistemas legacy (`<respuesta><concepto>...</concepto></respuesta>`).
* **YAML:** Ideal para configuración (`modelo: claude-opus-4-7`).

::visual{tipo="cuadro-comparativo" descripcion="Cuadro comparativo de 4 formatos de structured output con misma pregunta 'lista 3 LLMs con su empresa, año y URL': 1) MARKDOWN — tabla con 5 columnas. 2) JSON — array de objetos con campos. 3) XML — etiquetas anidadas. 4) YAML — claves con dos puntos y valores. Cada formato con caso de uso típico: Markdown para humanos, JSON para procesar con código, XML para legacy, YAML para configs." paginas=0.5}

## Cuándo combinarlos

Para tareas complejas y automatizadas, combina ambas técnicas:

```text
Eres analista experto en datos. (ROLE)
Tu tarea es clasificar correos. Para cada correo devuelve JSON: (FORMAT)
{
  "categoria": "PERSONAL | LABORAL | SPAM",
  "prioridad": "ALTA | MEDIA | BAJA"
}
Correo a clasificar: [contenido]
```
Esto genera una respuesta lista para un script de procesamiento masivo.
