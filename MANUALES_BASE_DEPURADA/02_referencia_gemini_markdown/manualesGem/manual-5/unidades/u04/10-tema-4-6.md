# 4.6 Detección de outliers

Tu scatter de horas-vs-cal muestra un alumno con 30 horas de estudio y calificación 4.5. ¿Error de captura? ¿Caso real? ¿Lo eliminas? Esta lección te enseña a decidir.

::concepto{titulo="Outliers"}
Un **outlier** es un valor "muy lejos" del resto de los datos. Tres métodos para detectarlo:

1. **IQR (Inter-Quartile Range)**: punto fuera de [Q1 - 1.5×IQR, Q3 + 1.5×IQR].
2. **Z-score**: punto con |z| > 3 (más de 3 desviaciones de la media). Asume normal.
3. **Percentil**: punto fuera de [P1, P99] (extremos absolutos).

Decidir qué hacer con un outlier es siempre **decisión humana**: error vs caso atípico real.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Outliers afectan media, std, correlación, modelos ML. Eliminarlos sin pensar pierde información valiosa; mantenerlos sin diagnóstico distorsiona análisis. La distinción entre "ruido" y "señal rara" es la habilidad clave.
::/interioriza

## Método IQR (recomendado por default)

```python
Q1 = df["cal_final"].quantile(0.25)
Q3 = df["cal_final"].quantile(0.75)
IQR = Q3 - Q1

limite_inf = Q1 - 1.5 * IQR
limite_sup = Q3 + 1.5 * IQR

outliers = df[(df["cal_final"] < limite_inf) | (df["cal_final"] > limite_sup)]
```

**Robusto a la distribución**. No asume normal.

## Método Z-score

```python
z = (df["cal_final"] - df["cal_final"].mean()) / df["cal_final"].std()
outliers = df[abs(z) > 3]
```

**Solo válido si distribución es aprox. normal.**

## Método percentil

```python
limite_inf = df["cal_final"].quantile(0.01)
limite_sup = df["cal_final"].quantile(0.99)
outliers = df[(df["cal_final"] < limite_inf) | (df["cal_final"] > limite_sup)]
```

Conservador: siempre habrá ~2 % marcados como outliers.

## Qué hacer con outliers

| Situación | Acción |
|---|---|
| Error obvio (cal=99 en escala 0-10) | Corregir o eliminar |
| Punto raro pero plausible | Mantener y documentar |
| Caso atípico de interés (genio o crisis) | Estudiar separadamente |
| Distorsiona modelo crítico | Evaluar quitarlo + reportar |
| Solo 1-2 outliers en 480 | Mantener (ruido normal) |

## Aplicado al case escolar

```python
# Detectar outliers en cal_final
Q1 = df["cal_final"].quantile(0.25)
Q3 = df["cal_final"].quantile(0.75)
IQR = Q3 - Q1
outliers = df[(df["cal_final"] < Q1 - 1.5*IQR) | (df["cal_final"] > Q3 + 1.5*IQR)]
print(f"Outliers cal_final: {len(outliers)}")

# Para cada outlier, decidir caso por caso
for _, row in outliers.iterrows():
    print(row[["id", "horas_estudio", "asistencia", "cal_anterior", "cal_final"]])
```

Si un outlier tiene horas=0 y asistencia=30 %, su cal=4.5 es coherente. **No es ruido**, es señal de un alumno en problemas. Vale la intervención.

## Anti-patrones

- **Eliminar outliers automáticamente con dropna(subset=outliers)**. Pierdes información.
- **Z-score en datos no-normales**. Da falsos negativos/positivos.
- **No documentar**. Si quitas 5 outliers, anótalo en metadata.
- **Suponer outlier = error**. A veces es el caso más interesante.
- **Eliminar a posteriori para "salvar" tu análisis**. Sesgo de confirmación.
