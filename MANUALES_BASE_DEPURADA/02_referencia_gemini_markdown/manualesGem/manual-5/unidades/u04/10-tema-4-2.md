# 4.2 Distribución normal y curva de campana

Si conoces media y desviación de una variable que se distribuye normal, sabes **cuántos datos** caen en cualquier rango. Esta es la base intuitiva de toda inferencia estadística.

::concepto{titulo="Distribución normal"}
La **distribución normal** (o gaussiana, o "campana") es una distribución de probabilidad simétrica caracterizada por su media μ y desviación estándar σ. Aparece **espontáneamente** en muchos fenómenos: alturas, calificaciones, errores de medición, tiempos.

**Regla 68-95-99.7:**
- 68 % de los datos caen en μ ± 1σ.
- 95 % caen en μ ± 2σ.
- 99.7 % caen en μ ± 3σ.

Esto te permite estimar probabilidades sin cálculos: "¿qué calificación es 'rara'?" → más allá de 2σ.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Si alguien te dice que sacó 9 cuando el grupo tuvo media 7 con std 1, sabes inmediatamente: **estaba a 2σ de distancia**, lo que pone su calificación en el top 2-3 % del grupo. Sin distribución normal, no podrías hacer esa estimación rápida.
::/interioriza

## Visualizar normal

```python
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats

# Curva normal teórica
x = np.linspace(0, 10, 100)
y = stats.norm.pdf(x, loc=7, scale=1.5)
plt.plot(x, y)
plt.fill_between(x, y, where=(x >= 5.5) & (x <= 8.5), alpha=0.3, label="±1σ (68%)")
plt.title("Distribución normal: μ=7, σ=1.5")
plt.legend()
plt.show()
```

## Aplicación práctica al dataset

```python
mu = df["cal_final"].mean()       # 7.42
sigma = df["cal_final"].std()     # 1.49

# ¿Qué % de alumnos sacó entre 6 y 9? (~1σ alrededor de la media)
print((df["cal_final"].between(mu-sigma, mu+sigma)).mean())  # ≈ 0.68

# ¿Cuántos sacaron > 9.5 (más de 1.4σ arriba)?
print((df["cal_final"] > 9.5).mean())  # ≈ 0.05 si normal

# ¿Es realmente normal?
# Si tu histograma se ve campana, sí. Si no, regla 68-95-99.7 falla.
```

## Cómo verificar normalidad

Visual:
```python
df["cal_final"].plot.hist(bins=30, density=True)
# Sobreponer normal teórica con misma μ y σ
```

Si las barras se aproximan a la curva, es razonable asumir normal. Si hay sesgos fuertes, no.

## Z-score: estandarizar

```python
df["cal_final_z"] = (df["cal_final"] - df["cal_final"].mean()) / df["cal_final"].std()
```

`z=2` significa "2 desviaciones arriba de la media". Permite comparar entre escalas distintas.

## Cuándo NO asumir normalidad

- **Variables acotadas** (ej.: edades en bachillerato, todas entre 15 y 19): no normal estricta.
- **Conteos** (asistencias, eventos): suelen ser Poisson.
- **Variables muy asimétricas** (salarios): log-normal.

Para nuestras calificaciones, normal es razonable como aproximación.

## Anti-patrones

- **Asumir normal sin verificar**. Histograma primero, siempre.
- **Aplicar regla 68-95-99.7 a datos asimétricos**. Falla.
- **Confundir distribución muestral con poblacional**. Tu muestra de 50 alumnos no necesariamente es normal aunque la población lo sea.
