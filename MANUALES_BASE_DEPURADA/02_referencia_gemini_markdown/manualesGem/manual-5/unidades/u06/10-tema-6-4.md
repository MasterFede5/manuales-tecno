# 6.4 PCA básico (intuición + uso)

Tienes 4 features (horas, asistencia, cal_ant, cal_final). No puedes graficar 4D. **PCA** los proyecta a 2D preservando lo máximo posible de variabilidad.

::concepto{titulo="PCA — Principal Component Analysis"}
**PCA** es una técnica de **reducción de dimensionalidad** que encuentra las direcciones (componentes principales) de máxima varianza en los datos y proyecta el dataset sobre ellas.

Resultado: dataset original con N features → dataset reducido con K componentes (K<N) preservando la mayor cantidad posible de información.

```python
from sklearn.decomposition import PCA
pca = PCA(n_components=2)
X_2d = pca.fit_transform(X_scaled)

print(f"Varianza explicada: {pca.explained_variance_ratio_}")
# [0.55, 0.28] → componente 1 explica 55%, componente 2 explica 28%, total 83%
```
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
PCA tiene 2 usos: (1) **visualización** de datos de muchas dimensiones en 2D/3D; (2) **acelerar modelos** con menos features. Sin PCA, 1000-feature datasets son inmanejables visualmente.
::/interioriza

## Caso uso 1: Visualizar clusters

```python
import seaborn as sns
import matplotlib.pyplot as plt

X_2d = PCA(n_components=2).fit_transform(X_scaled)

plt.figure(figsize=(10, 6))
sns.scatterplot(x=X_2d[:,0], y=X_2d[:,1], hue=df["cluster"], palette="viridis", s=50)
plt.title(f"Clusters proyectados a 2D (varianza: {pca.explained_variance_ratio_.sum():.0%})")
plt.xlabel("Componente 1")
plt.ylabel("Componente 2")
plt.show()
```

Los clusters se ven como zonas separadas si están bien definidos.

## Caso uso 2: Acelerar modelos con muchos features

```python
# De 100 features a 20 que preservan 95% de varianza
pca = PCA(n_components=0.95)  # mantén 95%
X_reducido = pca.fit_transform(X_scaled)
print(f"Features originales: {X_scaled.shape[1]} → Reducidas: {X_reducido.shape[1]}")
```

Modelos entrenan más rápido con menos features.

## Cuándo usar PCA

| Caso | PCA útil |
|---|---|
| Visualizar dataset con >3 features | ✓ |
| Acelerar modelos con >100 features | ✓ |
| Reducir ruido removiendo componentes pequeños | ✓ |
| Tienes 3-5 features | ✗ (no necesario) |
| Necesitas interpretabilidad por feature original | ✗ (PCA cambia features) |

## Componentes principales: ¿qué son?

Cada componente es **combinación lineal** de las features originales. Por ejemplo:

```
Componente 1 = 0.4*horas + 0.5*asistencia + 0.5*cal_ant + 0.5*cal_final
Componente 2 = 0.6*horas - 0.3*asistencia - 0.4*cal_ant + 0.5*cal_final
```

Componente 1 captura "el general factor de éxito"; componente 2 captura "esfuerzo aislado vs notas pasadas".

```python
print(pca.components_)
```

## Limitaciones

- **PCA es lineal**: no captura relaciones curvas.
- **Componentes pierden interpretabilidad**: ya no son features originales.
- **Sensible a scaling**: ¡siempre escala antes!.

Para no-lineal: t-SNE, UMAP (no en este manual).

## Anti-patrones

- **Sin scaling antes**. PCA será dominado por feature de mayor varianza.
- **PCA con datos categóricos crudos**. Convierte a numérico primero.
- **n_components = N** (igual al original). No reduce nada.
- **Interpretar componentes literalmente**. Son combinaciones, no variables originales.
