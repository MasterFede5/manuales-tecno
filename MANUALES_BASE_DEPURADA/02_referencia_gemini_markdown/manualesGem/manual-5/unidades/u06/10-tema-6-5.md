# 6.5 Detección de anomalías simple

Anomalías son puntos que **no encajan** en ningún cluster. En tu case: alumnos con perfil único — pueden ser geniales o en crisis. Vale la atención.

::concepto{titulo="Detección de anomalías"}
La **detección de anomalías** identifica puntos atípicos en un dataset. Útil para:

- Fraude (transacciones raras).
- Fallas (sensores fuera de rango).
- Alumnos atípicos (caso escolar).
- Errores de captura.

Métodos:
1. **Estadístico** (IQR, z-score) — visto en U4.
2. **Clustering** — puntos lejos de cualquier cluster.
3. **Isolation Forest** — algoritmo dedicado.
4. **One-Class SVM** — frontera de "normalidad".
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
En un dataset escolar, el alumno con asistencia 100 % pero calificación 5 es **anómalo**. Puede ser un caso interesante (problema personal, didáctico) o error de captura. Detectarlos automáticamente te alerta para revisar.
::/interioriza

## Método 1: Distancia al centroide más cercano

```python
from sklearn.cluster import KMeans
import numpy as np

kmeans = KMeans(n_clusters=4, random_state=42, n_init=10)
kmeans.fit(X_scaled)

# Distancia de cada punto al centroide más cercano
distancias = np.min(kmeans.transform(X_scaled), axis=1)

# Anomalías: top 5% más alejados
threshold = np.percentile(distancias, 95)
anomalias = df[distancias > threshold]
print(f"Anomalías: {len(anomalias)}")
```

## Método 2: Isolation Forest

```python
from sklearn.ensemble import IsolationForest

iso = IsolationForest(contamination=0.05, random_state=42)
df["es_anomalia"] = iso.fit_predict(X_scaled) == -1

print(f"Anomalías: {df['es_anomalia'].sum()}")
```

`contamination=0.05` espera 5 % de anomalías.

## Método 3: Local Outlier Factor (LOF)

```python
from sklearn.neighbors import LocalOutlierFactor

lof = LocalOutlierFactor(n_neighbors=20, contamination=0.05)
df["es_anomalia_lof"] = lof.fit_predict(X_scaled) == -1
```

LOF mide qué tan "denso" es el área alrededor de un punto. Si está en zona dispersa → anomalía.

## Aplicado al case escolar

```python
# Alumno extraño = alumno cuyo perfil no coincide con ningún arquetipo
iso = IsolationForest(contamination=0.05, random_state=42)
anomalias_mask = iso.fit_predict(X_scaled) == -1

print(f"Alumnos atípicos: {anomalias_mask.sum()}")
print(df[anomalias_mask][["horas_estudio", "asistencia", "cal_anterior", "cal_final"]])
```

## Decisión: ¿qué hacer con anomalías?

- **Error de captura** (cal=99 cuando es 9): corregir y reentrenar.
- **Caso atípico real** (genio): mantener y estudiar separadamente.
- **Alumno en crisis** (asistencia baja, cal media): intervención.
- **Outlier estadístico sin sentido**: documentar y mantener.

## Anti-patrones

- **Eliminar todas las anomalías** del dataset. Pierdes info.
- **`contamination` arbitrario**. Valida con dominio.
- **Anomalías sin investigar**. El número solo no sirve; cada caso requiere lectura.
