# 6.2 K-Means paso a paso

K-Means es **el algoritmo de clustering más popular** por simple, rápido y efectivo en muchos casos. Esta lección te lo explica en 4 pasos y código.

::concepto{titulo="K-Means"}
**K-Means** divide los datos en **K** clusters siguiendo un proceso iterativo:

1. **Inicializar**: elegir K centroides al azar.
2. **Asignar**: cada punto al centroide más cercano (euclidiana).
3. **Recalcular**: el centroide se mueve al promedio de sus puntos asignados.
4. **Repetir** 2-3 hasta convergencia (sin cambios).

Garantías: converge siempre, pero a un mínimo local (depende de inicialización).

```python
from sklearn.cluster import KMeans
kmeans = KMeans(n_clusters=4, random_state=42, n_init=10)
kmeans.fit(X_scaled)
labels = kmeans.labels_
centroids = kmeans.cluster_centers_
```
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
K-Means es como organizar tu armario por color: empiezas con cajas en lugares al azar, pones cada prenda en la caja del color más cercano, y luego mueves cada caja al "centro" de las prendas que tiene. Repites hasta estabilizar. Esto es exactamente K-Means.
::/interioriza

## Aplicado al case escolar

```python
import pandas as pd
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
import matplotlib.pyplot as plt
import seaborn as sns

df = pd.read_csv("dataset_escolar_limpio.csv")
features = ["horas_estudio", "asistencia", "cal_anterior", "cal_final"]
X = df[features]

# 1. Scaling (¡crucial!)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 2. Aplicar K-Means
kmeans = KMeans(n_clusters=4, random_state=42, n_init=10)
df["cluster"] = kmeans.fit_predict(X_scaled)

# 3. Caracterizar cada cluster
print(df.groupby("cluster")[features].mean().round(2))
```

Output típico:
```
         horas_estudio  asistencia  cal_anterior  cal_final
cluster
0                 6.5         0.92          8.5        8.7
1                 4.5         0.85          7.0        7.2
2                 2.0         0.55          5.0        4.5
3                 5.5         0.75          6.5        6.8
```

Interpretación:
- **Cluster 0**: estudiantes excelentes (estudian 6.5h, asisten 92%, ya venían bien).
- **Cluster 1**: buenos consistentes (4.5h, 85%, 7-7.5).
- **Cluster 2**: en riesgo (2h, 55% asistencia, reprueban).
- **Cluster 3**: medio-bajo (mejorable con apoyo).

## Inicialización: el detalle que importa

K-Means converge siempre, pero a **mínimo local**. Inicialización mala → clusters mediocres.

```python
KMeans(n_clusters=4, random_state=42, n_init=10)  # n_init=10 ← prueba 10 inicializaciones distintas, elige la mejor
```

`n_init=10` es default reciente. En versiones viejas era 1, causaba bugs. Verifica.

## Visualizar clusters

Con 2D (después de PCA, ver 6.4):

```python
from sklearn.decomposition import PCA
pca = PCA(n_components=2)
X_2d = pca.fit_transform(X_scaled)

```

## Inertia (criterio de calidad)

```python
print(f"Inertia: {kmeans.inertia_:.2f}")
```

Inertia = suma de distancias al cuadrado de cada punto a su centroide. **Menor es mejor**, pero siempre baja con más K (no es indicador único).

## Predecir cluster de un alumno nuevo

```python
nuevo = scaler.transform([[5.0, 0.9, 7.0, 0]])  # cal_final = 0 placeholder
cluster_predicho = kmeans.predict(nuevo)
print(f"Cluster: {cluster_predicho[0]}")
```

## Limitaciones de K-Means

- **K fijo**: tú decides cuántos. No descubre número óptimo solo.
- **Clusters esféricos**: no funciona bien con formas alargadas.
- **Sensible a outliers**: un outlier mueve el centroide.
- **Asume tamaños similares**.

Si tus clusters son raros, prueba DBSCAN o Gaussian Mixtures (no en este manual, glosario).

## Anti-patrones

- **Sin scaling**. Variable de mayor magnitud domina.
- **K arbitrario sin justificar**. K=3 "porque sí" → mal cluster. Usa codo/silueta (6.3).
- **Confiar en clusters como verdad absoluta**. Son descubrimientos exploratorios, no etiquetas verdaderas.
- **Olvidar `n_init`**. Versiones viejas con `n_init=1` dan clusters inestables.
