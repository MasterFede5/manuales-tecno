# 6.1 Concepto de clustering

Clustering es **agrupar elementos similares sin instrucciones humanas**. Tu computadora encuentra estructura latente en los datos.

::concepto{titulo="Clustering"}
**Clustering** es el proceso de dividir un conjunto de datos en grupos (**clusters**) tal que:
- Elementos dentro de un cluster son **similares** entre sí.
- Elementos de clusters distintos son **diferentes**.

A diferencia de clasificación supervisada, **no hay etiqueta de antemano**: el algoritmo descubre los grupos por similitud (típicamente distancia euclidiana entre features).

Algoritmos comunes:
- **K-Means**: el más simple y popular.
- **DBSCAN**: detecta clusters de forma arbitraria + outliers.
- **Hierarchical**: construye dendograma.
- **Gaussian Mixtures**: cluster probabilístico.

En este manual cubrimos K-Means, lo demás al glosario.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Sin etiquetas en datos, clasificación supervisada NO sirve. Clustering es la única vía para descubrir estructura. En negocio: segmentos de clientes. En educación: perfiles de aprendizaje. En medicina: subtipos de enfermedad.
::/interioriza

## Aplicaciones canónicas

- **Segmentación de clientes** (marketing).
- **Perfilado de usuarios** (UX).
- **Compresión de imágenes** (similar pixels = mismo color).
- **Detección de anomalías** (puntos alejados de cualquier cluster).
- **Recomendaciones** (usuarios similares).

Para nuestro case: **perfiles de aprendizaje** según patrón de horas, asistencia, calificaciones.

## Anatomía conceptual

```
Datos:
  Alumno 1: (5h, 0.9, 8)
  Alumno 2: (2h, 0.6, 5)
  Alumno 3: (5h, 0.85, 8.5)
  Alumno 4: (1h, 0.4, 4)
  ...

Clustering descubre:
  Cluster A (estudiosos): alumnos 1, 3, ...
  Cluster B (en riesgo): alumnos 2, 4, ...
  Cluster C (mediano): ...
```

## Distancia: el corazón del clustering

Para decir "similar", necesitas **métrica de distancia**:

- **Euclidiana**: línea recta. La más común.
- **Manhattan**: suma de diferencias absolutas.
- **Coseno**: ángulo entre vectores. Para texto.

K-Means usa euclidiana por default.

## Pre-procesamiento crítico: scaling

Si una variable es 0-1 (asistencia) y otra 0-100 (calificación porcentual), la calificación domina la distancia. **Normaliza** primero:

```python
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
```

`StandardScaler` lleva cada feature a media 0, std 1.

## Anti-patrones

- **Clustering sin scaling**. Una variable domina, distorsiona.
- **Asumir que K-Means descubre clusters de cualquier forma**. Solo encuentra esféricos. Para formas raras → DBSCAN.
- **Confundir cluster con label "verdadero"**. Los clusters son **descubrimiento**, no validación.
