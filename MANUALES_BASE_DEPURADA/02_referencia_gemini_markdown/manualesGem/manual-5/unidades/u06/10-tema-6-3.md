# 6.3 Cómo elegir K (codo, silueta)

K-Means te obliga a elegir K (cantidad de clusters). ¿Cómo eliges? Dos métodos clásicos: codo y silueta.

::concepto{titulo="Métodos para elegir K"}
**Método del codo (Elbow):** corres K-Means con K=2,3,4,...10. Graficas inertia vs K. El "codo" del gráfico (donde la pendiente cambia bruscamente) sugiere el K óptimo.

**Coeficiente de silueta:** mide qué tan bien cada punto pertenece a su cluster vs otros. Va de -1 a +1, mayor es mejor. Calculas para varios K y eliges el de mayor silueta.

```python
from sklearn.metrics import silhouette_score

inertias = []
silhouettes = []
for k in range(2, 11):
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    km.fit(X_scaled)
    inertias.append(km.inertia_)
    silhouettes.append(silhouette_score(X_scaled, km.labels_))
```
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
K mal elegido → clusters sin sentido. K=2 puede esconder grupos; K=20 puede dividir grupos reales en sub-grupos artificiales. La elección correcta es **decisión técnica + validación con expertos del dominio**.
::/interioriza

## Método del codo

```python
import matplotlib.pyplot as plt

plt.plot(range(2, 11), inertias, marker="o")
plt.xlabel("K (número de clusters)")
plt.ylabel("Inertia")
plt.title("Método del Codo")
plt.show()
```

Buscas el "punto donde la curva se aplana": antes baja rápido, después poco.

## Coeficiente de silueta

```python
plt.plot(range(2, 11), silhouettes, marker="o", color="orange")
plt.xlabel("K")
plt.ylabel("Silhouette Score")
plt.title("Silueta por K")
plt.show()
```

K óptimo = K con mayor silueta.

## Combinación con dominio

Métodos numéricos sugieren K=4. Pero **¿4 perfiles tiene sentido pedagógicamente?**. La respuesta debe combinar:
- Sugerencia numérica (codo, silueta).
- Conocimiento del dominio (¿coordinador puede actuar sobre 4 perfiles? ¿es manageable?).
- Interpretabilidad (¿son distinguibles cualitativamente?).

A veces K=3 es ligeramente peor numéricamente pero más útil operativamente.

## Otras señales

- **Davies-Bouldin Index**: menor es mejor. `from sklearn.metrics import davies_bouldin_score`.
- **Calinski-Harabasz**: mayor es mejor.
- **Gap Statistic**: complejo pero robusto.

Para principiante: codo + silueta + dominio bastan.

## Anti-patrones

- **Elegir K=3 "porque sí"**. Sin evidencia.
- **Confiar 100% en codo cuando no hay codo claro**. Algunos datasets dan curva suave.
- **Silueta alta con K=2 siempre ganando**. Solo dos clusters puede ser sub-segmentación insuficiente.
- **Ignorar dominio**. Tu coordinador no puede operar 12 intervenciones distintas.
