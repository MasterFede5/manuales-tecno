# 8.5 Mini-RAG sobre 3 documentos

Combinas embeddings + LLM para que tu Asistente responda con base en tus documentos institucionales. Esta lección construye RAG end-to-end en código.

::concepto{titulo="Mini-RAG"}
**RAG** (Retrieval-Augmented Generation) combina:
1. **Indexación**: chunks → embeddings → vector store.
2. **Consulta**: query → embedding → búsqueda → top-k chunks.
3. **Generación**: chunks como contexto + query → LLM → respuesta citada.

Versión "mini": <100 líneas de código. Reglamento + manual + política → Asistente experto en ello.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
LLM solo "alucina" datos institucionales. RAG lo aterriza en TUS documentos. La diferencia: respuestas reales y verificables.
::/interioriza

## Mini-RAG completo (~50 líneas)

```python
import os
import numpy as np
from anthropic import Anthropic
from openai import OpenAI

claude = Anthropic(api_key=os.environ.get("ANTHROPIC_API_KEY"))
openai_client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))

# 1. Cargar y chunkear
def cargar_y_chunkear(path, chunk_size=400):
    with open(path, "r", encoding="utf-8") as f:
        texto = f.read()
    
    palabras = texto.split()
    chunks = []
    for i in range(0, len(palabras), chunk_size):
        chunks.append({
            "texto": " ".join(palabras[i:i+chunk_size]),
            "fuente": path,
            "indice": i // chunk_size
        })
    return chunks

# 2. Embedder
def embed(texto):
    return openai_client.embeddings.create(
        model="text-embedding-3-small",
        input=texto
    ).data[0].embedding

# 3. Indexar
def indexar(rutas):
    todos = []
    for r in rutas:
        chunks = cargar_y_chunkear(r)
        for c in chunks:
            c["embedding"] = embed(c["texto"])
            todos.append(c)
    return todos

# 4. Buscar top-k
def buscar(query, indice, k=3):
    e_q = embed(query)
    sims = [
        np.dot(e_q, c["embedding"]) / (np.linalg.norm(e_q) * np.linalg.norm(c["embedding"]))
        for c in indice
    ]
    top_idx = np.argsort(sims)[::-1][:k]
    return [indice[i] for i in top_idx]

# 5. Generar respuesta
def responder(query, contexto):
    prompt = f"""Responde la pregunta usando exclusivamente los siguientes fragmentos.
Si la información no está, di "no la encontré en los documentos".
Cita la fuente al final entre [corchetes].

Fragmentos:
{chr(10).join([f"[{c['fuente']} chunk {c['indice']}]" + chr(10) + c['texto'] for c in contexto])}

Pregunta: {query}"""
    
    response = claude.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=500,
        messages=[{"role": "user", "content": prompt}]
    )
    return response.content[0].text

# === Uso ===
indice = indexar(["reglamento.txt", "manual_docente.txt", "politica.txt"])

query = "¿Cuántos días tengo para apelar una calificación?"
contexto = buscar(query, indice, k=3)
respuesta = responder(query, contexto)
print(respuesta)
```

Output ejemplo:
```
Tienes 5 días hábiles después de la publicación de la calificación 
para apelar [reglamento.txt chunk 7]. La solicitud debe entregarse 
por escrito a coordinación académica.
```

## Mejoras siguientes

1. **Persistir vector store** (ChromaDB) para no reembebir cada vez.
2. **Re-ranking** con modelo dedicado (Cohere) para refinar top.
3. **Hybrid search** (vector + keyword BM25).
4. **Citas estructuradas** con metadata clara.

## Aplicado al case escolar completo

```python
# Setup
docs = ["reglamento.txt", "manual_docente.txt", "politica_admisiones.txt"]
indice = indexar(docs)

# Query del coordinador
query = "Alumno con 50% asistencia, ¿qué dice el reglamento?"
contexto = buscar(query, indice, k=3)
respuesta = responder(query, contexto)
```

## Anti-patrones

- **Sin chunking**: documento entero como "chunk", contexto demasiado grande.
- **Chunks demasiado pequeños** (50 palabras): pierden contexto.
- **Top-k=20 siempre**: sobrecarga modelo.
- **Sin instrucción "no inventes"**: modelo alucina cuando no hay info.
