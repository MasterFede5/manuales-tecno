# 8.4 Embeddings y búsqueda semántica básica

Embeddings convierten texto a números. Búsqueda semántica encuentra textos similares por significado, no por palabras exactas. Esta lección programa ambos.

::concepto{titulo="Embeddings y búsqueda semántica"}
**Embedding**: vector de N dimensiones (1024-3072) que representa el "significado" de un texto. Textos similares → vectores cercanos.

**Búsqueda semántica**: dado una query, busca los N textos más similares a ella en tu corpus comparando distancia entre embeddings.

```python
# Pseudocódigo
embedding_query = embed("¿cómo apelar calificación?")
embedding_chunks = embed(reglamento_chunks)
top_3 = busqueda_similitud(embedding_query, embedding_chunks)
```

Usado en RAG, recomendación, deduplicación, clustering de texto.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Búsqueda por keywords ("apelación") falla si el reglamento dice "impugnación". Búsqueda semántica entiende: ambos hablan de lo mismo. Para tu Asistente, es la diferencia entre "no encontré nada" y "encontré sección relevante".
::/interioriza

## Generar embeddings con OpenAI

```python
from openai import OpenAI
client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))

def embed(texto: str) -> list:
    response = client.embeddings.create(
        model="text-embedding-3-small",
        input=texto
    )
    return response.data[0].embedding

vector = embed("¿cómo apelar calificación?")
print(len(vector))  # 1536 dimensiones
```

Costo: $0.02 por 1M tokens (centavos por reglamento entero).

## Búsqueda por similitud (coseno)

```python
import numpy as np

def similitud_coseno(v1, v2):
    return np.dot(v1, v2) / (np.linalg.norm(v1) * np.linalg.norm(v2))

# Documentos
docs = ["El plazo para apelar es 5 días.",
        "La asistencia mínima es 80%.",
        "La calificación final pondera tareas y exámenes."]

# Embeddings de cada doc
emb_docs = [embed(d) for d in docs]

# Query
query = "¿cuántos días para impugnar?"
emb_query = embed(query)

# Buscar top
sims = [similitud_coseno(emb_query, ed) for ed in emb_docs]
mejor = np.argmax(sims)
print(f"Mejor match: '{docs[mejor]}' (sim={sims[mejor]:.3f})")
```

## Vector stores

Para corpus grandes, no calcules similitud cada vez. Usa **vector store**:

- **ChromaDB**: simple, embebido. Para prototipos.
- **Pinecone, Weaviate**: cloud, escalable.
- **pgvector**: extensión Postgres.
- **FAISS** (Facebook): local, muy rápido.

```python
import chromadb

client = chromadb.Client()
collection = client.create_collection("reglamento")

# Indexar
collection.add(
    documents=["chunk 1", "chunk 2", ...],
    embeddings=[vec1, vec2, ...],
    ids=["id1", "id2", ...]
)

# Buscar
results = collection.query(
    query_embeddings=[emb_query],
    n_results=3
)
```

## Modelos de embeddings populares

| Modelo | Dimensiones | Idiomas | Costo /M |
|---|---|---|---|
| OpenAI `text-embedding-3-small` | 1536 | multi | $0.02 |
| OpenAI `text-embedding-3-large` | 3072 | multi | $0.13 |
| Voyage `voyage-3` | 1024 | multi | $0.06 |
| BGE-m3 (open) | 1024 | multilingüe excelente | $0 (local) |

Para español académico, BGE-m3 o Voyage suelen ser top.

## Anti-patrones

- **Embeddings de proveedor 1, búsqueda con proveedor 2**. No funciona.
- **Indexar sin chunking**. Documento entero en un vector pierde detalle.
- **Sin vector store con corpus grande**. Lento.
- **Embedder sin probar tu idioma**. Algunos son malos en español.
