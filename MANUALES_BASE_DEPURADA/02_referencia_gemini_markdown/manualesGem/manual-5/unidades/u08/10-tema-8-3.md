# 8.3 Streaming y tool use (introducción)

Dos features que elevan tu chatbot: **streaming** (texto que aparece progresivamente, mejor UX) y **tool use** (modelo invoca funciones Python tuyas).

::concepto{titulo="Streaming y tool use"}
**Streaming**: en lugar de esperar respuesta completa, recibes tokens conforme se generan. UX tipo ChatGPT.

**Tool use** (function calling): defines funciones Python; el modelo decide cuándo invocarlas con qué argumentos. Te conecta con el mundo real (BD, APIs, archivos).

Ambos son features modernas (2024-25) en Anthropic, OpenAI, Google.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
**Streaming** = UX profesional. Sin él, "asistente" parece "sistema lento". **Tool use** = bot puede actuar, no solo conversar. Ambas son piezas estándar de cualquier app IA moderna.
::/interioriza

## Streaming en código

```python
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ.get("ANTHROPIC_API_KEY"))

with client.messages.stream(
    model="claude-haiku-4-5",
    max_tokens=500,
    messages=[{"role": "user", "content": "Cuenta una historia corta"}]
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)

print()  # Newline al final
```

Resultado: texto aparece palabra por palabra. UX percibida como "mucho más rápida" aunque tarde lo mismo total.

## Tool use básico

Defines funciones que el modelo puede invocar:

```python
import json

def get_grade_info(student_id: int) -> dict:
    """Consulta calificaciones del alumno desde BD interna."""
    # En real: query a BD
    return {"id": student_id, "cal_final": 7.2, "asistencia": 0.85}

tools = [{
    "name": "get_grade_info",
    "description": "Consulta info de calificaciones de un alumno por ID",
    "input_schema": {
        "type": "object",
        "properties": {
            "student_id": {"type": "integer", "description": "ID del alumno"}
        },
        "required": ["student_id"]
    }
}]

# Loop con tool use
messages = [{"role": "user", "content": "¿Cómo va el alumno 124?"}]
while True:
    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1000,
        tools=tools,
        messages=messages
    )
    
    if response.stop_reason == "tool_use":
        # Modelo quiere invocar función
        for block in response.content:
            if block.type == "tool_use":
                result = get_grade_info(**block.input)
                
                messages.append({"role": "assistant", "content": response.content})
                messages.append({
                    "role": "user",
                    "content": [{
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": json.dumps(result)
                    }]
                })
    else:
        # Respuesta final
        print(response.content[0].text)
        break
```

## Caso aplicado al Asistente Predictor

```python
def predecir_alumno(features: dict) -> dict:
    """Usa tu modelo de ML entrenado en U5/U7."""
    import joblib
    model = joblib.load("modelo_predictor_v1.pkl")
    pred = model.predict([list(features.values())])[0]
    return {"prediccion": float(pred), "riesgo": "alto" if pred < 6 else "bajo"}

# Tool definition
tools = [{
    "name": "predecir_alumno",
    "description": "Predice calificación final de alumno basándose en features",
    "input_schema": {...}
}]

# Usuario: "Predice para Andrea: estudia 4 horas, asiste 80%, cal anterior 6.5"
# Claude → invoca predecir_alumno({"horas_estudio": 4, ...})
# Recibe predicción → la explica en lenguaje natural
```

Tu predictor + Claude = sistema que predice y explica.

## Anti-patrones

- **Tool use con permisos amplios**. Si tu tool puede modificar BD, requiere validación humana.
- **Sin streaming en UI lentas**. Usuario percibe lentitud aunque sea rápido.
- **Loop infinito sin tope**. Modelo invoca tool sin parar. Pone `max_iterations`.
