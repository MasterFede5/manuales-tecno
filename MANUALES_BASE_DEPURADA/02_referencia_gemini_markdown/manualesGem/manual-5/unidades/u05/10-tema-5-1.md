# 5.1 ¿Qué es ML? Supervisado vs no supervisado

ML no es magia. Es **estadística aplicada con muchos datos** para encontrar patrones predictivos. Esta lección te quita el misterio.

::concepto{titulo="Machine Learning"}
**Machine Learning (ML)** es la disciplina de construir programas que **aprenden de datos** en lugar de seguir reglas explícitamente programadas. Tres familias principales:

1. **Supervisado**: tienes ejemplos con respuesta correcta (`X` features → `y` etiqueta). El modelo aprende a predecir `y` dado `X`.
   - **Regresión**: `y` es número (cal_final).
   - **Clasificación**: `y` es categoría (aprobado / reprobado).

2. **No supervisado**: no hay etiqueta. Buscas estructuras (clusters, anomalías).

3. **Refuerzo**: aprendes por interacción con entorno. Más complejo, no en este manual.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
ML predice cosas que serían imposibles con reglas if/else manuales. Tu cerebro aplica ML constantemente: aprendiste a reconocer caras viendo miles, no programando "ojos a 6 cm de distancia". Tu modelo hará lo mismo con calificaciones.
::/interioriza

## Supervisado: el patrón canónico

```
Datos históricos:
  Alumno 1: horas=5, asis=0.9, cal_ant=8 → cal_final=8.2
  Alumno 2: horas=2, asis=0.6, cal_ant=5 → cal_final=4.5
  ...
  Alumno 480: ...

Modelo entrenado con los 480 ejemplos.

Para alumno nuevo:
  horas=4, asis=0.85, cal_ant=7
Modelo predice → cal_final ≈ 7.1
```

## Tipos de tareas

| Tarea | Ejemplo case escolar | Modelo típico |
|---|---|---|
| Regresión | Predecir cal_final exacta | Linear, Random Forest |
| Clasificación binaria | ¿Aprobará? | Logistic, Decision Tree |
| Clasificación multi-clase | ¿Excelente/Bueno/Suficiente/Reprobado? | Decision Tree, MLP |
| Clustering | Agrupar perfiles similares | K-Means (U6) |
| Detección anomalías | Alumnos atípicos | Isolation Forest |

## Vocabulario fundamental

- **Feature** (X): variable de entrada (horas, asistencia).
- **Label** / Target (y): lo que quieres predecir (cal_final).
- **Sample** / Instancia: una fila (un alumno).
- **Dataset**: tabla completa.
- **Train**: entrenar modelo con datos.
- **Predict**: usar modelo para hacer predicción nueva.
- **Score**: medir qué tan bien predice.

## Anti-patrones

- **Llamar "IA" a cualquier if/else**. ML aprende, reglas duras no.
- **Aplicar ML donde una regla simple basta**. Si "horas>3 → aprueba" funciona, no necesitas ML.
- **Confundir correlación 0.9 con predicción perfecta**. r=0.9 no es 100%.
- **Predecir sin contexto histórico**. ML extrapola; nuevos contextos pueden romper.
