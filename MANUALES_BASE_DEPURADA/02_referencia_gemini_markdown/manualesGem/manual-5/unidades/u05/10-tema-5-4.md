# 5.4 Métricas: accuracy, precision, recall, F1, RMSE

Una métrica mal elegida te hace creer que tu modelo es bueno cuando es malo. Esta lección te da las 8 métricas indispensables.

::concepto{titulo="Métricas de evaluación"}
**Para clasificación:**
- **Accuracy**: % de predicciones correctas. Engañosa con clases desbalanceadas.
- **Precision**: de los que predije positivos, ¿cuántos lo eran? Importa cuando false positive es caro.
- **Recall**: de los positivos reales, ¿cuántos atrapé? Importa cuando false negative es caro.
- **F1**: balance armónico entre precision y recall.
- **AUC-ROC**: área bajo curva, robusto a desbalance.

**Para regresión:**
- **MAE**: error absoluto medio. Misma unidad que y.
- **MSE**: error cuadrático medio. Penaliza errores grandes.
- **RMSE**: raíz de MSE. Misma unidad que y.
- **R²**: % de varianza explicada (0 a 1).
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
"Mi modelo tiene 95% accuracy" suena genial. Pero si el 95% de tus alumnos aprueban naturalmente, predecir "todos aprueban" da 95% accuracy sin aprender nada. **La métrica te dice si el modelo es realmente útil**.
::/interioriza

## Matriz de confusión (clasificación binaria)

```
              Predicho
              Aprueba    Reprueba
Real Aprueba  [TP=80]    [FN=5]      ← Recall = TP/(TP+FN) = 80/85 = 94%
Real Reprueba [FP=10]    [TN=5]      ← Precision = TP/(TP+FP) = 80/90 = 89%

Accuracy = (TP+TN)/total = 85/100 = 85%
F1 = 2 × P × R / (P + R)
```

## Cuándo cada métrica

| Caso | Métrica clave |
|---|---|
| Diagnóstico de cáncer (false neg = muerte) | Recall |
| Filtro spam (false pos = bloquear correo importante) | Precision |
| Datos balanceados general | Accuracy / F1 |
| Datos desbalanceados | F1, AUC |
| Predecir cal_final (regresión) | RMSE, R² |
| Predecir tiempo en horas | MAE (más interpretable) |

## Código

```python
from sklearn.metrics import (
    accuracy_score, precision_score, recall_score, f1_score,
    mean_absolute_error, mean_squared_error, r2_score,
    confusion_matrix, classification_report
)
import numpy as np

# Clasificación
y_pred_clf = clf.predict(X_test)
print(f"Accuracy: {accuracy_score(y_test, y_pred_clf):.3f}")
print(f"F1: {f1_score(y_test, y_pred_clf):.3f}")
print(classification_report(y_test, y_pred_clf))
print(confusion_matrix(y_test, y_pred_clf))

# Regresión
y_pred_reg = reg.predict(X_test)
print(f"MAE: {mean_absolute_error(y_test, y_pred_reg):.3f}")
print(f"RMSE: {np.sqrt(mean_squared_error(y_test, y_pred_reg)):.3f}")
print(f"R²: {r2_score(y_test, y_pred_reg):.3f}")
```

## Para el case escolar

Predecir cal_final (regresión):
- **RMSE = 0.85**: en promedio el modelo se equivoca por 0.85 puntos.
- **R² = 0.62**: explica 62% de la varianza.

Predecir aprobado (clasificación binaria, ~80% naturalmente aprueban):
- **Accuracy = 88%**: si modelo predice "todos aprueban" → 80%. 88% mejora poco.
- **F1 = 0.65** para clase "reprueba": ese es el valor real (es la clase difícil).

## Anti-patrones

- **Reportar solo accuracy**. Oculta debilidades en clases minoritarias.
- **Comparar entre datasets distintos**. RMSE de cal vs RMSE de salarios no son comparables.
- **Optimizar métrica equivocada**. Si tu negocio depende de detectar reprobados (recall alto en "reprueba"), no optimices accuracy.
