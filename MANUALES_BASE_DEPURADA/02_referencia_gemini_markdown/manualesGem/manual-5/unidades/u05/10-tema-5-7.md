# 5.7 Árboles de decisión

Un árbol pregunta "¿asistencia ≥ 0.7?", y según respuesta pregunta otra cosa. Al final, **decide**. Es el modelo más cercano a cómo razonan humanos.

::concepto{titulo="Árbol de decisión"}
Un **árbol de decisión** clasifica o predice mediante una serie de **preguntas binarias** sobre las features, formando una estructura de árbol. El algoritmo busca la pregunta que mejor divide los datos en cada nivel.

```python
from sklearn.tree import DecisionTreeClassifier
from sklearn.tree import plot_tree

clf = DecisionTreeClassifier(max_depth=4, random_state=42)
clf.fit(X_train, y_train)

# Visualizar
import matplotlib.pyplot as plt
plt.figure(figsize=(20, 10))
plot_tree(clf, feature_names=X.columns, class_names=["Reprobado", "Aprobado"], filled=True)
plt.show()
```

Funcionan tanto para clasificación (`DecisionTreeClassifier`) como regresión (`DecisionTreeRegressor`).
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Un árbol pequeño es **lo más interpretable que existe**. Puedes mostrarlo a stakeholders no técnicos y entender por qué el modelo decide "este alumno está en riesgo". Esa transparencia importa en educación, salud, finanzas.
::/interioriza

## Cómo funciona

1. Toma todos los datos.
2. Encuentra la **mejor pregunta** que divide los datos (ej.: "asistencia<0.7?").
3. Para cada respuesta (sí/no), repite el proceso.
4. Para cuando: hojas son puras o se alcanza max_depth.

## Aplicado al case

```python
from sklearn.tree import DecisionTreeRegressor

X = df[["horas_estudio", "asistencia", "cal_anterior"]]
y = df["cal_final"]

reg = DecisionTreeRegressor(max_depth=4, random_state=42)
reg.fit(X_train, y_train)

# Importancia de features
for col, imp in zip(X.columns, reg.feature_importances_):
    print(f"{col}: {imp:.3f}")
```

Output típico:
```
horas_estudio: 0.18
asistencia:    0.32
cal_anterior:  0.50
```

cal_anterior contribuye 50 % de las decisiones del árbol.

## Hiperparámetros importantes

| Parámetro | Qué controla |
|---|---|
| `max_depth` | profundidad máxima (default ilimitada → sobreajuste) |
| `min_samples_split` | mínimo de muestras para dividir un nodo |
| `min_samples_leaf` | mínimo en hoja |
| `max_features` | cuántas features consideran en cada split |

`max_depth=3-5` suele dar balance bueno entre simplicidad y poder.

## Reglas extraídas

Un árbol entrenado puede traducirse a if/else:

```
SI cal_anterior >= 6.5:
    SI asistencia >= 0.85:
        cal_final ≈ 8.5
    ELSE:
        cal_final ≈ 7.0
ELSE:
    SI horas_estudio >= 4:
        cal_final ≈ 6.5
    ELSE:
        cal_final ≈ 5.0
```

Esto se imprime con `export_text(reg, feature_names=X.columns.tolist())`.

## Random Forest (mención)

Un solo árbol sobreajusta fácil. **Random Forest** combina cientos de árboles diversos:

```python
from sklearn.ensemble import RandomForestRegressor
rf = RandomForestRegressor(n_estimators=100, max_depth=10, random_state=42)
```

Más preciso pero menos interpretable. En producción casi siempre se prefiere a un solo árbol.

## Anti-patrones

- **Árbol muy profundo sin max_depth**. Memoriza datos, sobreajusta.
- **Olvidar plot_tree**. La visualización es la mayor ventaja, úsala.
- **Cambiar features sin re-entrenar**. Árbol guarda decisiones específicas.
- **Confiar en feature_importance ciegamente**. Mide qué usa el árbol, no causalidad real.
