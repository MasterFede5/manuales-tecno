# 2.2 Pandas: Series y DataFrame

Pandas es **la** librería para manejar datos tabulares en Python. Si data scientist es tu carrera deseada, pandas será el 50 % de tu día a día.

::concepto{titulo="Pandas: Series y DataFrame"}
**Pandas** es la librería estándar para datos estructurados (tablas) en Python. Construida sobre NumPy, expone dos estructuras principales:

- **Series**: array 1D con **índice** (similar a columna de Excel con etiquetas de fila).
- **DataFrame**: tabla 2D con filas (índice) y columnas (cada una una Series).

```python
import pandas as pd

# Series
calificaciones = pd.Series([8.5, 6.0, 9.5], index=["Andrea", "Carlos", "María"])

# DataFrame
df = pd.DataFrame({
    "nombre": ["Andrea", "Carlos", "María"],
    "calificacion": [8.5, 6.0, 9.5],
    "horas": [5, 4, 6]
})
```

Es **el lenguaje común** entre Excel, SQL y Python. Si dominas pandas, dominas la transición de hojas de cálculo a programación.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Si tu trabajo toca datos, pandas reemplaza Excel. Misma lógica, 100× más poder, ilimitadas filas, código reproducible. La curva de aprendizaje vale: una semana invertida acelera años de carrera.
::/interioriza

## Crear DataFrames

```python
import pandas as pd

# Desde dict
df = pd.DataFrame({
    "id": [124, 125, 126],
    "nombre": ["Andrea", "Carlos", "María"],
    "calificacion": [8.5, 6.0, 9.5]
})

# Desde lista de dicts
df = pd.DataFrame([
    {"id": 124, "nombre": "Andrea", "calificacion": 8.5},
    {"id": 125, "nombre": "Carlos", "calificacion": 6.0}
])

# Desde array NumPy
import numpy as np
arr = np.random.rand(5, 3)
df = pd.DataFrame(arr, columns=["A", "B", "C"])
```

## Inspección rápida

```python
df.head()         # primeras 5 filas
df.tail()         # últimas 5 filas
df.shape          # (filas, columnas)
df.columns        # lista de columnas
df.dtypes         # tipos de cada columna
df.info()         # resumen completo
df.describe()     # estadísticas por columna numérica
```

`df.info()` y `df.describe()` son tus dos primeros comandos al recibir cualquier dataset.

## Acceso a columnas

```python
# Una columna (Series)
df["calificacion"]
df.calificacion         # equivalente, solo si nombre es identificador válido

# Varias columnas
df[["nombre", "calificacion"]]
```

## Acceso a filas

```python
# Por posición (.iloc)
df.iloc[0]              # primera fila
df.iloc[0:3]            # primeras 3
df.iloc[0, 1]           # fila 0, columna 1

# Por etiqueta (.loc)
df.loc[0, "nombre"]     # fila índice 0, columna "nombre"
df.loc[0:2, "nombre":"calificacion"]
```

`.iloc` para posiciones (entero), `.loc` para etiquetas/nombres.

## Operaciones vectorizadas (heredadas de NumPy)

```python
df["calificacion"] * 10                    # vector entero
df["promedio_ponderado"] = (df["cal_anterior"] + df["cal_final"]) / 2  # nueva columna
df["aprobado"] = df["calificacion"] >= 6   # bool nueva
```

## Agregación y filtrado básicos

```python
# Estadísticas
df["calificacion"].mean()
df["calificacion"].std()
df["calificacion"].quantile(0.5)  # mediana

# Filtrado boolean
aprobados = df[df["calificacion"] >= 6]
top_alumnos = df[df["calificacion"] > 9]
```

## Métodos clave

| Método | Qué hace |
|---|---|
| `df.head(n)` | primeras n filas |
| `df.tail(n)` | últimas n filas |
| `df.sort_values("col")` | ordenar |
| `df.drop_duplicates()` | quitar duplicados |
| `df.dropna()` | quitar filas con NaN |
| `df.fillna(0)` | rellenar NaN con valor |
| `df["col"].astype(int)` | cambiar tipo |
| `df.rename(columns={"a":"b"})` | renombrar |
| `df.drop(columns=["c"])` | eliminar columna |
| `df["col"].apply(funcion)` | aplicar función a cada valor |
| `df["col"].value_counts()` | conteo por valor único |

## Crear / actualizar columnas

```python
# Crear nueva
df["en_riesgo"] = (df["asistencia"] < 0.7) | (df["calificacion"] < 6)

# Modificar existente
df["calificacion"] = df["calificacion"].clip(0, 10)  # acota entre 0 y 10
df["nombre"] = df["nombre"].str.upper()
```

## Anti-patrones

- **Iterar fila por fila con `for`.** Mata el sentido de pandas. Vectoriza con operaciones de columna.
- **Mezclar `.loc` con `.iloc`.** Confusión. Decide cuál usar y mantente consistente.
- **Modificar copy (chained assignment).** `df[df["x"]>5]["y"] = 0` es ambiguo. Usa `df.loc[df["x"]>5, "y"] = 0`.
- **`.apply` cuando hay vectorización.** `.apply` es lento; vectoriza si puedes.
- **`describe()` sin entender las columnas.** No todas las columnas tienen sentido para describir (categóricas, ids).
