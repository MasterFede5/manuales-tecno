# 2.3 Lectura: CSV, Excel, JSON

En U1 leíste CSV con 10 líneas de Python. Pandas lo hace en 1. Esta lección cubre los 3 formatos más comunes.

::concepto{titulo="Lectura con pandas"}
Pandas expone funciones `read_*` para cada formato. La más usada:

```python
import pandas as pd

# CSV
df = pd.read_csv("archivo.csv")

# Excel
df = pd.read_excel("archivo.xlsx", sheet_name="Hoja1")

# JSON
df = pd.read_json("archivo.json")
```

Cada función tiene **decenas de parámetros** para casos especiales (separador europeo, encoding, fechas, etc.).
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Tu data live en Excel, en CSV exportados de sistemas, en JSON de APIs. Tres líneas de pandas las pone todas en el mismo formato (DataFrame) listo para procesar. Es el "puerto de entrada" universal de tus datos.
::/interioriza

## CSV: la función estrella

```python
df = pd.read_csv(
    "dataset_escolar.csv",
    encoding="utf-8",          # acentos
    sep=",",                    # separador (puede ser ";" o "\t")
    decimal=".",                # punto decimal (puede ser ",")
    parse_dates=["fecha"],      # convierte automáticamente a datetime
    index_col="id",             # columna a usar como índice
    na_values=["N/A", "??"],   # marcar como NaN
    nrows=1000                  # limitar filas (debug)
)
```

99 % de los casos se resuelven con `pd.read_csv("archivo.csv")` sin parámetros. El otro 1 % requiere los flags arriba.

## Excel

```python
# Una hoja
df = pd.read_excel("calificaciones.xlsx", sheet_name="2026")

# Todas las hojas
todas = pd.read_excel("calificaciones.xlsx", sheet_name=None)
# todas es dict {nombre_hoja: DataFrame}

# Con header en línea 3
df = pd.read_excel("archivo.xlsx", header=2)
```

Requiere `openpyxl` instalado: `pip install openpyxl`.

## JSON

```python
# JSON simple (lista de objetos)
df = pd.read_json("alumnos.json")

# JSON anidado: necesitas json_normalize
import json
with open("nested.json", "r") as f:
    raw = json.load(f)
df = pd.json_normalize(raw["alumnos"])
```

JSON es más complicado porque varía mucho la estructura. CSV y Excel son más predecibles.

## Escribir (export)

```python
df.to_csv("salida.csv", index=False, encoding="utf-8")
df.to_excel("salida.xlsx", index=False, sheet_name="Resultado")
df.to_json("salida.json", orient="records", indent=2)
```

`index=False` evita guardar el índice como columna extra.

## Casos especiales frecuentes

### CSV con separador europeo

```python
df = pd.read_csv("archivo.csv", sep=";", decimal=",")
```

### CSV con encoding latin1 (Windows típico)

```python
df = pd.read_csv("archivo.csv", encoding="latin1")
```

### Múltiples archivos a un DataFrame

```python
import glob
archivos = glob.glob("data/*.csv")
df = pd.concat([pd.read_csv(f) for f in archivos], ignore_index=True)
```

## Anti-patrones

- **No verificar al leer.** Después de `pd.read_csv`, siempre `df.head()`, `df.shape`, `df.dtypes`.
- **Olvidar `index=False` al escribir.** Tu archivo de salida queda con columna "Unnamed: 0".
- **Cargar archivo enorme entero.** Para >1 GB, usa `chunksize=10000` y procesa por chunks.
- **Asumir tipos.** Pandas a veces infiere mal. Verifica `dtypes` y convierte explícito.
- **No documentar fuente.** Si tu CSV cambia, ¿de dónde lo bajaste? Anota en código.
