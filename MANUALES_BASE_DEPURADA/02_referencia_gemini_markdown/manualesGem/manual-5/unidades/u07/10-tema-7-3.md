# 7.3 Forward pass y backpropagation (intuitivo, sin derivadas)

Una red entrena en bucle: hace predicción (forward), mide error, ajusta pesos para reducir error (backward), repite. Esta lección te lo explica sin tocar derivadas.

::concepto{titulo="Forward pass, loss, backward pass"}
**Forward pass:** datos atraviesan la red de input a output. Resultado: predicción.

**Loss (función de pérdida):** mide qué tan equivocada está la predicción vs la real. Ejemplos:
- **MSE** (Mean Squared Error): regresión.
- **BCE** (Binary Cross-Entropy): clasificación binaria.
- **CrossEntropy**: multi-clase.

**Backward pass (backpropagation):** algoritmo que **distribuye** la responsabilidad del error entre todos los pesos, calculando cuánto cambiar cada uno.

**Optimizador (SGD, Adam):** usa los gradientes calculados en backward para actualizar pesos.

Loop:
```
para cada batch de datos:
    1. forward: pred = model(X)
    2. loss = criterion(pred, y_real)
    3. backward: loss.backward()  ← PyTorch hace toda la matemática
    4. optimizer.step()  ← ajusta pesos
    5. optimizer.zero_grad()  ← limpia gradientes
```

Repite muchas veces (epochs) hasta que loss baje.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Backpropagation es **el algoritmo que hizo posible deep learning**. Sin él, ajustar millones de parámetros sería imposible. Tú no lo programas a mano: PyTorch lo hace; pero entender la idea te permite diagnosticar entrenamientos que fallan.
::/interioriza

## Intuición sin derivadas

Imagina que tu red predice "calificación = 8.5" pero la real es 6.5. Error = 2.

Backprop pregunta: *"¿Qué pesos contribuyeron más a este error?"*. Cambia esos pesos un poquito en la dirección que **reduce** el error.

Repite con muchos ejemplos. La red aprende qué pesos funcionan.

## Loss functions

```python
import torch.nn as nn

# Regresión (predecir cal_final)
criterion = nn.MSELoss()      # Mean Squared Error

# Clasificación binaria (aprueba/reprueba)
criterion = nn.BCELoss()       # con sigmoid en output
criterion = nn.BCEWithLogitsLoss()  # incluye sigmoid (más estable numéricamente)

# Clasificación multi-clase (4 niveles)
criterion = nn.CrossEntropyLoss()  # incluye softmax
```

## Optimizadores

```python
import torch.optim as optim

# SGD (gradiente descendente clásico)
optim.SGD(model.parameters(), lr=0.01)

# Adam (más robusto, default moderno)
optim.Adam(model.parameters(), lr=0.001)

# AdamW (con weight decay, mejor para deep learning moderno)
optim.AdamW(model.parameters(), lr=0.001, weight_decay=0.01)
```

`lr` (learning rate) es el hiperparámetro más importante: muy alto → no converge; muy bajo → tarda eternidad. Default `0.001` con Adam funciona casi siempre para tabular.

## Lo que PyTorch hace por ti

PyTorch tiene **autograd**: calcula automáticamente las derivadas de la loss respecto a cada peso. Tú escribes:

```python
loss.backward()  # Una línea. Magia.
```

Y PyTorch llena `param.grad` con los gradientes correctos.

## Conceptos clave que vas a oír

- **Epoch**: una pasada por todo el dataset.
- **Batch**: subconjunto de datos procesados juntos (ej.: 32, 64).
- **Iteration**: una actualización de pesos (un batch).
- **Learning rate**: tamaño de paso para ajustar pesos.
- **Gradient**: dirección en que el peso debe cambiar.
- **Convergencia**: cuando la loss deja de bajar.

## Anti-patrones

- **Olvidar `optimizer.zero_grad()`**: gradientes se acumulan, ajusta mal.
- **`lr` demasiado alto**: loss explota.
- **No verificar loss durante entrenamiento**: te enteras al final que no aprendió.
- **Usar SGD sin momentum** para problemas modernos. Adam es mejor default.
