# 7.6 Diagnóstico: curva de aprendizaje

La curva de loss vs epoch te dice exactamente **qué pasa con tu entrenamiento**. Saber leerla es habilidad fundamental.

::concepto{titulo="Curva de aprendizaje"}
La **curva de aprendizaje** grafica la loss (train y test/validation) por epoch. Patrones canónicos:

1. **Buen ajuste**: ambas bajan y convergen a valor bajo similar.
2. **Underfitting**: ambas se quedan en valores altos. Modelo muy simple.
3. **Overfitting**: train sigue bajando, test sube. Modelo memoriza.
4. **Inestable**: oscila mucho. lr muy alto o batch muy pequeño.

Cada patrón tiene una solución específica.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Mirar el final del entrenamiento ("loss=0.5") no basta. La **forma** de la curva te dice si el modelo aprendió bien, mal, demasiado o de manera inestable. Es radiografía de tu entrenamiento.
::/interioriza

## Los 4 patrones

### 1. Buen ajuste (objetivo)
- Train y test bajan juntas.
- Convergen a valor bajo.
- Gap pequeño entre ambas.

**Acción:** entrenamiento exitoso.

### 2. Underfitting (modelo muy simple)
- Ambas bajan poco.
- Se estancan en valor alto.
- Sin gap.

**Acción:** modelo más grande, más epochs, mejores features.

### 3. Overfitting (modelo memoriza)
- Train baja a 0.
- Test baja un rato y luego sube.
- Gap creciente.

**Acción:** regularización (Dropout, L2), early stopping, más datos, modelo más simple.

### 4. Inestable (lr o batch mal)
- Oscila mucho.
- Loss explota o salta.

**Acción:** reducir lr, aumentar batch size.

## Early stopping

Para evitar overfitting, detener entrenamiento cuando test loss empieza a subir:

```python
mejor_loss_test = float('inf')
paciencia = 10
contador = 0

for epoch in range(epochs):
    # ... entrenar ...
    
    if loss_test.item() < mejor_loss_test:
        mejor_loss_test = loss_test.item()
        torch.save(model.state_dict(), "mejor_modelo.pth")
        contador = 0
    else:
        contador += 1
        if contador >= paciencia:
            print(f"Early stopping at epoch {epoch}")
            break
```

## Otras señales de problema

- **Loss = nan**: gradientes explotaron. Reduce lr.
- **Loss baja luego sube luego baja**: lr muy alto.
- **Loss inicial muy alta** (>1000): probablemente sin scaling.
- **Train loss = test loss exactamente igual siempre**: olvidaste shuffle o estás evaluando en mismo set.

## Anti-patrones

- **No graficar curva**: te quedas ciego al problema.
- **Pocos epochs**: no llegas a converger ni detectar overfit.
- **Sin paciencia en early stopping**: detiene por ruido.
