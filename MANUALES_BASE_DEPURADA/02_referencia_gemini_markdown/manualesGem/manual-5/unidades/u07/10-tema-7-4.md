# 7.4 PyTorch: estilo recetario (tensors, nn.Module, optimizer)

PyTorch tiene 3 piezas: **tensors** (datos), **nn.Module** (modelo), **optimizer** (entrenamiento). Esta lección te enseña la receta.

::concepto{titulo="PyTorch básico"}
**PyTorch** es la librería de deep learning más popular en research y growing en producción. Sus 3 piezas:

1. **Tensor**: array multi-dimensional similar a NumPy pero con soporte GPU y diferenciación automática.
2. **nn.Module**: clase base para modelos. Defines arquitectura.
3. **Optimizer**: ajusta pesos según gradientes.

```python
import torch
import torch.nn as nn
import torch.optim as optim
```
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
PyTorch reemplazó a TensorFlow en research por ser más Pythónico y intuitivo. Saber PyTorch = puerta al deep learning moderno (LLMs, visión, RL).
::/interioriza

## Receta 1: Tensor desde DataFrame

```python
import pandas as pd
import torch

df = pd.read_csv("dataset_escolar_limpio.csv")
X = df[["horas_estudio", "asistencia", "cal_anterior"]].values
y = df["cal_final"].values

# Convertir a tensor
X_tensor = torch.tensor(X, dtype=torch.float32)
y_tensor = torch.tensor(y, dtype=torch.float32).unsqueeze(1)  # [n, 1]

print(X_tensor.shape)  # torch.Size([n, 3])
```

`unsqueeze(1)` agrega dimensión: y de [n] pasa a [n, 1]. Importante para regresión.

## Receta 2: Modelo con nn.Sequential (simple)

```python
model = nn.Sequential(
    nn.Linear(3, 16),    # 3 features → 16 neuronas
    nn.ReLU(),
    nn.Linear(16, 8),
    nn.ReLU(),
    nn.Linear(8, 1)      # 1 salida (regresión)
)

print(model)
# Total parameters
total_params = sum(p.numel() for p in model.parameters())
print(f"Parámetros: {total_params}")
```

## Receta 3: Modelo con nn.Module class (flexible)

```python
class PredictorEscolar(nn.Module):
    def __init__(self, input_size=3, hidden_size=16, output_size=1):
        super().__init__()
        self.fc1 = nn.Linear(input_size, hidden_size)
        self.relu1 = nn.ReLU()
        self.fc2 = nn.Linear(hidden_size, output_size)
    
    def forward(self, x):
        x = self.fc1(x)
        x = self.relu1(x)
        x = self.fc2(x)
        return x

model = PredictorEscolar()
```

Para arquitecturas custom, esta forma da control.

## Receta 4: Loop de entrenamiento

```python
criterion = nn.MSELoss()
optimizer = optim.Adam(model.parameters(), lr=0.001)

epochs = 100
for epoch in range(epochs):
    # Forward
    pred = model(X_tensor)
    loss = criterion(pred, y_tensor)
    
    # Backward
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    
    if epoch % 10 == 0:
        print(f"Epoch {epoch}: Loss = {loss.item():.4f}")
```

## Receta 5: Predecir con modelo entrenado

```python
model.eval()  # modo evaluación (importante con Dropout/BatchNorm)
with torch.no_grad():  # no calcula gradientes (ahorra memoria)
    nuevo = torch.tensor([[5.0, 0.92, 7.5]], dtype=torch.float32)
    prediccion = model(nuevo)
    print(f"Predicción: {prediccion.item():.2f}")
```

## Receta 6: Guardar y cargar

```python
# Guardar
torch.save(model.state_dict(), "modelo_v1.pth")

# Cargar
model2 = PredictorEscolar()
model2.load_state_dict(torch.load("modelo_v1.pth"))
model2.eval()
```

## DataLoader para batches

Para datasets grandes, no procesas todo de una:

```python
from torch.utils.data import TensorDataset, DataLoader

dataset = TensorDataset(X_tensor, y_tensor)
loader = DataLoader(dataset, batch_size=32, shuffle=True)

for epoch in range(epochs):
    for X_batch, y_batch in loader:
        pred = model(X_batch)
        loss = criterion(pred, y_batch)
        # ...
```

## Anti-patrones

- **Olvidar `optimizer.zero_grad()`**: gradientes acumulados.
- **No usar `model.eval()` en evaluación**: Dropout/BatchNorm activos.
- **Pasar lista en lugar de tensor**: `model([5, 0.9, 7.5])` falla. Usa `torch.tensor([[5, 0.9, 7.5]])`.
- **Sin batches**: out of memory para datasets grandes.
