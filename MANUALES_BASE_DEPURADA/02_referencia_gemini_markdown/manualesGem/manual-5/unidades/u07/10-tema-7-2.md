# 7.2 Capas, activaciones (ReLU, Softmax)

Apilas neuronas en capas. Cada capa transforma los datos. **La elección de capas y activaciones es la "arquitectura" de tu red**.

::concepto{titulo="Capas y activaciones"}
Una **red neuronal** organiza neuronas en **capas**:

- **Input layer**: tantas neuronas como features (sin activación).
- **Hidden layers**: capas intermedias con activación ReLU (estándar moderno).
- **Output layer**: depende de la tarea:
  - Regresión: 1 neurona, sin activación.
  - Clasificación binaria: 1 neurona, sigmoid (o sin para BCEWithLogits).
  - Clasificación multi-clase: K neuronas, softmax (o sin para CrossEntropy).

**MLP** (Multi-Layer Perceptron) = red densamente conectada con capas ocultas.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
La arquitectura controla qué patrones la red puede aprender. Demasiado simple → no aprende; demasiado compleja → sobreajusta. Encontrar la arquitectura correcta es 50 % del trabajo.
::/interioriza

## Activaciones más usadas

```python
import torch.nn.functional as F

# ReLU: max(0, x). Default para hidden layers.
F.relu(x)

# Sigmoid: 0 a 1. Para output binario.
torch.sigmoid(x)

# Softmax: probabilidades sumando 1. Para multi-clase.
F.softmax(x, dim=1)

# Tanh: -1 a 1. Menos usado.
torch.tanh(x)
```

```

```

97 + 136 + 9 = 242 parámetros aprendibles.

Clasificación binaria (aprueba):
```
Input(4) → Linear(4→16) + ReLU → Linear(16→1) + Sigmoid
```

Clasificación multi-clase (Excelente/Bueno/Suficiente/Reprobado):
```
Input(4) → Linear(4→32) + ReLU → Linear(32→4) + Softmax
```

## Reglas heurísticas

- **2-3 capas ocultas** suelen bastar para problemas tabulares simples.
- **Capa oculta de 16-128 neuronas** para datasets pequeños.
- **Más capas / neuronas** para datos complejos (imágenes, NLP).
- **Más datos = puedes usar red más grande** sin sobreajustar.

## Capas especiales

- **BatchNorm**: normaliza dentro de mini-batch. Acelera entrenamiento.
- **Dropout**: apaga aleatoriamente neuronas durante train. Previene sobreajuste.
- **Conv2D**: para imágenes (no en este manual).
- **LSTM/Transformer**: para secuencias (no en este manual).

```python
nn.Sequential(
    nn.Linear(4, 16),
    nn.ReLU(),
    nn.BatchNorm1d(16),
    nn.Dropout(0.2),  # 20% probabilidad de apagar neurona
    nn.Linear(16, 1)
)
```

## Anti-patrones

- **Sigmoid en hidden layers**. Causa "vanishing gradient". Usa ReLU.
- **Output con activación que limita rango erróneamente**. Cal_final puede ser 0-10, sigmoid limita a 0-1.
- **Dropout en testing**. PyTorch lo desactiva automáticamente con `model.eval()`.
- **Demasiadas capas para poco data**. Overfitting garantizado.
