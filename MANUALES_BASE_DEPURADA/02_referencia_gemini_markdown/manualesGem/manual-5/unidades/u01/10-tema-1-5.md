# 1.5 Listas, tuplas, diccionarios

Una variable guarda **un valor**. Para guardar **muchos** (480 calificaciones, perfiles de alumnos), Python tiene 4 estructuras. Esta lección te las enseña.

::concepto{titulo="Estructuras de datos básicas"}
Las **estructuras de datos** son contenedores que guardan múltiples valores con reglas distintas:

- **Lista (`list`)** — secuencia ordenada y **mutable** (puedes cambiarla).
- **Tupla (`tuple`)** — secuencia ordenada e **inmutable** (fija).
- **Diccionario (`dict`)** — pares clave-valor desordenados (en Python 3.7+ mantienen orden de inserción).
- **Conjunto (`set`)** — colección de elementos únicos sin orden.

Cada una tiene caso ideal. Aprender las 4 te da el vocabulario para representar cualquier dato.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Pega imagina un cuaderno escolar: la columna de calificaciones es una **lista** (orden importa, puede cambiar). Una pareja "alumno-promedio" es una **tupla** (orden fijo). Un alumno con sus datos {nombre, edad, materias} es un **diccionario**. Las claves de las matrículas únicas son un **set**. Cada día programando vas a usar las 4.
::/interioriza

## Lista (la más común)

```python
# Crear
calificaciones = [8.5, 6.0, 9.5, 5.5, 7.8]
materias = ["Mate", "Física", "Química"]

# Acceder por índice (empieza en 0)
print(calificaciones[0])   # 8.5
print(calificaciones[-1])  # 7.8 (último)
print(materias[0:2])       # ["Mate", "Física"]

# Modificar
calificaciones[0] = 9.0
calificaciones.append(8.0)        # agrega al final
calificaciones.insert(0, 10.0)    # agrega al inicio
calificaciones.remove(5.5)         # elimina por valor

# Tamaño
print(len(calificaciones))

# Recorrer
for cal in calificaciones:
    print(cal)
```

## Tupla (cuando NO quieres cambios)

```python
# Crear
coord_alumno = (124, 8.5)  # (id, calificacion)

# Acceder
id_a, cal_a = coord_alumno
print(id_a, cal_a)

# Tuplas son inmutables → coord_alumno[0] = 200 da ERROR
```

Útil para datos que no deben modificarse: coordenadas, registros fijos, retornos múltiples de funciones.

## Diccionario (clave-valor)

```python
# Crear
alumno = {
    "id": 124,
    "nombre": "Andrea",
    "calificaciones": [8.5, 9.0, 7.5],
    "promedio": 8.33,
    "asistencia": 0.92
}

# Acceder
print(alumno["nombre"])  # "Andrea"
print(alumno.get("edad", "no registrada"))  # devuelve default si no existe

# Modificar
alumno["promedio"] = 8.5
alumno["edad"] = 17

# Recorrer
for clave, valor in alumno.items():
    print(clave, ":", valor)
```

Diccionarios son **fundamentales** en datos: cada fila de tu CSV se vuelve un dict en memoria.

## Set (elementos únicos)

```python
# Crear
materias_cursadas = {"Mate", "Física", "Química", "Mate"}  # duplicado se elimina
print(materias_cursadas)  # {"Mate", "Física", "Química"}

# Operaciones
inscritas = {"Mate", "Física", "Química"}
aprobadas = {"Mate", "Física"}

reprobadas = inscritas - aprobadas
print(reprobadas)  # {"Química"}
```

Útil para deduplicar y operaciones de conjuntos.

## Anidación (combinar estructuras)

```python
# Lista de diccionarios — la forma más común para tabular data
estudiantes = [
    {"id": 124, "nombre": "Andrea", "promedio": 8.5},
    {"id": 125, "nombre": "Carlos", "promedio": 7.0},
    {"id": 126, "nombre": "María", "promedio": 9.2}
]

# Recorrer
for e in estudiantes:
    print(e["nombre"], "-", e["promedio"])

# Filtrar (basics)
aprobados = []
for e in estudiantes:
    if e["promedio"] >= 6:
        aprobados.append(e)
```

Esta estructura "lista de diccionarios" es **el formato que tu CSV se vuelve cuando lo cargas**. Lo verás en U2 con Pandas DataFrame.

## List comprehension (sintaxis bonita)

```python
# Forma larga
nombres = []
for e in estudiantes:
    nombres.append(e["nombre"])

# Forma corta (list comprehension)
nombres = [e["nombre"] for e in estudiantes]

# Con filtro
nombres_aprobados = [e["nombre"] for e in estudiantes if e["promedio"] >= 6]
```

Útil cuando quieres transformar una lista en otra rápidamente.

## Anti-patrones

- **Confundir lista con set en orden importante.** Set no garantiza orden; si necesitas orden, usa lista.
- **Modificar dict mientras lo recorres.** Crea bugs. Itera sobre `list(d.items())` si necesitas modificar.
- **Acceso a clave inexistente sin `.get()`.** `d["clave_que_no_existe"]` da `KeyError`. Usa `d.get("clave", default)`.
- **Tuplas mutables esperadas.** Si vas a modificar, usa lista, no tupla.
- **Comprehension monstruosa.** Si tu list comprehension tiene 3 líneas, mejor escribe bucle for.
