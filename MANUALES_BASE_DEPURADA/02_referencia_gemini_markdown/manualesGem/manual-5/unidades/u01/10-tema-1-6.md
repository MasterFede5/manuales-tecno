# 1.6 Lectura y escritura de archivos (CSV, JSON)

Aquí cumples el Episodio 1: tu programa **lee el CSV de coordinación** y muestra los datos. Esta lección te enseña los dos formatos más comunes en datos: CSV y JSON.

::concepto{titulo="Archivos CSV y JSON"}
**CSV** (*Comma-Separated Values*) es un formato de texto plano donde cada línea es una fila y los valores están separados por comas. Es el formato más común para datos tabulares.

```
id,nombre,calificacion
124,Andrea,8.5
125,Carlos,7.0
```

**JSON** (*JavaScript Object Notation*) es un formato de texto que representa estructuras anidadas (dict de Python).

```json
{
  "alumnos": [
    {"id": 124, "nombre": "Andrea", "calificacion": 8.5},
    {"id": 125, "nombre": "Carlos", "calificacion": 7.0}
  ]
}
```

Python lee/escribe ambos con módulos estándar (`csv`, `json`).
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Tus datos NO viven en variables Python: viven en archivos. Excel, BD, web, sensores — todo termina en CSV o JSON al exportar. Saber leerlos es la **puerta de entrada** al mundo real de los datos.
::/interioriza

## Leer CSV con módulo `csv`

```python
import csv

# Asume archivo "calificaciones.csv" con header
with open("calificaciones.csv", "r", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for fila in reader:
        print(fila)
        # {'id': '124', 'nombre': 'Andrea', 'calificacion': '8.5'}
```

**Notas:**
- `with open(...) as f:` abre y cierra automáticamente el archivo.
- `encoding="utf-8"` evita problemas con acentos.
- `DictReader` lee cada fila como diccionario usando la primera línea como claves.
- Los valores siempre vienen como **strings**; conviertes con `float()`, `int()` cuando corresponda.

## Leer CSV y procesar

```python
import csv

calificaciones = []
with open("calificaciones.csv", "r", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for fila in reader:
        cal = float(fila["calificacion"])
        calificaciones.append(cal)

promedio = sum(calificaciones) / len(calificaciones)
print(f"Promedio general: {promedio:.2f}")
print(f"Total estudiantes: {len(calificaciones)}")
```

## Escribir CSV

```python
import csv

datos = [
    {"id": 124, "nombre": "Andrea", "promedio": 8.5},
    {"id": 125, "nombre": "Carlos", "promedio": 7.0}
]

with open("salida.csv", "w", encoding="utf-8", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=["id", "nombre", "promedio"])
    writer.writeheader()
    writer.writerows(datos)
```

`newline=""` evita líneas en blanco extras (problema típico Windows).

## Leer y escribir JSON

```python
import json

# Leer
with open("alumnos.json", "r", encoding="utf-8") as f:
    datos = json.load(f)
print(datos["alumnos"][0]["nombre"])

# Escribir
with open("salida.json", "w", encoding="utf-8") as f:
    json.dump(datos, f, indent=2, ensure_ascii=False)
```

`indent=2` formatea el JSON legible. `ensure_ascii=False` preserva acentos.

## Caso aplicado: el CSV escolar del Episodio 1

```python
import csv

# Cargar
estudiantes = []
with open("dataset_escolar.csv", "r", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for fila in reader:
        estudiantes.append({
            "id": int(fila["id"]),
            "horas_estudio": float(fila["horas_estudio"]),
            "asistencia": float(fila["asistencia"]),
            "cal_anterior": float(fila["cal_anterior"]),
            "cal_final": float(fila["cal_final"])
        })

# Reporte
print(f"Total: {len(estudiantes)} estudiantes")
print(f"Primer alumno: {estudiantes[0]}")

aprobados = [e for e in estudiantes if e["cal_final"] >= 6]
print(f"Aprobados: {len(aprobados)} ({len(aprobados)/len(estudiantes)*100:.1f}%)")
```

¡Episodio 1 cumplido! En U2 esto se vuelve 3 líneas con Pandas, pero entender el flujo manualmente vale.

## Excel (.xlsx) y otros formatos

Para Excel necesitas `openpyxl` o `pandas` (U2). Para PDFs, `pypdf`. Para imágenes, `Pillow`. CSV y JSON cubren 80 % de casos en data science.

## Anti-patrones

- **No usar `with`.** `f = open(...)` sin cerrar puede corromper archivos. Usa siempre `with`.
- **Olvidar encoding.** Acentos rotos. Pon `encoding="utf-8"` siempre.
- **Hardcodear paths absolutos.** `"C:/Users/yo/archivo.csv"` solo funciona en tu PC. Usa `pathlib` o paths relativos.
- **No validar existencia.** `open("no_existe.csv")` da `FileNotFoundError`. Verifica con `os.path.exists()`.
- **Procesar todo el archivo en RAM.** Para archivos >1 GB, usa `pandas.read_csv(chunksize=)` o procesa línea a línea.
