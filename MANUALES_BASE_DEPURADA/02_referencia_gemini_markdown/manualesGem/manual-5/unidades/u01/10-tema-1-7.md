# 1.7 Errores y excepciones básicas

Tu programa cargará 480 filas. La fila 327 tiene "ABC" en vez de un número. Sin manejo de errores, tu programa **muere** ahí. Con manejo, sigue procesando las 479 buenas y reporta la mala.

::concepto{titulo="Excepciones y manejo de errores"}
Una **excepción** es un evento de error en tiempo de ejecución que **interrumpe** el flujo normal del programa. Python trae docenas pre-definidas: `FileNotFoundError`, `ValueError`, `KeyError`, `TypeError`, `ZeroDivisionError`, `IndexError`, etc.

El bloque `try/except` permite **capturar** excepciones y manejarlas en lugar de que el programa muera:

```python
try:
    # código que puede fallar
    cal = float(input_usuario)
except ValueError:
    # qué hacer si falla
    print("Valor inválido")
```

Sin este patrón, tu programa es frágil; con él, es robusto.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Cuando un cajero automático no logra leer tu tarjeta, no se incendia: dice "tarjeta no reconocida, vuelve a intentar". Eso es manejo de errores. Tu programa Python con datos del mundo real necesita exactamente esa elegancia: errores son inevitables; manejarlos es lo que hace al programa profesional.
::/interioriza

## Las 6 excepciones que más vas a ver

| Excepción | Cuándo ocurre | Ejemplo |
|---|---|---|
| `FileNotFoundError` | Archivo no existe | `open("xx.csv")` |
| `ValueError` | Tipo correcto pero valor inválido | `float("ABC")` |
| `KeyError` | Clave de dict no existe | `d["clave_inexistente"]` |
| `TypeError` | Operación con tipo incorrecto | `5 + "hola"` |
| `ZeroDivisionError` | División entre 0 | `5 / 0` |
| `IndexError` | Índice fuera de rango | `lista[100]` con lista de 5 |

## Anatomía de try/except

```python
try:
    cal = float(input("Calificación: "))
    if cal < 0 or cal > 10:
        raise ValueError("Calificación fuera de rango 0-10")
    print("Calificación válida:", cal)
except ValueError as e:
    print(f"Error: {e}")
except Exception as e:
    print(f"Error inesperado: {e}")
finally:
    print("Programa termina")
```

**Bloques:**
- `try`: código que puede fallar.
- `except`: qué hacer si ocurre.
- `else`: opcional; ejecuta solo si NO hubo excepción.
- `finally`: opcional; ejecuta SIEMPRE (haya error o no). Útil para cerrar archivos.

## Aplicado al caso escolar

```python
import csv

def cargar_calificaciones(path):
    estudiantes = []
    errores = []
    
    try:
        with open(path, "r", encoding="utf-8") as f:
            reader = csv.DictReader(f)
            for i, fila in enumerate(reader, start=2):  # 2 porque línea 1 es header
                try:
                    estudiantes.append({
                        "id": int(fila["id"]),
                        "calificacion": float(fila["calificacion"])
                    })
                except (ValueError, KeyError) as e:
                    errores.append((i, str(e), fila))
    except FileNotFoundError:
        print(f"Archivo no encontrado: {path}")
        return [], []
    
    return estudiantes, errores

estudiantes, errores = cargar_calificaciones("dataset_escolar.csv")
print(f"Cargados: {len(estudiantes)}")
print(f"Errores: {len(errores)}")
for linea, err, fila in errores[:5]:
    print(f"  Línea {linea}: {err}")
```

Tu programa carga lo bueno, reporta lo malo, no muere.

## Reglas de oro

1. **No captures `Exception` genérico** sin razón. Captura específicas para saber qué pasó.
2. **No silencies errores.** `except: pass` es trampa: oculta bugs.
3. **Loguea o reporta.** Un error sin reportar es un error que vuelve.
4. **Usa `finally` para cleanup.** Cerrar archivos, conexiones, etc.
5. **Levanta tus propios errores** con `raise ValueError("mensaje")` cuando detectas situación inválida.

## Anti-patrones

- **`try` enorme cubriendo todo el programa.** Difícil saber qué falló. Usa try/except por bloque pequeño.
- **Try/except sin acción real.** Si no manejas, deja que el error caiga y arregla el bug.
- **Bare `except:`** sin tipo. Captura todo, incluso `KeyboardInterrupt`. Mal.
- **Usar excepciones para control de flujo normal.** Si vas a iterar 1000 veces y el except es esperado, replantea.
