# 3.4 Refinamiento iterativo: cómo mejorar un prompt sin empezar de cero

La primera versión de un prompt **rara vez** es la final. Los profesionales **iteran**: prueba → observa → ajusta → re-prueba. La diferencia entre un usuario casual y uno potente está en cómo iteran. Esta sección te enseña el ciclo completo.

::concepto{titulo="Loop de refinamiento de prompt"}
Un prompt útil rara vez sale a la primera. El proceso profesional tiene 5 pasos cíclicos:

1. **Escribir** prompt v1 con anatomía completa.
2. **Ejecutar** y observar la respuesta.
3. **Evaluar** contra criterios definidos.
4. **Refinar** identificando la causa raíz del problema.
5. **Re-ejecutar** y comparar versus v1.

Si la nueva versión es **claramente mejor**, guarda el prompt. Si no, vuelve al paso 4.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Cada prompt que iteras y guardas es un **activo reutilizable**. Tras 3 meses tendrás un banco personal de 50+ prompts probados que producen resultados consistentes. Esa biblioteca es lo que separa al usuario casual del profesional. Y construirla requiere disciplina de iteración, no genio.
::/interioriza

### Paso 1 — Escribe v1 con anatomía completa

Aplica las 6 capas (R-T-C-R-F-E) que viste en 3.1. **NO** improvises sin estructura.

### Paso 2 — Ejecuta y observa

Pega el prompt en tu plataforma. Anota:
- Tiempo de respuesta.
- Caracteres / tokens generados.
- Si necesitó retries.
- Si el formato fue correcto.

### Paso 3 — Evalúa con criterios

**No te limites a "me gustó / no me gustó".** Define criterios objetivos:

| Criterio | Cómo medirlo |
|---|---|
| **Precisión factual** | ¿Datos correctos? Verifica con segunda fuente. |
| **Formato cumplido** | ¿Sigue la estructura pedida? |
| **Tono adecuado** | ¿Coincide con el rol asignado? |
| **Completitud** | ¿Cubre todo lo pedido sin omitir? |
| **Sin alucinaciones** | ¿Inventa citas/datos que no existen? |
| **Largo correcto** | ¿Respeta restricción de palabras? |

### Paso 4 — Refina identificando causa raíz

| Síntoma | Causa probable | Ajuste |
|---|---|---|
| Respuesta vaga | Falta contexto | Añade contexto específico |
| Tono inadecuado | Rol genérico | Refina rol con detalle |
| Inventa datos | Pregunta fuera de su entrenamiento | Activa búsqueda web o limita scope |
| Demasiado largo | Sin restricción de longitud | Añade "máximo X palabras" |
| Formato incorrecto | Output no estructurado | Define formato JSON/markdown explícito |
| Repite obviedades | Asume que no sabes | Especifica nivel: "asume que conozco cálculo I" |
| Fría | Falta tone en role | Añade adjetivos de personalidad |

### Paso 5 — Re-ejecuta y compara

**No reescribas desde cero.** Toma v1 y modifica solo la capa que identificaste como causa raíz. Compara los dos outputs lado a lado.

> **Trampa común:** "tirar todo y volver a empezar" en vez de cirugía precisa. Cada iteración debe cambiar **una variable** para entender qué funciona.

## Meta-prompts: pídele al modelo que mejore tu prompt

Truco profesional: **el propio modelo puede ayudarte a iterar**.

```
Mira el prompt que te di abajo y la respuesta que generaste.

Prompt:
[tu prompt v1]

Respuesta:
[output que recibiste]

Crítica el prompt: ¿qué le falta para que la respuesta sea más precisa,
más estructurada y más útil para mi caso (que soy estudiante de
bachillerato)? Sugiere una v2 mejorada y explica los cambios.
```

> Es **prompt sobre prompt**. Útil cuando tú mismo no detectas qué le falta.

## Sistema de versiones de prompts

Mantén tus prompts como **artefactos versionados** (Notion, Apple Notes, Excel):

```
PROMPT: Tutor Bernoulli
v1 (28-feb): rol+tarea+contexto. Output: bueno pero genérico.
v2 (1-mar): añadí formato 5-pasos. Output: estructura clara.
v3 (3-mar): añadí ejemplo F1. Output: aplicado al case study.
v4 (10-mar): añadí "asume que conozco F=ma". Output: nivel correcto.
✓ ACTUAL: v4 (PRODUCCIÓN).
```

> **Ventaja.** Si en futuro el modelo cambia y v4 deja de funcionar, regresas a v3 sin reinventar.

## Ejemplo real de refinamiento

```
v1: "Explícame el principio de Bernoulli."

→ Output: definición Wikipedia genérica, sin ejemplo numérico.

v2: "Eres mi tutor de física. Explícame Bernoulli con un ejemplo del coche F1."

→ Output: explicación + ejemplo F1, pero sin pregunta de comprobación.

v3: [Prompt completo con 6 capas como en 3.1]

→ Output: definición + analogía + fórmula + ejemplo F1 numérico + pregunta. ✓
```

3 iteraciones, 5 minutos. Output final reutilizable cada vez que estudies física.
