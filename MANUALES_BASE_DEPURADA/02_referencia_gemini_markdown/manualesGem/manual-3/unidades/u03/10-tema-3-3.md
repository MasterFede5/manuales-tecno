# 3.3 Role prompting y output structuring

Dos técnicas que parecen pequeñas pero **transforman** tu uso de la IA: 1) **darle un rol específico** al modelo (no "asistente genérico" sino "tutor de cálculo paciente que detecta errores comunes"), y 2) **forzar la estructura de salida** (no texto suelto sino JSON con campos exactos que tu programa puede procesar).

::concepto{titulo="Role prompting y structured output"}
**Role prompting.** Dar al modelo una **personalidad específica** afecta tono, vocabulario, profundidad y prioridades. Es la primera capa del prompt (3.1) llevada al extremo.

**Structured output.** Pedir al modelo que devuelva la respuesta en un **formato exacto y parseable** (JSON, XML, YAML, tabla markdown, lista numerada).
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Sin role, el modelo asume "asistente útil neutro". Con role, lo conviertes en **lo que necesites**: tutor paciente, editor severo, abogado defensor, devil's advocate. Sin structured output, recibes párrafos que tienes que leer; con él, recibes datos que **tu programa o tu Excel pueden procesar**. Estas dos técnicas convierten al LLM de "asistente conversacional" en "componente de pipeline".
::/interioriza

## Role prompting: el modelo es quien le digas

Compara estos dos prompts:

**Sin role:**
```
Explícame el principio de Bernoulli.
```

**Con role:**
```
Eres un tutor de física para bachillerato. Tu estilo es el de un coach paciente:
detectas malentendidos comunes, das ejemplos cotidianos, repites con
distintas palabras hasta que confirmas comprensión. Antes de explicar,
pregunta qué sabe el estudiante para no aburrirlo con lo obvio.

Explícame el principio de Bernoulli.
```

El segundo cambia **completamente** la respuesta: en vez de definición de manual, recibes diálogo pedagógico personalizado.

### Roles potentes para estudio

```
Eres mi tutor paciente. Antes de responder, pregúntame qué sé.
```
```
Eres mi devil's advocate. Critica mi argumento sin ofender pero sin suavizar.
```
```
Eres editor severo. Marca toda redundancia, frase pasiva o jerga vaga.
```
```
Eres mi mamá explicando esto: cariñosa pero sin condescendencia.
```
```
Eres profesor universitario riguroso. Si mi pregunta tiene error, lo señalas.
```

### Persona-style con tone y voice

Para tareas creativas, refina más:

```
Eres copywriter senior con 10 años en agencias mexicanas.
Estilo: directo, conversacional, con humor seco.
Voice: tú-tú al lector. Activa, no pasiva.
Tone: persuasivo pero sin clichés de "transforma tu vida".
```

## Structured output: el modelo te devuelve datos

Cuando vas a **procesar** la respuesta —con código, hoja de cálculo, otro modelo— pídele un formato exacto.

### JSON

```
Devuelve la respuesta en JSON con esta estructura exacta:

{
  "concepto": "string",
  "definicion": "string (máx 100 caracteres)",
  "formula": "string (LaTeX)",
  "ejemplo_numerico": {
    "datos": "string",
    "calculo": "string",
    "resultado": "string"
  },
  "preguntas_comprension": ["string", "string", "string"]
}

NO incluyas texto fuera del JSON.
```

> Buena práctica: **muestra el esquema explícito**. Mejor aún: usa few-shot con un ejemplo del JSON ya completado.

### Tabla markdown

```
Lista 5 LLMs en formato tabla markdown con columnas:
Nombre | Empresa | Año | Ventana_contexto | URL_oficial
```

### XML

```
Responde en XML:
<respuesta>
  <concepto>...</concepto>
  <ejemplo>...</ejemplo>
</respuesta>
```

### YAML (configuración)

```
Genera la configuración como YAML:
modelo: claude-opus-4-7
temperatura: 0.7
max_tokens: 1000
sistema: |
  Eres mi tutor de cálculo...
```

## Cuándo combinarlos

Para tareas serias, combina ambas técnicas:

```
Eres analista experto en datos. (ROLE)
Tu tarea es clasificar correos. (TASK)
Para cada correo devuelve JSON: (FORMAT)
{
  "categoria": "PERSONAL | LABORAL | SPAM | NEWSLETTER",
  "prioridad": "ALTA | MEDIA | BAJA",
  "necesita_respuesta": true|false,
  "tiempo_estimado_lectura_segundos": int
}

Correo a clasificar:
[contenido]
```

Resultado: respuesta lista para tu Python script de procesamiento masivo.
