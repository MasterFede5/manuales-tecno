# 3.1 Anatomía de un prompt profesional

Un prompt profesional **no es una pregunta** — es una **especificación**. Tiene 4-6 capas claramente definidas. Si pones las 6, el modelo te entrega exactamente lo que necesitas. Si pones solo la pregunta, te entrega lo que él considera promedio. Esta sección te enseña esa anatomía.

::concepto{titulo="Capas de un prompt profesional"}
Un prompt completo tiene hasta seis capas:

1. **Rol (Role)** — quién quieres que el modelo sea.
2. **Tarea (Task)** — qué quieres que haga.
3. **Contexto (Context)** — quién eres tú, qué sabes, para qué lo necesitas.
4. **Restricciones (Constraints)** — longitud, idioma, tono, lo prohibido.
5. **Formato (Format)** — cómo quieres la respuesta (estructura, secciones, tipo).
6. **Ejemplos (Examples)** — opcionales pero potentes (few-shot).

> Mnemotécnico **R-T-C-R-F-E**: "Rol-Tarea-Contexto-Restricciones-Formato-Ejemplos".
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Imagina que contratas a un asistente humano y le dices "ayúdame con la tarea". Ese asistente, por bueno que sea, te va a preguntar: "¿qué tarea? ¿de qué materia? ¿para cuándo? ¿qué tan formal? ¿lo entrego en Word o slides?". Las 6 capas del prompt son esas mismas preguntas **respondidas por adelantado**. Le ahorras al modelo tener que adivinar y le quitas su instinto de "responder lo más probable" para entregarte lo que **realmente** necesitas.
::/interioriza

## Capa 1 — Rol

Define **quién** debe ser el modelo. Cambia su tono, vocabulario y profundidad.

```
Eres un tutor de física experto en mecánica clásica para bachillerato mexicano.
```

> Sin rol, el modelo asume "asistente genérico". Con rol, ajusta vocabulario y nivel.

## Capa 2 — Tarea

**Qué** quieres que haga. **Verbo de acción explícito** (explicar, generar, comparar, analizar, listar, traducir).

```
Explícame el principio de Bernoulli.
```

> Trampa común. "Háblame de X" es ambiguo. Prefiere "explica X", "analiza X", "compara X y Y", "genera 5 ejemplos de X".

## Capa 3 — Contexto

**Quién eres tú**, qué ya sabes, para qué lo necesitas. Reduce alucinaciones y genera personalización.

```
Soy estudiante mexicano de 16 años en bachillerato.
Tengo examen el viernes.
Ya conozco la segunda ley de Newton (F = ma).
Mi profesor pone preguntas conceptuales más que cálculo.
```

> Más contexto ≠ siempre mejor. **Solo el contexto relevante** para esta tarea.

## Capa 4 — Restricciones

Lo que el modelo **NO** debe hacer y los límites que sí debe respetar.

```
- Máximo 500 palabras.
- Español neutro (no jerga regional).
- No uses cálculo diferencial; usa solo álgebra y geometría básica.
- Sin emoji ni jerga "cool".
```

## Capa 5 — Formato

Estructura exacta que quieres en la respuesta.

```
Estructura tu respuesta así:
1. Definición formal en 1-2 oraciones.
2. Analogía cotidiana en 1 párrafo.
3. Fórmula con explicación variable por variable.
4. Ejemplo numérico (con datos realistas y solución paso a paso).
5. Pregunta de comprobación al final.
```

> El formato hace tu respuesta **scaneable** y **reutilizable** después.

## Capa 6 — Ejemplos (few-shot)

Cuando la tarea es repetitiva o tiene formato no obvio, dale **1-3 ejemplos del input-output ideal**. (Lo verás a fondo en 3.2.)

## Ejemplo de prompt completo

::visual{tipo="infografia" descripcion="Anatomía completa de un prompt profesional descompuesta en 6 capas con colores distintos: 1) ROL azul. 2) TAREA verde. 3) CONTEXTO amarillo. 4) RESTRICCIONES naranja. 5) FORMATO morado. 6) EJEMPLOS rosa. Cada capa muestra el texto literal del prompt de Bernoulli aplicado. Al final, flecha hacia 'PROMPT FINAL ENVIADO AL MODELO'." paginas="1" src="../manualesGem/assets/visuales/manual-3/u03/10-tema-3-1-v01.svg"}

```
[ROL]
Eres un tutor de física experto en mecánica clásica para bachillerato mexicano.

[TAREA]
Explícame el principio de Bernoulli aplicado al alerón de un coche F1.

[CONTEXTO]
Soy estudiante de 16 años. Tengo examen de física el viernes.
Ya conozco fuerzas, presión y energía cinética.
Mi profesor evalúa más comprensión conceptual que memorización.

[RESTRICCIONES]
- Máximo 500 palabras.
- Español neutro.
- No uses cálculo diferencial.

[FORMATO]
Estructura tu respuesta así:
1. Definición formal (1-2 oraciones).
2. Analogía cotidiana (1 párrafo).
3. Fórmula con explicación.
4. Ejemplo numérico (alerón F1, datos realistas).
5. Pregunta de comprobación final.
```

## Comparativa: prompt malo vs prompt bueno

::visual{tipo="cuadro-comparativo" descripcion="Tabla comparativa de Prompt Malo vs Bueno con dos columnas: PROMPT MALO 'ayúdame con la tarea de física, es Bernoulli' → respuesta esperada vaga, definición de Wikipedia genérica. PROMPT BUENO con las 6 capas estructuradas → respuesta esperada detallada, personalizada al case study, con ejemplo F1 numérico y pregunta de comprobación. Tercera columna: lecciones extraídas del contraste." paginas="1" src="../manualesGem/assets/visuales/manual-3/u03/10-tema-3-1-v02.svg"}

| Prompt malo | Prompt bueno |
|---|---|
| "Ayúdame con la tarea de física, es Bernoulli" | El prompt completo con 6 capas |
| Respuesta vaga, tipo Wikipedia | Respuesta a tu medida, con ejemplo y pregunta |
| Te toca **iterar** mucho | Funciona en el primer turno |
| 30 segundos de prompt → 10 minutos de iteración | 3 minutos de prompt → 0 iteración |

> **Heurística profesional.** *"Si la respuesta del modelo no es lo que esperabas, casi siempre el problema está en el prompt, no en el modelo."*
