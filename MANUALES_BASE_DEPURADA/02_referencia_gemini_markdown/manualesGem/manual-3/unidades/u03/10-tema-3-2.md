# 3.2 Zero-shot, few-shot, chain-of-thought y técnicas avanzadas

Un mismo modelo puede responder con calidad de **escolar promedio** o de **PhD** según qué técnica de prompting uses. Esta sección te enseña las cinco más importantes, todas con ejemplos.

::concepto{titulo="Técnicas de prompting"}
**Zero-shot.** Le pides directamente sin ejemplos. El modelo usa lo que aprendió en entrenamiento.

**Few-shot.** Le das 2-5 ejemplos del formato deseado, luego haces la pregunta nueva. El modelo imita el patrón.

**Chain-of-thought (CoT).** Le pides que piense paso a paso antes de responder. Útil para matemáticas, lógica, planning.

**Self-consistency.** Le pides la misma pregunta varias veces y tomas la respuesta más frecuente. Reduce errores aleatorios.

**ReAct (Reason + Act).** Alternas razonamiento con acciones (consultar herramientas externas).
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Cada técnica tiene su contexto óptimo. Pedirle a un modelo que "razone paso a paso" cuando solo le preguntas la fecha de hoy es absurdo y desperdicia tokens; pedírselo en un problema de probabilidad puede triplicar la precisión. Saber **cuándo** usar cada una es la diferencia entre desperdiciar suscripción y aprovecharla.
::/interioriza

## Zero-shot

Es el **default**. Sin ejemplos, sin pasos. Le preguntas y responde.

```
Clasifica el siguiente tweet como POSITIVO, NEUTRAL o NEGATIVO:

"El nuevo procesador es rapidísimo, pero la batería dura nada."
```

> Funciona bien para tareas comunes y bien definidas. **Falla** cuando el formato es no obvio o la tarea es ambigua.

## Few-shot

Le das **ejemplos del input-output deseado**. El modelo aprende el patrón sin reentrenamiento.

```
Clasifica el sentimiento del tweet como POSITIVO, NEUTRAL o NEGATIVO.

Ejemplos:
Tweet: "Excelente servicio, lo recomiendo."
Sentimiento: POSITIVO

Tweet: "El producto llegó tarde y dañado."
Sentimiento: NEGATIVO

Tweet: "El paquete llegó hoy a las 3pm."
Sentimiento: NEUTRAL

Tweet: "El nuevo procesador es rapidísimo, pero la batería dura nada."
Sentimiento:
```

> **Cuándo usarlo:** clasificación, formato custom, generación de código en estilo específico, traducciones a tono particular.

> **Tip pro.** 3-5 ejemplos suelen ser óptimos. Más de 8 raramente mejora y consume contexto.

## Chain-of-thought (CoT)

Le pides que **razone paso a paso** antes de dar la respuesta final. Mágico para matemáticas, lógica, planning.

```
Pregunta: María tiene 23 manzanas. Compra 47 más, regala 18 y se le caen 5.
¿Cuántas manzanas tiene al final?

Piensa paso a paso antes de responder.
```

vs zero-shot ("Cuántas manzanas tiene al final?") que en problemas largos puede equivocarse.

> **Variantes:**
> - "*Let's think step by step.*" (frase mágica que descubrieron por accidente — Kojima 2022).
> - "*Razona como un experto en X antes de responder.*"
> - "*Lista los pasos de tu razonamiento numerados, luego da tu conclusión final.*"

## Self-consistency

Le pides la misma pregunta **5 veces** (con temperatura > 0 para que varíe) y tomas la respuesta más común. Reduce errores aleatorios.

```
[Mismo prompt repetido 5 veces]
→ Respuesta 1: 47
→ Respuesta 2: 47
→ Respuesta 3: 48
→ Respuesta 4: 47
→ Respuesta 5: 47

Mayoría: 47 (4 de 5).
```

> **Costo.** 5× más tokens. Solo úsala cuando importa MUCHO la precisión y puedes permitirte el costo.

## ReAct (Reason + Act)

El modelo **alterna** razonamiento con **acciones** (consultar una API, buscar en internet, leer un archivo). Patrón típico:

```
Thought: Necesito averiguar la población actual de México.
Action: search("población de México 2025")
Observation: 130 millones según INEGI 2024.
Thought: Ahora dividir entre PIB. Necesito el PIB.
Action: search("PIB México 2024")
Observation: 1.79 trillones USD.
Thought: PIB per cápita = 1.79e12 / 130e6 = 13,769 USD.
Final Answer: PIB per cápita 2024 ≈ 13,769 USD.
```

> **Lo usan agentes** (ChatGPT con browsing, Claude con tools, Gemini con Workspace). Cuando los configuras tú mismo (U7), defines qué tools puede llamar.

## Tree-of-thoughts (avanzado)

El modelo explora **varios caminos de razonamiento** y elige el mejor. Útil en problemas con múltiples soluciones (puzzles, planning).

## Cuándo usar cada una

| Tarea | Técnica recomendada |
|---|---|
| Pregunta simple, conocimiento general | **Zero-shot** |
| Clasificación o formato custom | **Few-shot** |
| Matemáticas, lógica, planning | **Chain-of-thought** |
| Decisiones críticas (medicina, legal) | **Self-consistency** |
| Tareas que requieren herramientas | **ReAct** |
| Problemas con múltiples soluciones | **Tree-of-thoughts** |

> **Combinaciones potentes:** few-shot + CoT, ReAct + self-consistency. No tienes que usar una sola.
