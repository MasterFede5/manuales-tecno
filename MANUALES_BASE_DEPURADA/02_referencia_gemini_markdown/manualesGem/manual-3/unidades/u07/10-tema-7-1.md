# 7.1 GPTs personalizados: instrucciones, knowledge files y actions

Un **GPT** es una versión personalizada de ChatGPT. No reentrenas el modelo (no podrías), pero le das **instrucciones detalladas, archivos como knowledge y capabilities específicas**. Es la forma más accesible de construir un asistente IA propio.

::concepto{titulo="GPT personalizado"}
Un **GPT** es una configuración guardada en ChatGPT que combina:
1. **Identidad** — nombre, descripción, avatar.
2. **Instructions** — system prompt detallado (qué debe hacer, qué no, tono, formato).
3. **Knowledge** — hasta 20 archivos que el GPT puede consultar (PDFs, docx, txt, csv).
4. **Capabilities** — web browsing, DALL·E (genera imágenes), Code Interpreter (corre Python).
5. **Actions** — conexiones a APIs externas (avanzado).
6. **Distribución** — privado (solo tú), link compartido, o público en GPT Store.

**Acceso:** ChatGPT Plus ($20/mes). Free tier: solo usar GPTs ajenos.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Cada vez que necesitas el mismo asistente —tutor de cálculo, redactor de correos formales, coach de gimnasio— sin GPT escribirías el mismo system prompt en cada chat nuevo. Con un GPT lo haces **una vez** y lo invocas con un click. Y si lo publicas, otros estudiantes pueden usarlo. Es el primer paso para volverte productor, no solo consumidor de IA.
::/interioriza

### Capa 1 — Identidad

```
Name: Tutor Física Albatros
Description: Tu tutor personal de física para bachillerato mexicano.
Especializado en mecánica, termodinámica y electromagnetismo.
Avatar: [genera con DALL·E o sube imagen]
```

### Capa 2 — Instructions (la más importante)

Aquí escribes el **system prompt completo**. Recomendación: 500-2000 palabras con esta estructura:

```
[ROL Y EXPERTICIA]
Eres "Tutor Física Albatros", un tutor experto en física a nivel bachillerato
mexicano. Conoces a fondo el programa oficial SEP-UNAM Área I.

[PERSONALIDAD Y TONO]
Tu estilo:
- Paciente, nunca condescendiente.
- Usa analogías cotidianas mexicanas (futbol, tráfico CDMX, cocina).
- Detecta malentendidos comunes y los corrige sin acusar.
- Usa "tú" en lugar de "usted".

[METODOLOGÍA PEDAGÓGICA]
1. Antes de explicar algo nuevo, pregunta qué sabe el estudiante.
2. Empieza con analogía cotidiana, luego concepto formal, luego ejemplo numérico.
3. Pregunta de comprobación al final ("¿lo expliqué claro?", "¿quieres
   un ejemplo más?").
4. Si detectas error en lo que dice el estudiante, corrige con respeto.

[CASOS DE EXCEPCIÓN]
- Si pregunta sobre temas fuera de física, redirígelo gentilmente.
- Si pide hacer su tarea por él, ofrece guiarlo paso a paso, no resolverla.
- Si parece frustrado, baja un nivel de complejidad.

[FORMATO DE RESPUESTA]
- Máximo 400 palabras por respuesta.
- Usa LaTeX para fórmulas: $E = mc^2$.
- Negritas para términos clave.
- Ejemplos en bloques visualmente distintos.

[KNOWLEDGE]
Tienes acceso a:
- Libro Tippens (PDF) — referencia principal.
- Programa oficial SEP-UNAM (PDF).
- 10 slides del profesor (PDFs).
- 5 exámenes anteriores (PDFs).
Cita siempre el archivo y página cuando uses esta knowledge.

[LO QUE NO HACES]
- No inventas datos, fórmulas o citas.
- Si no estás seguro, dilo: "no estoy seguro de eso, te recomiendo
  consultar X".
- No haces tareas completas; guías a hacerlas.
```

### Capa 3 — Knowledge

Carga hasta 20 archivos (cada uno hasta 512 MB). Tipos: PDF, docx, xlsx, csv, txt.

**Buenas prácticas:**
- Carga **lo esencial** del curso, no toda la biblioteca.
- Documentos bien estructurados (con títulos H1, H2) son consultados mejor.
- PDFs escaneados de baja calidad son **menos útiles** (OCR malo).

### Capa 4 — Capabilities

Activa según necesidad:
- **Web Browsing** — útil si tu GPT necesita info actualizada.
- **DALL·E Image Generation** — si necesitas generar diagramas.
- **Code Interpreter** — Python para cálculos numéricos, gráficos.

> **Tip.** Si tu GPT es educativo de física, activa **Code Interpreter** — puede graficar funciones, simular movimientos, resolver ecuaciones.

### Capa 5 — Actions (avanzado)

Conecta APIs externas. Ejemplos:
- Wolfram Alpha para cálculo.
- Tu propia API si tienes un sistema externo.
- APIs públicas (clima, traducción, etc.).

> **Para principiantes**, omite actions. Las activas cuando seas usuario avanzado.

### Conversation Starters

4 frases predefinidas que aparecen al iniciar el chat. Diseñalas para que el usuario sepa **qué puede preguntar**:

```
- "Explícame el principio de Bernoulli"
- "Genera 5 problemas de cálculo de movimiento"
- "Tengo dudas sobre la unidad 4 del libro"
- "Hazme un quiz rápido del tema X"
```

## Cómo construir paso a paso

```
1. Ve a chatgpt.com → Explore GPTs → Create.
2. Modo "Configure" (más control que "Create" conversacional).
3. Pegas el Instructions completo.
4. Cargas tus 12-20 archivos en Knowledge.
5. Activas capabilities relevantes.
6. Escribes 4 conversation starters.
7. Generas avatar con DALL·E.
8. Probar 5 prompts distintos en preview.
9. Iteras instructions con base en pruebas.
10. Save → eliges visibilidad.
```

## Distribución

**Tres niveles:**
- **Only me** — privado. Bueno para empezar.
- **Anyone with link** — comparten link directo. Ideal para tu equipo de proyecto.
- **GPT Store** — público, indexado, descubrible. Necesita verificación de cuenta.

> **Tip de monetización (2024+).** OpenAI paga revenue share a creadores cuyos GPTs son usados por suscriptores Plus. No te harás rico, pero genera ingresos modestos.

## Limitaciones de GPTs

- **Solo en ChatGPT.** Si tu usuario no tiene Plus, no puede usarlo (con tier gratuito limitado).
- **Knowledge no se actualiza solo.** Si subes nuevos archivos, lo haces a mano.
- **No mantiene memoria entre usuarios.** Cada usuario empieza desde cero.
- **Privacidad de archivos.** Los archivos suben a OpenAI; quien tenga acceso al GPT puede ver el contenido si pregunta.
