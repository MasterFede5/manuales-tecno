# 10.2 Costos por tokens y optimización

Tu Asistente cuesta $300/mes en API. Con técnicas correctas, lo bajas a $80/mes manteniendo calidad. Esta lección te enseña esas técnicas.

::concepto{titulo="Costos por tokens y optimización"}
Los **tokens** son las unidades en que los proveedores cobran inferencia. Cada modelo tiene precio por 1M de tokens de **input** y **output**. La optimización combina:

- **Modelo correcto por tarea** (Haiku para clasificación, Opus para razonamiento).
- **Prompt caching** (reusar contexto largo sin re-cobrar).
- **Batch API** (procesar offline a 50 % de costo).
- **Reducción de tokens** (prompts concisos, max_tokens limitado).
- **RAG eficiente** (chunks justos, no toda la BD).
- **Local cuando aplica** (Ollama gratis, U7).
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
La diferencia entre $300 y $80 mensuales es $2 640 anuales — el costo de un nuevo proyecto educativo de tu institución. La disciplina de optimización no es ahorro técnico: es **reasignación de presupuesto** a lo que realmente importa.
::/interioriza

## Las 6 técnicas de optimización

### 1. Modelo correcto por tarea

Patrón híbrido (cubrimos en U6):
- Clasificación / extracción → Haiku / Flash / mini.
- Razonamiento / redacción / generación creativa → Sonnet / GPT-5 / Pro.
- Razonamiento crítico extremo → Opus / o3.

Si todo lo tiras a Opus, gastas 18× más que con Haiku donde Haiku basta.

### 2. Prompt caching

Si tu Asistente envía el mismo system prompt + RAG (10k tokens) en cada llamada, **paga 10k tokens cada vez**. Con caching:
- Primer envío: 1.25× costo normal (escribe en cache).
- Llamadas siguientes: 0.10× costo (lee de cache).

Resultado: ~85 % menos costo en la parte cacheable. Anthropic, OpenAI, Google ofrecen caching desde 2024-2025.

### 3. Batch API

Si tienes 1000 procesamientos no urgentes (ej.: clasificar 1000 correos), envíalos batch:
- Anthropic Batch API: 50 % descuento, latencia 24h.
- OpenAI Batch API: 50 % descuento, latencia 24h.

Para tareas nocturnas / asíncronas, ahorro inmediato.

### 4. Reducción de tokens

- Prompts concisos: cada palabra extra cuesta. Edita.
- `max_tokens` limitado: si esperas respuesta de 200 tokens, no pongas 4000.
- System prompts cortos pero precisos.
- Few-shot mínimo: 3 ejemplos > 10 ejemplos similares.

### 5. RAG eficiente

- Re-ranker reduce de top-50 a top-5 antes de pasar al LLM.
- Chunks de 400-600 tokens (no 2000).
- Caching de queries frecuentes.

### 6. Híbrido cloud-local

Tareas masivas y rutinarias: local (Ollama).
Tareas críticas con datos no sensibles: cloud.

## Caso real: optimización del Asistente Institucional

Antes de optimizar:
```
80 inscripciones/mes × Sonnet $3+15 × 5k tokens cada conversación
= ~6M tokens/mes en API
= ~$80/mes solo en clasificación
```

Después:
```
80 inscripciones/mes × Haiku $0.80+4 × 5k tokens
+ caching de system prompt (90% reducido)
+ batch para reportes nocturnos
= ~$15/mes
```

**Ahorro: $65/mes = $780/año**, sin pérdida de calidad.

## El cálculo del costo total (TCO)

```
TCO mensual = 
   Costos API
 + Subscripciones planes (ChatGPT Team, Claude Pro, etc.)
 + Hardware local amortizado
 + Electricidad
 + Tiempo TI mantenimiento × tarifa
 + Plataformas no-code (n8n cloud, Dify, etc.)
 + Servicios externos (LangFuse, Vector DBs cloud)
```

Comparado con beneficios:
```
ROI = (Tiempo humano ahorrado × tarifa)
    + (Casos resueltos sin escalación × valor de cada uno)
    + (Reducción de errores × costo de error)
    - TCO
```

Documenta esto en el plan a 12 meses.

## Anti-patrones

- **No medir.** Sin tracking, no sabes qué optimizar.
- **Optimizar sin medir calidad.** Ahorros que pierden 20 % calidad no son ahorros.
- **Caching agresivo en datos sensibles.** El cache puede mantener PII más tiempo del esperado.
- **Batch para tareas urgentes.** Latencia 24h no sirve para chat en vivo.
- **Modelo barato cuando necesitas razonamiento.** Sale más caro arreglar errores.
