# 10.5 Confidencialidad y datos sensibles

Tu Asistente vive con datos institucionales todos los días. Esta lección define **dónde puede vivir cada tipo de dato** según su sensibilidad y dónde NO debe vivir nunca.

::concepto{titulo="Clasificación de datos y sistemas autorizados"}
La **clasificación de datos** es el ejercicio de etiquetar cada categoría de información de tu institución según su sensibilidad y emparejarla con sistemas autorizados para procesarla.

Cuatro niveles canónicos:
- **Públicos**: web, brochures, calendario público.
- **Internos**: comunicaciones generales, materiales pedagógicos, plantillas.
- **Confidenciales**: calificaciones, expedientes, comunicaciones con padres, datos de salud (no específica), PII de menores.
- **Restringidos**: contratos, NDAs, datos de salud específicos, información sujeta a investigación, secretos comerciales.

Cada nivel tiene un universo de sistemas IA donde puede procesarse, y otros donde **NO**.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
La diferencia entre "incidente menor" y "multa LFPDPPP de millones" suele ser un solo descuido: alguien que pegó algo de nivel 3 en un sistema de nivel 1. La matriz dato↔sistema previene 90 % de esos incidentes si el staff la conoce.
::/interioriza

## La matriz dato × sistema

| Sistema → / Datos ↓ | ChatGPT free | ChatGPT Pro/Plus | Claude Pro | Plan Enterprise | Asistente Local | Sin IA |
|---|---|---|---|---|---|---|
| **Públicos** | ✓ | ✓ | ✓ | ✓ | ✓ | (innecesario) |
| **Internos** | ✗ | ✓ con opt-out | ✓ con opt-out | ✓ | ✓ | ✓ |
| **Confidenciales** | ✗ | ✗ | ✗ | ✓ con DPA | ✓ | ✓ |
| **Restringidos** | ✗ | ✗ | ✗ | ✗ | ✓ con auditoría | ✓ |

DPA = Data Processing Agreement.

## Lista canónica de datos sensibles en institución educativa

- Nombre completo de alumno + datos asociados → **confidencial**.
- Calificaciones individuales → **confidencial**.
- Datos de salud declarados → **confidencial / restringido**.
- Datos de necesidades educativas especiales → **confidencial**.
- Datos económicos (becas, pagos) → **confidencial**.
- Comunicaciones disciplinarias → **restringido**.
- Investigaciones internas → **restringido**.
- Contratos y NDAs → **restringido**.
- PII de personal → **confidencial**.
- Datos de menores siempre → mínimo **confidencial**.

## Controles técnicos por nivel

### Para datos confidenciales

- **Plan Enterprise** del proveedor cloud (que no entrene con tus datos) **+ DPA firmado**.
- O **Asistente Local** (Ollama, U7).
- **Logs auditables** de cada acceso.
- **Tokens con permisos mínimos** (read-only siempre que aplique).
- **Cifrado en reposo** (provider o tu BD).

### Para datos restringidos

- **Solo Asistente Local** o sistema en infraestructura de la institución.
- **Logs revisados mensualmente**.
- **Auth multi-factor** para acceder.
- **Auditoría externa** anual.
- **Retención mínima**.

## El control humano "two-man rule"

Para acciones sobre datos restringidos: **dos personas** deben aprobar (uno autoriza, otro ejecuta o audita). Reduce 95 % los incidentes accidentales.

## Plan de respuesta a incidente

Cada institución tiene **runbook de incidente IA** con datos sensibles:

```
1. Detección — quién detecta, cómo se reporta.
2. Contención — apagar el flujo, rotar credenciales, snapshot logs.
3. Notificación — INAI / autoridad si aplica (72h GDPR / depending LFPDPPP).
4. Comunicación interna — staff, dirección, patronato.
5. Comunicación afectados — padres si datos de menores se filtraron.
6. Análisis — qué falló específicamente.
7. Remediación — fix técnico + actualizar política.
8. Post-mortem — sin culpa personal, foco en sistema.
```

Sin runbook ensayado, los primeros 30 minutos del incidente se desperdician en pánico.

## Anti-patrones

- **Sin clasificación.** Cada quien decide qué es confidencial.
- **Tokens compartidos.** Si el token de Notion lo tienen 5 personas, no auditar.
- **DPA implícito.** "Confío en OpenAI" no cuenta. Documenta DPA firmado o evita.
- **Sin runbook.** Incidente cuando llegue será caótico.
- **Logs sin revisión.** Tener logs sin auditarlos es como cámaras sin operador.
