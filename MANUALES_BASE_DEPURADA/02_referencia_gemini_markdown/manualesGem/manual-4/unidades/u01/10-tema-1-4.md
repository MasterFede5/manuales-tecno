# 1.4 Self-consistency y Tree-of-Thoughts

Le pides al modelo "diseña un plan de inscripciones para 200 alumnos en 5 días". Te entrega un plan plausible, pero ¿es el mejor de los posibles? Le pides otra vez y te entrega uno distinto, también plausible. ¿Cuál llevas a la junta? Este subtema te da el método.

::concepto{titulo="Self-consistency y Tree-of-Thoughts (ToT)"}
**Self-consistency** es una técnica donde se generan **múltiples cadenas de razonamiento independientes** para la misma pregunta y se elige la respuesta más frecuente (mayoría). Funciona en problemas con respuesta única bien definida (cálculos, lógica, clasificación).

**Tree-of-Thoughts (ToT)** es un patrón en el que el modelo **explora un árbol de razonamientos**, evalúa nodos parciales, descarta ramas pobres y profundiza en las prometedoras. Funciona en problemas abiertos con múltiples soluciones (planeación, diseño, escritura creativa).

Ambas son extensiones de CoT: en lugar de una sola cadena, exploran varias.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Cuando un comité decide a quién contratar, escucha a varios entrevistadores y vota. Una sola opinión es ruido; tres opiniones independientes filtran ruido. La decisión final es más estable. Self-consistency hace lo mismo con tu modelo: en lugar de fiarte de un solo intento, comparas varios y la respuesta más repetida suele ser la correcta.
::/interioriza

## Self-consistency en 3 pasos

1. **Misma pregunta, N veces.** Lanzas el mismo prompt con `temperature=0.7` (no determinista) entre 3 y 10 veces.
2. **Extrae la respuesta final** de cada corrida (ignora el razonamiento intermedio).
3. **Voto por mayoría.** La respuesta más frecuente gana. Si empatan dos, lanzas más corridas o pides arbitraje al modelo (*"¿Cuál de estas dos cadenas es más sólida y por qué?"*).

```
Pregunta: ¿Cuál de los siguientes 3 horarios es óptimo para sesión informativa?
A) Lunes 18:00 · B) Sábado 10:00 · C) Miércoles 19:30

[Corrida 1] → razonamiento → respuesta: B
[Corrida 2] → razonamiento → respuesta: B
[Corrida 3] → razonamiento → respuesta: A
[Corrida 4] → razonamiento → respuesta: B
[Corrida 5] → razonamiento → respuesta: B

Voto: B (4/5). Respuesta final: B.
```

Self-consistency cuesta más (N llamadas), pero **reduce el error** entre 10 y 25 % en tareas matemáticas y lógicas según los papers originales (Wang et al. 2022).

## Tree-of-Thoughts en 4 pasos

1. **Descomposición.** El modelo divide el problema en pasos / sub-decisiones.
2. **Generación de candidatos.** Para cada paso, el modelo propone 3–5 alternativas.
3. **Evaluación.** El modelo asigna a cada alternativa un score (0–10) según criterios definidos.
4. **Búsqueda.** Se profundiza en la rama de mayor score; se podan las pobres. Se itera hasta llegar a una hoja (solución completa) o a un presupuesto agotado de pasos.

Patrón típico para planeación:

```xml
<task>
Diseña el plan de inscripciones para 200 alumnos en 5 días.
</task>

<tree_of_thoughts>
Paso 1: Genera 3 estrategias generales distintas.
Paso 2: Para cada estrategia, lista 3 sub-tácticas.
Paso 3: Evalúa cada combinación 1-9 según: factibilidad, costo,
        riesgo, satisfacción del usuario.
Paso 4: Elige las 2 mejores combinaciones, profundiza con cronograma día a día.
Paso 5: Compara los 2 cronogramas finales y recomienda uno.
</tree_of_thoughts>
```

ToT es **caro** (10–50× CoT simple) pero produce planes mucho más sólidos. Úsalo solo cuando el costo de un plan mal diseñado supere el costo de muchas llamadas al modelo.

## ¿Cuándo elijo cada una?

| Característica del problema | Self-consistency | Tree-of-Thoughts |
|---|---|---|
| Respuesta única correcta | ✓ | (overkill) |
| Múltiples soluciones aceptables | (no aplica) | ✓ |
| Cálculo o lógica | ✓ | (no necesario) |
| Diseño, planeación, escritura | (no resuelve) | ✓ |
| Presupuesto bajo | ✓ (3 corridas) | ✗ |
| Decisión de alto impacto | ✓ + auditoría | ✓ con criterios explícitos |

## Reglas operativas para tu Asistente

1. **Default = una sola pasada.** No pagues self-consistency ni ToT en consultas casuales.
2. **Self-consistency para reactivos.** Cuando el Asistente clasifica solicitudes en categorías (5 categorías fijas), corres 3 veces y votas.
3. **ToT para planes mensuales.** Una vez al mes, cuando el Asistente propone un plan de comunicaciones, costo extra justificado.
4. **Logueo de votación.** En self-consistency, guardas las N corridas; si una vota distinto, la quieres ver para detectar bordes.
