# 1.2 Meta-prompting

¿Quién está mejor calificado para escribir un prompt para Claude que el propio Claude? Ese es el chiste —y la herramienta operativa— del meta-prompting.

::concepto{titulo="Meta-prompting"}
El **meta-prompting** consiste en usar un modelo de IA para diseñar, refinar o evaluar otros prompts. En lugar de escribir tú mismo el prompt definitivo, le pides al modelo que **redacte la mejor versión posible** dada una tarea, criterios y restricciones.

Tres modalidades:
- **Prompt-author** — el modelo te entrega un prompt listo para usar.
- **Prompt-critic** — el modelo evalúa un prompt existente y sugiere mejoras.
- **Prompt-rewriter** — el modelo reescribe iterativamente hasta cumplir criterios.

El meta-prompting funciona porque el modelo conoce sus propios mecanismos de atención mejor que un humano externo: sabe qué estructura responde mejor, qué palabras lo enganchan, qué orden de instrucciones reduce alucinaciones.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Los equipos de Anthropic, OpenAI y Google tienen "ingenieros de prompts" cuyo trabajo es justamente este: escribirlos, criticarlos, versionarlos. Tú no necesitas contratar uno: **tu modelo ya es ese ingeniero**, solo tienes que pedírselo. Diez minutos de meta-prompting bien hecho ahorran 10 horas de prueba y error con prompts mediocres.
::/interioriza

## El meta-prompt patrón (úsalo tal cual)

```
Eres un ingeniero de prompts experto. Voy a darte una tarea y necesito
que diseñes el prompt óptimo para resolverla con un LLM.

<objetivo>
[describe en 1-2 frases qué quieres lograr]
</objetivo>

<audiencia_destino_del_output>
[quién va a leer/usar el resultado del prompt]
</audiencia_destino_del_output>

<criterios_de_éxito>
[3-5 criterios verificables: longitud, tono, formato, etc.]
</criterios_de_éxito>

<restricciones>
[qué NO debe hacer el output]
</restricciones>

Tarea:
1. Diseña un prompt en estructura XML con secciones <role>, <context>,
   <task>, <constraints>, <examples> y <output_format>.
2. Justifica cada decisión de diseño en 1 línea.
3. Propón 3 ejemplos de input que probarían bordes del prompt.
```

Pega ese meta-prompt en Claude o ChatGPT con los datos de tu tarea y obtienes el prompt y su justificación en una sola pasada.

## Bucle de refinamiento iterativo

Un solo round rara vez basta. El bucle profesional es:

1. **Genera v1** con el meta-prompt anterior.
2. **Ejecuta v1** contra 5 inputs reales.
3. **Pide crítica:** *"Aquí está el prompt v1 y los 5 outputs. ¿Dónde falla? ¿qué dice de más, qué olvida, qué inconsistencias hay?"*
4. **Pide reescritura:** *"Reescribe el prompt corrigiendo los 3 problemas más graves. Marca con `// CHANGE:` los cambios."*
5. **Repite** hasta que la crítica de paso 3 te devuelva "no encuentro mejoras sustantivas".

Suelen bastar 2–4 vueltas para llegar a un prompt de calidad de producción.

## Anti-patrones del meta-prompting

- **Meta-promptear sin criterios.** "Hazme un prompt para resumir" devuelve algo genérico. Define cómo medirás éxito antes de pedir el prompt.
- **Aceptar el primer output como definitivo.** El bucle de refinamiento es donde está el valor.
- **Confundir meta-prompt con prompt final.** El meta-prompt es la maquinaria que produce; el prompt es el producto.
- **Perder versiones.** Cada iteración tiene un número y se guarda. Más en el subtema 1.8.

## Caso aplicado: prompt para responder dudas de inscripción

Tu Asistente Institucional debe responder preguntas frecuentes sobre inscripciones. Pasos:

1. **Meta-prompt** con objetivo "responder dudas de padres de familia sobre inscripciones".
2. **v1 generada** por Claude: 220 palabras, secciones XML, 2 ejemplos.
3. **Pruebas con 5 preguntas reales** del equipo de admisiones.
4. **Crítica:** v1 no maneja bien preguntas con tono molesto.
5. **v2:** se agrega una restricción `<de_escalation>` con 3 frases empáticas.
6. **Pruebas** vuelven y el v2 maneja tono molesto correctamente.
7. **Versionado:** `prompt-admisiones-v2.0.xml` queda en el repo del equipo.
