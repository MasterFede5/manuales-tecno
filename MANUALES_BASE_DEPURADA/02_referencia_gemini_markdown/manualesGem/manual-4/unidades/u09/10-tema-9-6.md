# 9.6 Generación de datasets sintéticos

Tu sistema necesita 200 ejemplos reales para entrenarse o testear. Tienes 30. Antes: meses recolectando. Hoy: pides al modelo que genere 170 ejemplos sintéticos plausibles. Esta lección te enseña cómo hacerlo bien.

::concepto{titulo="Datasets sintéticos"}
Los **datasets sintéticos** son colecciones de datos **generados por IA** que imitan la estructura, distribución y patrones de datos reales. Usos:

- **Entrenamiento** cuando no hay suficientes datos reales.
- **Testing y QA** sin exponer datos sensibles.
- **Golden sets** para evaluar prompts (ver U1).
- **Simulación** de escenarios edge.
- **Balanceo de clases** (cuando una categoría está sub-representada).
- **Privacidad** (alternativa a anonimización).

Caveats:
- Sintéticos pueden replicar sesgos del modelo generador.
- Puede haber distribución artificial (no captura cola larga real).
- No sustituyen datos reales para casos críticos.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
La frase "no tenemos suficientes datos para X" mata muchos proyectos. Datasets sintéticos los desbloquean — pero solo si los generas con criterio. Generarlos a tontas es producir basura tabulada.
::/interioriza

## Caso 1: Golden set para evaluación de prompts

Tu Asistente clasifica solicitudes en 5 categorías. Necesitas 30-50 ejemplos golden con clase verdadera para evaluar el prompt.

```
Genera 50 mensajes simulados de padres de familia escribiendo a admisiones.
Cada mensaje en español mexicano natural, 30-150 palabras.

Distribución:
- 10 inscripciones (5 con duda directa, 3 con preguntas adicionales, 2 con tono molesto)
- 10 calificaciones (4 directos, 4 quejas, 2 sobre revisión)
- 10 becas (3 hermanos, 4 económicas, 3 mérito)
- 10 quejas (3 docentes, 4 administrativas, 3 disciplina)
- 10 otro (5 servicios, 5 dudas variadas)

OUTPUT: JSONL con {mensaje, categoria_real, tono, complejidad}.
```

Resultado: golden set robusto en 5 minutos.

## Caso 2: Datos de prueba para BD

Tu sistema de inscripciones nuevo necesita poblarse con 500 alumnos ficticios para QA.

```
Genera 500 registros de alumnos sintéticos en formato CSV.

Columnas:
- nombre (mexicano realista, distribución por género balanceada)
- fecha_nacimiento (entre 2008-2012 para 1.º bachillerato)
- promedio_secundaria (distribución normal media 8.5, desv 0.8)
- ciudad_origen (mayoría CDMX, distribuir)
- materia_favorita (variada)
- requiere_beca (15% sí)
- email (ficticios pero formato realista)

OUTPUT: CSV listo para importar.
```

Datos sintéticos = QA puede empezar inmediato sin esperar matrícula real.

## Caso 3: Simulaciones de escenarios edge

Necesitas testear "qué pasa si llegan 50 solicitudes en una hora".

```
Genera 50 solicitudes con timestamps en intervalos de 30-90 segundos
durante una hora del lunes 9-10am.

Mezcla: 30 inscripción, 10 info, 5 quejas, 5 otro.
Algunas con typos, otras con texto perfecto.
3 deben tener formato sospechoso (prompt injection adversarial).

OUTPUT: JSONL con timestamp, mensaje, expected_category.
```

Tu sistema enfrenta carga sintética y revela cuellos de botella sin esperar a producción.

## Caso 4: Balanceo de clases para fine-tuning o evaluación

Tienes 100 ejemplos reales: 80 categoría A, 5 categoría B, 15 categoría C. Si entrenas/evaluas, el modelo se concentra en A y falla en B.

```
Genera 75 ejemplos sintéticos para categoría B siguiendo este patrón:
[15 ejemplos reales de B pegados]

Variaciones esperadas:
- Distintos tonos
- Distintas longitudes
- Distintos sub-temas dentro de B
- 5 % con casos de borde

OUTPUT: JSONL.
```

Resultado: dataset balanceado para entrenamiento o evaluación más justa.

## Reglas de oro

1. **Schema explícito.** Define columnas, tipos, dominios, distribuciones esperadas.
2. **Ejemplos reales como semilla.** 5-15 ejemplos reales que el modelo imite.
3. **Validación estadística.** Verifica que la distribución del sintético se parezca a la real.
4. **Etiquetado de origen.** Cada fila debe tener `is_synthetic: true` para no confundir con real.
5. **Auditoría de sesgos.** El modelo replica sus sesgos; revisa demografía, temas, lenguaje.

## Anti-patrones

- **Prompt vago: "genera datos de alumnos".** Resultado: lorem ipsum tabulado.
- **Sin ejemplos semilla.** Resultado: distribución artificial alejada de la realidad.
- **Mezclar real con sintético sin etiquetar.** Imposible auditar después.
- **Sintético para datos sensibles regulados.** Para LFPDPPP, sintético no es "anonimización"; es algo distinto. Cuidado al asumir.
- **Sin verificación humana.** Datos sintéticos pueden tener errores semánticos (un nombre "femenino" con género masculino).

## Cuándo NO usar sintéticos

- Decisiones de alto impacto basadas solo en simulación.
- Cumplimiento regulatorio que exige datos reales documentados.
- Investigación científica (excepto como complemento explícito).
- Producción con usuarios reales (claro).
