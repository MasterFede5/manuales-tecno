# 5.1 n8n con IA

Treinta minutos manuales por solicitud × 80 solicitudes al mes = 40 horas perdidas. Un workflow en n8n te quita esas 40 horas a costa de un día de aprendizaje y $0 si lo self-hosteas. Esta lección te enseña el motor.

::concepto{titulo="n8n con nodos AI"}
**n8n** (pronunciado "n-eight-n") es una plataforma open-source de automatización visual de workflows. Cada workflow es un **canvas de nodos conectados**: cada nodo recibe datos, los transforma o llama a un servicio, y pasa output al siguiente. Desde 2024 trae **nodos AI nativos** (LangChain, OpenAI, Anthropic, agentes) que permiten construir flujos de IA sin código.

Características clave:
- **Open source y self-hostable** (Docker en 5 min) o cloud n8n.io de pago.
- **400+ integraciones nativas** (Notion, Airtable, Slack, Google Workspace, etc.).
- **Nodos JavaScript** para lógica custom cuando se necesita.
- **Webhook triggers** para recibir eventos de cualquier sistema externo.

Es el "Zapier para developers" — más potente, más barato y más privado.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Antes de los workflows, un pedido de inscripción era una conversación interminable: "ya pasé el correo a Pepe, Pepe lo subió a Notion, Pepe le dijo a Lucía que mandara confirmación, Lucía olvidó". n8n convierte esa cadena de favores humanos en una **tubería confiable** que se ejecuta igual el lunes a las 8:00 que el sábado a las 23:00. Y nadie tiene que recordar nada.
::/interioriza

## Anatomía de un workflow

Un workflow n8n tiene:
- **Trigger** — qué arranca el workflow (webhook, schedule, app event).
- **Nodos** — cada paso del proceso (HTTP, transformación, AI, integración).
- **Conexiones** — flechas entre nodos (lineales, ramificadas, mergeadas).
- **Datos** — los items que viajan; cada nodo los recibe, transforma y emite.

```
[Webhook trigger] → [HTTP request a Claude] → [Switch por categoría]
                                                  ├── [Notion: crear ticket]
                                                  ├── [Gmail: enviar respuesta]
                                                  └── [Slack: notificar]
```

Cada nodo se configura en panel lateral. Pruebas en vivo con datos de ejemplo. Cuando funciona, lo activas y queda escuchando.

## Setup self-hosted en 10 minutos

```bash
# Pre-requisitos: Docker
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  docker.n8n.io/n8nio/n8n
```

Abres `http://localhost:5678`, creas usuario admin, listo. Puedes self-hostear en una raspberry pi, en tu servidor de la institución, o en una VPS de $5/mes.

Si prefieres cloud sin instalar: n8n Cloud desde $20/mes.

## Workflow ejemplo: solicitud de inscripción del case study

```
1. [Webhook] /webhooks/inscripcion
   Recibe POST con: {nombre, email, mensaje}

2. [Anthropic Chat] Clasificar mensaje
   Prompt: "Clasifica en: inscripcion, info, otro"
   Output: {categoria}

3. [IF categoria == "inscripcion"] sí ↓ no →[Slack: revisar humano]

4. [Notion: Database append]
   Crea fila en BD "Solicitudes de admisión"
   {nombre, email, mensaje, fecha, status: "pendiente"}

5. [Gmail: Send email]
   Plantilla: "Bienvenida a [nombre]. Tu solicitud quedó registrada con folio X. Te contactaremos en 48h."

6. [Google Calendar: Create event]
   Agenda entrevista en slot disponible.

7. [Slack: Post to channel]
   "📥 Nueva solicitud de inscripción: [nombre]. Folio X."

8. [Respond to Webhook]
   Devuelve {status: "ok", folio: X} al sistema que llamó.
```

Tiempo de construcción: 90 minutos. Tiempo ahorrado por solicitud: 30 minutos. ROI a la 4ª solicitud.

## Patrón "Workflow + RAG" (la combinación de oro)

Un workflow puede llamar a tu RAG (Unidad 4) como herramienta:

```
[Webhook recibido]
   ↓
[HTTP request al endpoint Dify de tu RAG]
   ↓
[Recibe respuesta con citas]
   ↓
[IF necesita acción] → [serie de nodos de acción]
[ELSE] → [Solo enviar respuesta]
```

n8n orquesta; el RAG entrega contexto. Cada uno hace lo que mejor hace.

## Buenas prácticas de n8n profesional

1. **Nombrar workflows** con prefijo: `[Asistente] Inscripción nueva`, `[Reportes] Resumen semanal`.
2. **Usar credenciales centralizadas** en n8n (no hardcodear API keys en nodos).
3. **Logging por workflow**: nodo "Workflow Logs" al inicio y al final.
4. **Manejo de errores**: nodo "Error Trigger" en cada workflow para capturar fallos a Slack/email.
5. **Versionar exportando JSON**: `Export workflow → guardar en Git`. Revertir en segundos.
6. **Test data**: cada nodo guarda último input/output, fácil debug.

## Anti-patrones

- **Un solo workflow gigante de 50 nodos.** Difícil debug. Parte en sub-workflows reutilizables.
- **Hardcodear datos de prueba.** Cuando va a producción, falla. Usa variables de entorno.
- **Sin retry policy.** Una API que falla 1 % rompe tu workflow. Configura retry 3 veces.
- **Sin manejo de duplicados.** El mismo webhook llega 2 veces y creas 2 tickets. Usa idempotency keys.
- **Sin monitoreo.** No sabes si el workflow corre. Activa Workflow Statistics o conecta a tu dashboard.

## Costo realista

- **Self-hosted**: ~$5/mes VPS DigitalOcean + APIs LLM ($30-100/mes según volumen).
- **Cloud n8n**: $20-100/mes según ejecuciones.
- **Setup time inicial**: 8-12 horas (curva).
- **Mantenimiento**: 1-2 horas/mes para una organización pequeña.

ROI típico para automatización de inscripciones (40 h/mes humanas a $200 = $8 000 ahorrados/mes contra $50/mes infrastructure): muy positivo.
