# 6.3 Tipos: reactivos, deliberativos, multi-agente

No hay un tipo único de agente. Hay arquitecturas que sirven a problemas distintos. Para tu Asistente Institucional, necesitas saber cuál elegir y cuándo escalar a la siguiente.

::concepto{titulo="Tipologías de agentes"}
Las 4 arquitecturas más usadas en 2025:

1. **Reactivos** — un solo agente con loop simple Thought-Action-Observation.
2. **Deliberativos / Plan-and-execute** — primero planifica todo, luego ejecuta.
3. **Jerárquicos** — un agente supervisor que delega a sub-agentes especializados.
4. **Multi-agente colaborativos** — varios agentes conversan entre sí para resolver.

A más complejidad, más capacidad pero más costo, latencia y riesgo. La regla: empieza con el más simple que funciona.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Igual que no contratas a un equipo de 10 personas para regar tus plantas, no diseñas un sistema multi-agente para responder dudas básicas. Conocer los tipos te permite no sobrediseñar — error caro de novatos en agentes.
::/interioriza

## Tipo 1 — Reactivo

```
[Objetivo]
   ↓ loop
   Pensar → Acción → Observación → Pensar → Acción → … → Respuesta
```

**Cuándo:** tareas con espacio de búsqueda pequeño, decisiones simples, recursos limitados.

**Caso del Asistente:** clasificar solicitud + invocar 1-2 tools (RAG + crear ticket).

**Ventajas:** simple, barato, latencia baja, fácil de debuggear.

**Limitaciones:** sin "vista global", puede deshacer pasos previos por no recordar.

## Tipo 2 — Deliberativo / Plan-and-execute

```
[Objetivo]
   ↓
[Planificar] — modelo genera lista de pasos
   ↓
[Ejecutar paso 1] → observar → ¿plan sigue válido?
   ↓
[Ejecutar paso 2] → observar → …
   ↓
[Respuesta final]
```

**Cuándo:** tareas con varios pasos interdependientes, dominio bien delimitado.

**Caso del Asistente:** procesar cambio de turno (5-7 pasos secuenciales, cada uno depende del anterior).

**Ventajas:** vista global, menos sorpresas, fácil mostrar plan al usuario antes de ejecutar.

**Limitaciones:** plan puede quedar obsoleto si el mundo cambia entre pasos.

## Tipo 3 — Jerárquico

```
[Supervisor]
   ↓ delega
   ├── [Sub-agente A: investigador]
   ├── [Sub-agente B: redactor]
   └── [Sub-agente C: validador]
   ↓ resultados
[Supervisor sintetiza]
```

**Cuándo:** problema complejo con sub-tareas claramente separables, dominios distintos.

**Caso del Asistente:** generar reporte trimestral (sub-agente investiga, sub-agente redacta, sub-agente valida factualidad).

**Ventajas:** especialización, paralelismo posible, modular.

**Limitaciones:** más caro (cada sub-agente cuesta), comunicación supervisor-subagente puede fallar.

## Tipo 4 — Multi-agente colaborativo

```
[Agente Investigador] ↔ [Agente Crítico] ↔ [Agente Editor]
              ↓ conversación
        [Producto final]
```

**Cuándo:** tareas creativas, debate entre roles, problemas donde varias perspectivas mejoran resultado.

**Caso del Asistente:** redactar comunicado políticamente sensible (un agente redacta, otro lo critica como padre molesto, un tercero como abogado, iteran).

**Plataformas:** AutoGen (Microsoft), CrewAI, LangGraph multi-agent.

**Ventajas:** calidad alta en tareas complejas, modeling de roles humanos.

**Limitaciones:** muy caro, latencia alta, debug pesadilla.

## Comparativo en una tabla

| Dimensión | Reactivo | Deliberativo | Jerárquico | Multi-agente |
|---|---|---|---|---|
| Complejidad de implementación | baja | media | alta | muy alta |
| Costo por ejecución | $ | $$ | $$$ | $$$$ |
| Latencia | baja | media | alta | muy alta |
| Calidad en tareas simples | alta | media | media | baja |
| Calidad en tareas complejas | baja | alta | muy alta | excelente |
| Debuggabilidad | alta | media | baja | muy baja |

## Recomendación de evolución para tu Asistente

1. **Mes 1-2** — agentes reactivos para casos comunes (FAQ, clasificación, ticket simple).
2. **Mes 3-4** — deliberativos para procesos multi-paso (cambio de turno, beca completa).
3. **Mes 6-9** — un jerárquico para reportes y análisis (si justifica costo).
4. **Mes 12+** — multi-agente solo si tienes caso muy especial (raro institucional).

90 % de los Asistentes Institucionales viven cómodos en niveles 1-2.

## Anti-patrones

- **Multi-agente para tareas simples.** Sobreingeniería, gasta 10× sin ganar nada.
- **Reactivo para procesos largos.** Olvida pasos, repite, falla.
- **Jerárquico mal especializado.** Si los sub-agentes hacen lo mismo, no es jerarquía: es duplicación.
- **No medir si el upgrade vale.** Cambias de reactivo a deliberativo "porque es más cool" sin métricas → pagas más sin ganar nada.
