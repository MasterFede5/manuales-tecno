# 3.4 Mini-apps con Artifacts (React, HTML, p5.js)

Tu institución no necesita "el siguiente Spotify": necesita **20 mini-apps cotidianas** que hoy se hacen a mano: cuestionarios, calculadoras, simuladores, formularios, dashboards. Esta lección te enseña a producir cada una en menos de 30 minutos sin tocar `npm install`.

::concepto{titulo="Mini-app como Artifact"}
Una **mini-app como Artifact** es una pieza de software autocontenida (≤200 KB) que vive dentro de un Artifact, ejecuta lógica interactiva en el navegador del usuario y se construye conversacionalmente. Soporta tres familias técnicas:

- **HTML/CSS/JS estático** — formularios, landings, cuestionarios sin estado complejo.
- **React + Tailwind + libs curadas** — dashboards, herramientas con estado, visualizaciones interactivas.
- **p5.js** — simulaciones físicas, juegos sencillos, arte generativo, demos educativas.

Lo que las distingue de "una página web normal": **se generan, iteran y publican sin salir del chat**.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Antes de Artifacts, una mini-app educativa requería: idea → wireframe → setup → desarrollo → deploy → 4 horas mínimo. Hoy: idea → prompt estructurado → 25 minutos. Esa diferencia te permite construir cosas que **nunca habrías intentado** porque "no valían el esfuerzo". Tu Asistente puede tener una calculadora de cupos, un simulador de horarios y un cuestionario de orientación vocacional sin que tu equipo deje de respirar.
::/interioriza

## Mini-app 1: dashboard React (la más común)

Stack: React + Tailwind + recharts + lucide-react.

```
Crea un Artifact React con dashboard de métricas para [tu caso].

ESTRUCTURA:
- Header con título y selector de periodo (semana / mes / trimestre).
- Grid de 4 métric-cards (cada una: ícono lucide, número grande, etiqueta, sparkline mini).
- Tabla con columnas ordenables.
- Tab opcional "Detalle" con gráfica recharts (line o bar).

ESTADO:
- Periodo seleccionado controla los datos mostrados.
- Búsqueda en tabla filtra en tiempo real.

DATOS DE EJEMPLO: incluye objeto JS con 12 puntos por métrica.

ESTILO: Tailwind, paleta [tus colores], shadows-sm, espaciado generoso.

RESTRICCIONES: mobile-first, sin libs externas a las soportadas.
```

Producción típica: 25 minutos con 3-5 iteraciones de pulido.

## Mini-app 2: calculadora con lógica de negocio

Stack: HTML + CSS + JS plano (más simple, mismo poder).

Caso: "calculadora de cupos disponibles según escenarios de admisión".

```
Crea un Artifact HTML+JS con una calculadora.

INPUTS (sliders + inputs numéricos):
- Cupos totales por grado (1-100).
- Tasa de retención esperada (50-100%).
- Solicitudes nuevas estimadas (0-200).

CÁLCULO:
- Cupos disponibles = totales - (totales × retención) + bajas estimadas.
- Mostrar también "porcentaje de demanda no cubierta".

OUTPUT:
- Número grande con resultado.
- Indicador semáforo según rango.
- Mini-chart de cómo varía si cambias retención ±5%.

ESTILO: minimalista, fondo claro, números prominentes.
```

## Mini-app 3: simulador p5.js

Stack: p5.js (incluido en Claude Artifacts).

Caso: simulador de "trayectoria de un comunicado" mostrando cómo se distribuye un mensaje desde la coordinación hasta cada papá.

```
Crea un Artifact con p5.js que simule la propagación de un mensaje.

ESCENA:
- Canvas 600×400.
- Nodo central "Coordinación" en la izquierda.
- 30 nodos circulares "padres" distribuidos en grid a la derecha.
- Líneas que se animan desde Coordinación a cada padre con retardo random.
- Cada padre cambia de color cuando "recibe" el mensaje.

CONTROL:
- Botón "Enviar comunicado" reinicia animación.
- Slider para velocidad de propagación.
- Contador de "% recibido".

PROPÓSITO PEDAGÓGICO: visualizar que un comunicado no llega instantáneo a todos.
```

p5.js es ideal para visualizaciones, simulaciones y arte generativo. No para apps de productividad.

## Mini-app 4: formulario inteligente

Stack: React + react-hook-form (no soportado nativamente, pero useState basta).

Caso: pre-formulario de inscripciones que valida antes de enviar.

```
Crea un Artifact React con un formulario de pre-inscripción de 5 pasos.

PASOS:
1) Datos del estudiante (nombre, fecha nac., grado deseado).
2) Datos del tutor (nombre, contacto, parentesco).
3) Antecedentes académicos (escuela previa, promedio).
4) Documentos (checklist de archivos a entregar).
5) Confirmación con resumen.

VALIDACIÓN:
- Campos obligatorios marcados con *.
- Validación en cliente: edad coherente con grado, email formato, teléfono 10 dígitos.

UX:
- Barra de progreso arriba.
- Botones Anterior/Siguiente.
- Al final: descargar resumen como PDF (usa window.print).

ESTILO: corporativo, paleta Albatros, accesible.
```

## Mini-app 5: cuestionario / quiz educativo

Caso: examen de práctica del Asistente para autoevaluación.

```
Crea un Artifact React con quiz de 10 preguntas opción múltiple sobre [tema].

LÓGICA:
- Una pregunta visible a la vez.
- Botones de opciones (a, b, c, d).
- Al responder, mostrar feedback inmediato (correcta / incorrecta + explicación).
- Al final: score, tiempo total, gráfica de respuestas correctas/incorrectas.

PERSONALIZACIÓN:
- Preguntas como prop / array editable en el código.
- Posibilidad de mezclar orden cada vez (Math.random).

ESTILO: limpio, lectura cómoda, feedback con animación suave.
```

## Mini-app 6: generador de algo (template engine)

Caso: generador de comunicados a partir de plantilla con variables.

```
Crea un Artifact React tipo "template generator".

INPUTS:
- Selector de tipo de comunicado (suspensión, evento, recordatorio).
- Inputs por tipo (fecha, motivo, alcance, contacto).
- Botón "Generar".

OUTPUT:
- Texto del comunicado en una caja editable.
- Botón "Copiar al portapapeles".
- Botón "Exportar PDF" (window.print con print.css).

LÓGICA:
- Template strings que combinan inputs según tipo.
- 3 plantillas pre-cargadas que el usuario puede modificar.
```

## Patrón general: descomposición progresiva

No pidas la mini-app entera de un golpe. Descompón:

1. **Layout vacío** — estructura visual sin lógica. 5 minutos.
2. **Datos de ejemplo** — agrega contenido falso pero realista. 5 minutos.
3. **Lógica básica** — interacciones principales. 10 minutos.
4. **Estados de borde** — vacío, error, loading. 5 minutos.
5. **Polish** — accesibilidad, mobile, microinteracciones. 5 minutos.

Cada paso es una iteración pequeña. Si pides los 5 juntos, alguno saldrá flojo.

## Cuándo migrar de Artifact a app real

Tu mini-app pasó de demo a herramienta usada diariamente. Señales para migrar:

- Necesitas autenticación real.
- Necesitas datos persistentes entre sesiones.
- Más de 20 usuarios concurrentes.
- Integración con APIs externas (cuyo CORS bloqueas en sandbox).
- Auditoría legal del código.

La migración: descargas el código del Artifact, lo metes a un proyecto React real (Vite, Next), agregas auth (Clerk, Auth0), DB (Supabase, Firebase) y deploy en Vercel. Una tarde de trabajo.
