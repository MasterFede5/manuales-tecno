# 3.3 ChatGPT Canvas y Gemini Canvas

Tu equipo usa Claude para algunos módulos y ChatGPT para otros. Tu directora prefiere Gemini porque ya paga Google Workspace. Si solo conoces Claude Artifacts, vas a tener tres equipos hablando lenguajes distintos sobre el mismo dashboard. Este subtema te equipa para los tres.

::concepto{titulo="ChatGPT Canvas y Gemini Canvas"}
**ChatGPT Canvas** (OpenAI) y **Gemini Canvas** (Google) son las contrapartes de Claude Artifacts en sus respectivos ecosistemas. Ambos abren un panel separado del chat para alojar contenido editable colaborativamente, con diferencias significativas en tipos soportados, ejecución de código y modelo de publicación.

Al cierre de este manual (2025), los tres jugadores convergen en concepto y divergen en detalle. La elección depende menos del Artifact en sí y más de **dónde ya vive tu equipo**.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Cuando una empresa elige Slack vs Teams vs Discord no decide solo por feature: decide por **dónde está su gente**. Lo mismo aquí: si tu institución ya paga ChatGPT Enterprise, tiene sentido invertir en Canvas; si ya paga Google Workspace + Gemini, en Gemini Canvas. Cambiar de plataforma por features menores rara vez vale la pena.
::/interioriza

## ChatGPT Canvas en una página

**Acceso:** ChatGPT Plus o Team o Enterprise. Disponible en la web y en la app desktop.

**Tipos soportados:**
- Texto largo (su fortaleza original).
- Código (con ejecución sandbox para Python).
- HTML/CSS/JS estático.
- React (limitado, recientemente expandido).

**Diferenciadores:**
- **Edición tipo Word** — selecciona texto y pide al modelo cambios solo en esa parte. Muy fluido para textos largos.
- **Code interpreter integrado** — para Python, ejecuta y muestra outputs directamente en el Canvas (perfil estadístico, gráficas).
- **Sugerencias proactivas** — el modelo te ofrece "shorter / longer / more formal / add emoji" como botones contextuales.
- **Compartir como link público** vía botón Share.

**Limitaciones:**
- React menos pulido que en Claude.
- Versionado más opaco (sí guarda historia, pero menos navegable).
- Sin Mermaid nativo (puedes pedir SVG, pero no se renderiza Mermaid como diagrama vivo).

**Cuándo brilla:**
- Documentos largos colaborativos (ensayos, propuestas, drafts de PRD).
- Análisis de datos con Python (scripts que generan gráficas).
- Equipos que ya usan ChatGPT y necesitan menos fricción.

## Gemini Canvas en una página

**Acceso:** Gemini Advanced (con Google One AI Premium) o Workspace con Gemini Business / Enterprise.

**Tipos soportados:**
- Texto largo.
- Código.
- HTML/CSS estático con preview.
- Diagramas (con apoyo de Google Charts/D3).

**Diferenciadores:**
- **Integración nativa con Google Workspace** — exporta directo a Docs, Slides, Sheets en un click.
- **Multilingüe extra fluido** — particularmente bueno para textos en español académico.
- **Multimodal** — puedes pasarle imágenes o PDFs en el contexto y el Canvas los integra.
- **Deep Research** que produce reportes largos directamente en Canvas (cubierto en U9).

**Limitaciones:**
- React/JS interactivo menos potente que Claude.
- Publicación pública menos directa (orientado a compartir dentro de tu org Workspace).
- Documentación menos clara para developers que la de Anthropic.

**Cuándo brilla:**
- Organizaciones 100 % Google Workspace.
- Reportes de investigación largos con citas (Deep Research).
- Educación con contenido multimodal.

## Patrón "multi-canvas" para tu Asistente

En la práctica, los equipos serios **no eligen uno**: usan los tres para distintas tareas.

| Tarea del Asistente | Canvas recomendado |
|---|---|
| Dashboard React con sliders e interactividad | Claude Artifacts |
| Documento técnico largo en español académico | ChatGPT Canvas |
| Reporte de investigación de mercado | Gemini Canvas (Deep Research) |
| Mockup HTML/CSS estático | Cualquiera |
| Diagrama de arquitectura del sistema | Claude Artifacts (Mermaid) |
| Análisis estadístico con gráficas Python | ChatGPT Canvas (Code Interpreter) |

## Diferencias en el primer prompt

El primer prompt cambia ligeramente según el proveedor:

- **Claude:** funciona mejor con XML estructurado y descripciones técnicas precisas.
- **ChatGPT:** funciona mejor con instrucciones conversacionales y referencias a "haz algo similar a X".
- **Gemini:** funciona mejor con contexto multimodal (incluir captura, PDF, ejemplo).

Adapta tu prompt al proveedor para resultados óptimos. Lo aprendiste en U1 (1.6 few-shot estratégico): los ejemplos correctos importan.

## Anti-patrones cross-canvas

- **Usar Claude para análisis Python intenso.** Funciona, pero sin code interpreter integrado lo verás solo como código.
- **Usar ChatGPT para Mermaid complejo.** Funciona, pero el rendering es menos pulido.
- **Usar Gemini para mini-app React puro.** Es el menos sólido en interactividad.
- **Cambiar de proveedor a media iteración.** Pierdes versiones; la migración manual cuesta.
