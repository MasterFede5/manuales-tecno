# 3.1 ¿Qué son los Artifacts? Tipos

Le pides al modelo "explícame regresión lineal" y te entrega 600 palabras. Útil. Le pides "muéstrame regresión lineal con un slider que mueva la pendiente" y te entrega una **mini-app interactiva** en el panel lateral. Esa diferencia se llama Artifact.

::concepto{titulo="Artifact / Canvas (concepto)"}
Un **Artifact** (Claude) o **Canvas** (ChatGPT, Gemini) es un **panel separado del chat**, generado por la IA, que aloja contenido **manipulable, ejecutable o iterable**: documentos largos, código, mini-apps web, diagramas, SVGs. Vive lado a lado con la conversación: el chat lo crea, lo edita y lo refina; el artifact lo muestra, lo ejecuta y lo conserva.

La diferencia con una respuesta normal:
- **Respuesta normal** → texto inline en el chat, efímero, difícil de iterar.
- **Artifact** → panel persistente, editable por el modelo a tu pedido, exportable y embebible.

Los Artifacts son la pieza que convierte un chatbot en una **superficie de creación**.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Cuando un arquitecto explica una casa, no la describe en párrafos: te muestra **planos**. Los planos son artifacts. La conversación es contexto, los planos son el entregable. Tu Asistente sin artifacts es un arquitecto que solo te describe la casa; con artifacts, te entrega los planos imprimibles.
::/interioriza

## Los 8 tipos canónicos

| Tipo | Qué es | Ejemplo en tu Asistente | Mejor proveedor |
|---|---|---|---|
| **Texto largo** | Documento >800 palabras editable | PRD, política de uso, propuesta a dirección | Claude Artifacts, ChatGPT Canvas |
| **Código** | Bloques de código ejecutables o listos para copiar | Script `evals/runner.py`, esquema JSON | Cualquiera |
| **HTML/CSS** | Página estática lista para ver | Mockup landing del Asistente | Claude Artifacts (renderiza en vivo) |
| **React** | Componente interactivo con estado | Dashboard con sliders, formularios, gráficas | Claude Artifacts (rendering nativo) |
| **Mermaid** | Diagramas declarativos | Arquitectura del Asistente, flujos n8n | Claude Artifacts, Notion AI |
| **SVG** | Gráfico vectorial editable | Logo experimental, infografías | Claude Artifacts, ChatGPT Canvas |
| **Markdown formateado** | Texto con renderizado rico | Manuales, READMEs, posts | Cualquiera |
| **JSON / YAML** | Datos estructurados | Schema de output, config de prompt | Cualquiera |

## Ciclo de vida de un Artifact

1. **Creación** — pides algo y el modelo decide que merece artifact (>15 líneas, >800 palabras, código ejecutable, contenido visual).
2. **Iteración** — pides cambios en el chat (*"haz el botón rojo y agrega un input"*) y el modelo edita el artifact en su lugar.
3. **Versionado** — cada cambio queda como versión navegable (en Claude verás v1, v2, v3 con flechas).
4. **Compartir** — botón de "publish" genera URL pública.
5. **Embeber o exportar** — copias HTML, descargas, integras en sitio.

## Cuándo SÍ usar Artifact

- Texto >15 líneas que vas a editar en pasadas.
- Código que el usuario quiere copiar o ejecutar.
- Cualquier cosa con HTML/SVG/diagrama.
- Mini-app interactiva (slider, formulario, gráfica).
- Pieza que pretendes compartir con alguien que no está en el chat.

## Cuándo NO usar Artifact

- Respuesta corta (<3 párrafos).
- Conversación exploratoria sin entregable.
- Información que cambiará 30 veces sin estabilizarse.
- Datos sensibles que no deberían tener URL pública.

El modelo a veces decide mal y crea Artifact donde no hace falta. Tú puedes decir *"no uses Artifact, respóndeme inline"* y obedece.

## Limitaciones reales (lo que no te dicen los marketing)

- **Sin acceso a internet en tiempo de ejecución del Artifact.** El React que el modelo genera corre en sandbox sin fetch externo (excepto a APIs explícitamente permitidas por el proveedor).
- **Bibliotecas limitadas.** Claude Artifacts soporta React, Tailwind y un set curado de libs (recharts, lucide). No instalas npm.
- **Estado efímero.** El artifact no persiste estado entre cargas; cada visita arranca desde cero salvo que uses localStorage o pases datos por URL.
- **Tamaño máximo.** Claude Artifacts ~150KB de código; ChatGPT Canvas similar. Excedido, el modelo lo trocea o pierde fluidez.
- **Sin backend nativo.** Para datos persistentes y APIs reales necesitas U5 (workflows) o U6 (agentes).

## Caso aplicado: dashboard del Asistente

Tu Episodio 3 pide un dashboard. Decisiones de tipo:

- **Resumen de métricas** → React con `recharts` para gráficas + cards.
- **Estado de módulos** → tabla HTML con badges de status.
- **Bitácora de decisiones (ADRs)** → Markdown formateado con buscador.
- **Diagrama de arquitectura** → Mermaid embebido.

Todo en **un solo Artifact React** con tabs. El usuario abre el link, ve el dashboard sin instalar nada.
