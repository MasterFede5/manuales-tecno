# 8.3 Claude Desktop con MCP

Claude Desktop es el cliente MCP **referencia** porque Anthropic creó el estándar y la app lo lleva integrado nativamente. Esta lección te enseña a configurarlo y operarlo en producción institucional.

::concepto{titulo="Claude Desktop"}
**Claude Desktop** es la aplicación oficial de Anthropic para macOS y Windows que ofrece interfaz tipo ChatGPT con la diferencia clave de **integración nativa MCP**. Cualquier servidor MCP que configures aparece como tools disponibles para Claude en cada conversación.

Características relevantes:
- **MCP nativo** desde noviembre 2024.
- **Multi-server**: configura N servidores y todos están disponibles.
- **Indicador de tools** en la UI mostrando qué servidores están activos.
- **Approval visual** antes de ejecutar acciones críticas.
- **Logs de tool calls** auditables.
- **Sandbox**: los servidores corren como subprocesos aislados.

Es la diferencia entre Claude.ai (web) y Claude Desktop: el segundo tiene MCP, el primero no.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Si tu equipo usa Claude.ai en navegador, está funcionalmente "ciego" a tus archivos locales y sistemas internos. Claude Desktop con MCP convierte la misma conversación en una **sesión que puede leer, escribir, buscar y actuar** sobre tu mundo institucional. Es la migración del cliente que cambia las reglas.
::/interioriza

## Instalación y configuración

### macOS / Windows

1. Descargar de `claude.ai/download`.
2. Instalar.
3. Login con cuenta Claude (Pro recomendado para aprovechar Opus 4.7).
4. Editar archivo de config:
   - **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`.
   - **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`.

### Config inicial recomendada

```json
{
  "mcpServers": {
    "filesystem-albatros": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/me/AlbatrosDocs"
      ]
    },
    "memory": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"]
    },
    "time": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-time"]
    }
  }
}
```

Cierra y reabre Claude Desktop. En la barra inferior aparece icono de tools indicando "3 MCP servers connected".

## Tu primer prompt con MCP

Conversación de ejemplo:

```
Tú: ¿Qué archivos tengo en AlbatrosDocs sobre "becas"?

Claude (invoca filesystem.search_files): "encontré:
  - reglamento_becas_2026.pdf
  - politica_becas_hermanos.md
  - solicitud_beca_form.docx"

Tú: Resume el reglamento de becas.

Claude (invoca filesystem.read_file): "Leo el contenido...
  Resumen: ..."

Tú: Recuerda que los hermanos tienen 15% de descuento adicional.

Claude (invoca memory.create_entity): "Anotado en memoria.
  Entidad: política_becas_hermanos
  Observación: 15% descuento adicional para hermanos."
```

Sin programar nada, ya tienes Asistente con acceso a archivos y memoria persistente.

## Configuración avanzada institucional

```json
{
  "mcpServers": {
    "fs-publico": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem",
               "/Users/me/AlbatrosDocs/publico"]
    },
    "fs-confidencial": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem",
               "/Users/me/AlbatrosDocs/confidencial"]
    },
    "github-asistente": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {"GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxxxx"}
    },
    "slack-bot": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-slack"],
      "env": {
        "SLACK_BOT_TOKEN": "xoxb-xxxx",
        "SLACK_TEAM_ID": "T01XXXXXX"
      }
    },
    "postgres-readonly": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres",
               "postgresql://readonly:xxx@db.albatros.local/asistente"]
    },
    "memory-asistente": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"]
    }
  }
}
```

6 servidores conectados. Tu Claude Desktop ahora tiene acceso a todo el ecosistema institucional.

## Approval flow

Por seguridad, Claude Desktop te pregunta antes de ejecutar acciones de escritura/destructivas:

```
[Claude quiere ejecutar:]
  filesystem.write_file(
    path="docs/borrador.md",
    content="..."
  )
  
[ Aprobar ] [ Rechazar ]
```

Las acciones de lectura suelen ser auto-aprobadas; las de escritura requieren tu OK.

## Distribución a tu equipo

Cómo dejas Claude Desktop con MCP listo para tu coordinadora académica:

1. Crea un script de setup que:
   - Instala Claude Desktop.
   - Genera el `claude_desktop_config.json` con servidores institucionales.
   - Configura tokens (idealmente desde env o keychain del sistema).

2. Documenta:
   - Qué servidores tiene activos.
   - Qué tools puede invocar.
   - Política de aprobaciones.

3. Capacita en una sesión de 30 min: cómo conversar pidiendo cosas concretas que invocan MCP.

## Anti-patrones

- **Tokens hardcoded en config.** Si compartes el config sin pensar, filtras tokens. Usa env vars o secret manager.
- **Servidores sin restricciones de path.** filesystem con `/` (raíz) = Claude puede leer todo el disco. Restringe.
- **Activar 15 servidores.** El modelo se confunde con tools. Cura.
- **Sin logging.** Activa logs para auditar uso, especialmente al inicio.
- **Asumir que Claude Desktop es para todos.** No: es para usuarios power. Coordinadora académica probablemente prefiera Open WebUI con tools predefinidos.
