# 8.4 ChatGPT con conectores empresariales

OpenAI llegó tarde a MCP pero llegó. Y mientras lo abraza progresivamente, ChatGPT Team y Enterprise ofrecen **Connectors** propios que cumplen función similar para grandes ecosistemas (Google Workspace, Microsoft 365, Box, Salesforce).

::concepto{titulo="ChatGPT Connectors y soporte MCP"}
**Connectors** son integraciones nativas de ChatGPT (Plus, Team, Enterprise) que permiten al modelo acceder a servicios empresariales con autenticación OAuth corporativa, con UI guiada y sin configurar archivos JSON. En 2025, OpenAI también añadió **soporte MCP (limitado)** en Tools, alineándose con el estándar.

Diferencia con Claude Desktop+MCP:
- **Connectors** son servicios first-party de OpenAI: están preconfigurados, autenticados con OAuth, con UI integrada.
- **MCP en ChatGPT** es más limitado al cierre de este manual; soporta servidores que sigan reglas específicas, pero el ecosistema es menos maduro que en Claude.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Si tu institución ya está parada en el ecosistema OpenAI (paga ChatGPT Team o Enterprise), forzar Claude Desktop solo por MCP puede no ser óptimo. Los Connectors de ChatGPT cubren los casos mainstream con autenticación corporativa que no requiere intervención técnica de tu equipo.
::/interioriza

## Connectors disponibles (2025)

| Connector | Auth | Caso típico |
|---|---|---|
| **Google Drive** | OAuth Google | leer documentos, hojas, presentaciones |
| **Google Calendar** | OAuth | revisar y crear eventos |
| **Microsoft 365 (OneDrive, Outlook, SharePoint)** | OAuth Microsoft | igual ecosistema MS |
| **Box** | OAuth | enterprise file storage |
| **Dropbox** | OAuth | files |
| **Salesforce** | OAuth | CRM |
| **GitHub** | OAuth | repos |
| **Notion** | OAuth | espacios |
| **Slack** | OAuth Slack | mensajes |
| **Zapier AI Actions** | API key | 6000+ apps via Zapier |

Cada connector se activa en ChatGPT Settings → Connected Apps. Login OAuth corporativo. Listo.

## Diferencias prácticas Claude Desktop vs ChatGPT

| Dimensión | Claude Desktop + MCP | ChatGPT + Connectors |
|---|---|---|
| **Filosofía** | abierto, configurable, MCP estándar | empaquetado, OAuth corporativo |
| **Setup** | editar config JSON | UI guiada |
| **Servidores custom** | trivial (escribes server) | requiere developer + Custom GPT/API |
| **Ecosistema MCP** | total | limitado |
| **Servicios empresariales pulidos** | dependen comunidad | first-party con calidad alta |
| **Audit logs** | tú los implementas | enterprise grade ya |
| **Costo** | Pro $20 + servers gratis | Plus/Team/Enterprise |
| **Privacidad** | datos pueden quedarse local con MCP local | siempre cloud OpenAI |

## Cuándo elegir Connectors de ChatGPT

- Institución 100 % comprometida con OpenAI.
- Workflow mainstream con apps "first-class" (Google Workspace, M365, Salesforce).
- Equipo no técnico que necesita UI guiada.
- Compliance enterprise (Connectors tienen audit logs, retention policies).

## Cuándo elegir Claude Desktop + MCP

- Necesitas servidores custom (sistema interno, BD propietaria).
- Quieres flexibilidad: el ecosistema MCP comunitario te da 100+ servidores que ChatGPT aún no tiene.
- Privacidad: muchos servidores MCP corren local; Connectors siempre pasan por OpenAI cloud.
- Apertura: si OpenAI cambia política, MCP te permite migrar a Cursor, Windsurf, Continue, etc.

## ChatGPT MCP (estado 2025)

OpenAI lanzó soporte MCP gradual en 2025. Diferencias:
- Más restrictivo en tipos de servidores soportados.
- Mejora en cada release.
- Para casos avanzados, Claude Desktop sigue siendo el cliente MCP más completo.

## Camino híbrido

Las instituciones serias **usan ambos**:
- ChatGPT con Connectors para tareas mainstream del staff (Drive, Calendar, Slack).
- Claude Desktop con MCP para tareas con sistemas internos custom.

Cada usuario tiene ambos clientes. El gasto se distribuye según uso real.

## Anti-patrones

- **Cancelar OpenAI por entusiasmo MCP.** Tus connectores Google Drive son útiles; no los pierdas sin razón.
- **Cancelar Claude por simpleza ChatGPT.** Sin Claude+MCP pierdes flexibilidad para servidores custom.
- **Configurar mismos OAuth tokens en ambos.** Replicas exposición. Auditoría confusa.
- **Confundir Connectors con MCP.** Son cosas distintas; el primero es de OpenAI, el segundo es estándar abierto.
