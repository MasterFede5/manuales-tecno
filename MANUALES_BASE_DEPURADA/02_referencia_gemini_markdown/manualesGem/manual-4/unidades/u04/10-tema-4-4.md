# 4.4 Plataformas no-code: Dify, Flowise, Stack AI

Custom GPT, Claude Projects y NotebookLM resuelven 80 % de los casos. ¿Y el 20 % restante? Quieres elegir el embedder, ajustar chunking, conectar a Notion, exponer una API propia, controlar costos. Bienvenido al mundo no-code técnico.

::concepto{titulo="Plataformas no-code para LLM apps"}
**Dify, Flowise y Stack AI** son plataformas que permiten construir aplicaciones LLM (chatbots, RAG, agentes, workflows) **arrastrando bloques visuales** en un canvas, sin escribir código pero con todo el control que da una orquestación profesional. Permiten:

- Elegir embedder, vector store, modelo generador independientemente.
- Definir pipeline de chunking, re-ranking, post-procesamiento.
- Conectar a fuentes (Notion, Google Drive, sitios web, APIs).
- Exponer API propia para integrar con tu sitio o app.
- Versionar y monitorear desde el dashboard.

Vives entre las dos orillas: ni Custom GPT (cero código pero cerrado) ni programación pura (LangChain).
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Si has manejado coches automáticos, sabes lo cómodos que son. Pero hay un momento en que la cuesta es muy empinada y necesitas cambiar manualmente — el automático no entiende el contexto. Las plataformas no-code te dan ese cambio manual: control quirúrgico cuando lo necesitas, comodidad el resto del tiempo.
::/interioriza

## Dify — la opción más madura para producción

**Web:** dify.ai · **Hosting:** cloud (saas) o self-hosted (Docker, Kubernetes).

**Fortalezas:**
- UI muy pulida, similar a Notion en cuanto a usabilidad.
- Soporte nativo a workflows complejos, agentes, RAG, multi-modelo.
- Self-hosted gratuito con licencia abierta para empresas pequeñas.
- Marketplace de plantillas listas (chatbot RH, atención al cliente, etc.).
- API REST para cada app que construyas.

**Cuándo elegir:**
- Producción a escala.
- Self-hosting por requisitos de privacidad.
- Equipo con 1 técnico capaz de manejar Docker.

**Cuándo NO:**
- Solo necesitas chatbot simple → usa NotebookLM o Custom GPT.
- No tienes capacidad de DevOps mínima.

## Flowise — open-source, ultra-flexible

**Web:** flowiseai.com · **Hosting:** principalmente self-hosted (npm install).

**Fortalezas:**
- Open source genuino (Apache 2.0).
- Construido sobre LangChain.js → todos los componentes que existen en LangChain están disponibles como nodos.
- Comunidad muy activa con plantillas en GitHub.
- Customización a nivel de código JavaScript dentro de cada nodo.

**Cuándo elegir:**
- Quieres open source puro.
- Tienes equipo técnico cómodo con Node.js.
- Necesitas integraciones obscure que solo LangChain soporta.

**Cuándo NO:**
- No quieres mantener servidor.
- UI menos pulida que Dify.

## Stack AI — enterprise con énfasis en cumplimiento

**Web:** stack-ai.com · **Hosting:** cloud principalmente; self-hosted para Enterprise.

**Fortalezas:**
- Diseño orientado a empresas (compliance, audit logs, role-based access).
- Integraciones nativas con Salesforce, ServiceNow, SharePoint, Snowflake.
- Servicio de implementación incluido en planes Enterprise.
- Builders especializados (chat, agente, automatización, knowledge base).

**Cuándo elegir:**
- Empresa grande con requisitos legales (banca, salud, gobierno).
- Necesitas integración con sistemas corporativos legacy.
- Tienes presupuesto para servicio profesional.

**Cuándo NO:**
- Equipos pequeños o startups (sobredimensionado).
- Pruebas iniciales (planes empiezan en cifras 4 dígitos).

## Patrón típico de pipeline RAG en cualquiera de las tres

Los tres exponen el mismo flujo a alto nivel:

```
[Input usuario]
   ↓
[Pre-processing: limpieza, idioma, intención]
   ↓
[Vector store query → top-k chunks]
   ↓
[Optional: re-ranking con modelo dedicado]
   ↓
[Augmented prompt construction]
   ↓
[LLM generation]
   ↓
[Post-processing: citas, formato, validación schema]
   ↓
[Output usuario]
```

Cada bloque es un nodo que arrastras al canvas. Configuras parámetros en panel lateral. Pruebas en debugger inline. Despliegas con un click.

## Receta concreta: setup en Dify (45 minutos)

1. **5 min** — Instala Dify self-hosted con Docker (`docker compose up -d`) o crea cuenta SaaS.
2. **5 min** — Configura proveedores: tus API keys de OpenAI/Claude/Cohere.
3. **10 min** — Crea Knowledge Base: subes 12 PDFs institucionales, eliges embedder Cohere multilingual, chunking 500 con overlap 50.
4. **10 min** — Creas App tipo "Chatbot": eliges Knowledge Base, prompt de sistema, modelo generador (Sonnet 4.6).
5. **5 min** — Pruebas en debugger con 5 preguntas reales.
6. **5 min** — Publish: obtienes URL pública o widget para embeber + endpoint API.
7. **5 min** — Documentas y compartes.

## Anti-patrones

- **Empezar por Stack AI sin pasar por Dify.** Sobredimensionado para 90 % de los casos.
- **Self-host sin equipo de soporte.** Tarde o temprano se cae.
- **Probar 5 plataformas simultáneamente.** Pierde foco. Elige una y profundiza.
- **Skip de evaluación.** Aún en plataformas no-code, golden set es obligatorio (subtema 1.8).
