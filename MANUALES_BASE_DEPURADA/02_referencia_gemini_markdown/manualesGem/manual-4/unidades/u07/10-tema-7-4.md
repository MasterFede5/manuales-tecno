# 7.4 LM Studio y Jan

Ollama + Open WebUI es el stack institucional. Para usuarios individuales o equipos de 1-3 personas, hay opciones más simples: **LM Studio** y **Jan**, apps desktop con UI nativa.

::concepto{titulo="LM Studio y Jan"}
**LM Studio** es una aplicación desktop (Mac, Windows, Linux) que combina runtime + UI + biblioteca de modelos en un solo binario. Click-and-go.

**Jan** es similar, también desktop, con énfasis en privacidad y open source (Jan es totalmente AGPL-3.0; LM Studio tiene partes propietarias aunque uso es gratis).

Ambos son alternativas a Ollama+Open WebUI cuando:
- Usuario individual.
- No quieres Docker ni configuración de servidor.
- Prefieres app de escritorio nativa.
- No necesitas multi-usuario.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Tu coordinadora académica probablemente prefiere abrir una app que arrancar Docker. Para usuarios individuales del staff que quieren su propio chat local sin compartir, LM Studio o Jan son superiores en experiencia. Para producción institucional con 50 staff, Ollama+Open WebUI sigue ganando por el multi-user.
::/interioriza

## LM Studio

**Web:** lmstudio.ai · **Plataformas:** macOS, Windows, Linux.

**Pros:**
- Instalación 1-click sin Docker.
- Browser de modelos integrado con HuggingFace.
- Server local (puerto 1234) compatible OpenAI API.
- Ajuste de parámetros (temperature, top-k, etc.) con sliders visuales.
- Soporte GGUF nativo, optimización Apple Silicon excelente.

**Contras:**
- Cerrado parcialmente (no full open source).
- No multi-user.
- No knowledge base nativa (debes integrar via API).

**Cuándo:** developer o usuario power individual con Mac M1+.

## Jan

**Web:** jan.ai · **Plataformas:** macOS, Windows, Linux.

**Pros:**
- Open source genuino (AGPL-3.0).
- UI elegante.
- API local OpenAI-compatible.
- Soporte de Hugging Face directo.
- Privacidad por diseño (cero telemetría).

**Contras:**
- Comunidad más pequeña que Ollama.
- Menos modelos pre-curados.
- Sin multi-user.

**Cuándo:** usuario que valora open source absoluto y privacidad máxima individual.

## Cuándo elegir cada uno

- **Ollama + Open WebUI:** institución multi-user, servidor compartido, gobernanza. **Default Albatros.**
- **LM Studio:** profesional individual con Mac M1+ que quiere ajustes finos. Power user.
- **Jan:** activista de privacidad, dev experimentando, casos sensibles individuales.

## Qué tienen en común los 3

- Inferencia local 100 % (no telemetría a servers externos).
- API local compatible OpenAI (mismo código cliente).
- Soporte GGUF de Hugging Face.
- Cuantización Q4/Q5/Q8.
- Gratis para uso (LM Studio puede tener planes empresariales).

## Anti-patrones

- **Usar 3 stacks al mismo tiempo.** Si tu institución usa los 3, dispersión y confusión. Elige uno principal.
- **No medir consumo.** Una laptop con LM Studio consumiendo GPU al 100 % por horas se calienta y desgasta.
- **Compartir API local sin auth.** Si abres puerto 1234 a la red institucional sin auth, exposición.
