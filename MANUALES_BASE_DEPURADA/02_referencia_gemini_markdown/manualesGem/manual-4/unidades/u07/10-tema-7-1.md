# 7.1 ¿Por qué local? Privacidad, costo, control

Tres razones legítimas para correr tu Asistente local: **privacidad** (los datos no salen), **costo** (a volumen, sale más barato), **control** (no dependes de cambios del proveedor). Esta lección te ayuda a saber si **tu** caso aplica.

::concepto{titulo="IA local"}
**IA local** es ejecutar un modelo de lenguaje en infraestructura propia (una laptop, un servidor de la institución, una VM tuya), sin enviar datos a proveedores externos. Implica:

- **Modelo abierto** (Llama, Mistral, Qwen, etc.) descargado a tu disco.
- **Runtime** que ejecute el modelo (Ollama, LM Studio, vLLM, llama.cpp).
- **Hardware** suficiente (GPU, RAM, CPU según tamaño).
- **Mantenimiento** humano (actualizaciones, monitoreo, backups).

A cambio, ganas **soberanía total** sobre los datos.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Cuando una empresa adopta una herramienta cloud crítica, contrae una **dependencia inversa**: el proveedor decide precios, términos, disponibilidad. IA local invierte la relación. Para una institución educativa que maneja datos de menores, calificaciones y registros médicos, esa inversión puede valer su peso en oro — o ser sobreingeniería costosa para nada. Saber distinguir es lo que esta unidad construye.
::/interioriza

## Las 3 razones reales

### Privacidad y compliance

- Datos de menores (LFPDPPP México, GDPR UE, COPPA EE.UU.) bajo régimen estricto.
- Calificaciones, expedientes disciplinarios, datos de salud.
- Investigaciones donde la información es propietaria.
- Sectores regulados (salud, gobierno, defensa).

Con cloud: cumples si tu proveedor cumple. Con local: cumples tú directamente.

### Costo a volumen

- API cloud: por tokens; lineal con uso.
- Local: costo fijo (hardware + electricidad + mantenimiento).

Punto de cruce típico: ~50 millones de tokens/mes a Sonnet 4.6. Por debajo, cloud gana. Arriba, local empieza a empatar.

### Control

- No depender de actualizaciones de proveedor que cambian comportamiento del modelo (Anthropic actualiza Claude → tu sistema responde distinto).
- No depender de internet (zonas rurales, viajes, contingencia).
- Personalizar fine-tuning sin pagar premium.
- Auditar comportamiento del modelo línea por línea (modelos abiertos lo permiten).

## Comparativa cuantitativa

| Dimensión | Cloud | Local |
|---|---|---|
| **Setup inicial** | minutos | 1-3 horas |
| **Costo inicial** | $0 | $500-3000 (GPU) |
| **Costo por 1M tokens** | $3-30 | $0 (electricidad despreciable) |
| **Privacidad** | depende proveedor | total |
| **Latencia** | 200ms-2s | 100ms-30s (según hardware) |
| **Calidad** | excelente (modelos top) | buena-muy buena (Llama 70B se acerca) |
| **Internet** | requerido | no requerido |
| **Mantenimiento** | proveedor | tú |
| **Lock-in** | alto | nulo |

## Cuándo NO migrar a local

- Volumen bajo (<5M tokens/mes). Cloud es más simple y barato.
- Sin equipo TI capacitado en Linux/Docker básico.
- Hardware no disponible y sin presupuesto.
- Necesitas modelos top (GPT-5, Claude Opus 4.7, Gemini Ultra) — los modelos abiertos están detrás aún.
- Casos de uso que requieren tools propietarias (Computer Use, Operator).

## Cuándo SÍ migrar a local

- Datos sensibles de salud, menores, defensa, financieros.
- Volumen alto y constante (>50M tokens/mes).
- Requisitos de compliance específicos (gobierno mexicano, sector salud).
- Educación con datos de estudiantes que NO deben salir de la institución.
- Investigación con información propietaria.
- Disponibilidad sin internet (escuelas rurales).

## El patrón híbrido (lo más común)

La mayoría de instituciones serias **no eligen una**: usan ambas.

```
                ┌─ Local (Ollama + Llama 3.1 70B)
                │   - Clasificación
                │   - Datos sensibles
                │   - RAG sobre reglamento
                │
Asistente ──────┤
                │
                └─ Cloud (Claude Opus 4.7)
                    - Razonamiento complejo
                    - Tareas que justifiquen el costo
                    - Backup cuando local falla
```

Decides por **costo + sensibilidad de datos** caso por caso.

## El costo verdadero del "local"

No olvidar:
- **Hardware** ($500-3000 inicial, depreciación 3 años).
- **Electricidad** (~$30-100/mes para servidor con GPU).
- **Tiempo de TI** (~5-20 horas/mes setup + 2-5 horas/mes mantenimiento).
- **Espacio físico** (servidor en algún lugar).
- **Backup y redundancia** (un servidor solo es punto único de falla).
- **Actualizaciones de modelos** (descargar nuevas versiones).

Si no tienes equipo TI, multiplica todo por 2-3 (servicios de tercero).
