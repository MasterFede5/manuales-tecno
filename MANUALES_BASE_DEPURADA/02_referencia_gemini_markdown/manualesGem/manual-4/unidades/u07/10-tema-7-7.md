# 7.7 Conexión con bases de conocimiento locales

Tu modelo corre local. Tus PDFs viven local. Pero si embebes con OpenAI API, tus chunks pasan por servidores externos. Esta lección cierra el circuito: **RAG 100 % en tu infraestructura**.

::concepto{titulo="RAG local 100 %"}
**RAG completamente local** combina:

- **Modelo de lenguaje**: Ollama (Llama, Qwen, Mistral, etc.).
- **Modelo de embeddings local**: bge-m3, nomic-embed-text, multilingual-e5.
- **Vector store local**: ChromaDB, Qdrant local, pgvector con PostgreSQL local.
- **Documentos**: en tu drive institucional.
- **Interfaz**: Open WebUI con knowledge base nativa.

Resultado: cero datos cruzando internet. La cadena más segura posible para datos de menores, salud o gubernamentales.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
La promesa de "IA local" se rompe si una pieza del pipeline manda datos afuera. Es como tener bóveda de banco con una ventana abierta. Cerrar el círculo completo es lo que diferencia "soberanía teórica" de "soberanía real".
::/interioriza

## Stack mínimo recomendado

```
┌─────────────────────────────────────┐
│  Open WebUI (UI institucional)       │  → :3000
└────────────┬─────────────────────────┘
             │
┌────────────┴─────────────────────────┐
│  Ollama API local (Llama/Qwen 70B)   │  → :11434
└────────────┬─────────────────────────┘
             │
┌────────────┴─────────────────────────┐
│  ChromaDB local (vector store)        │  → :8000
└────────────┬─────────────────────────┘
             │
┌────────────┴─────────────────────────┐
│  bge-m3 vía Ollama (embeddings)      │
└──────────────────────────────────────┘
             ▲
             │
        Documentos PDF/MD
        en disco local
```

Todo en mismo servidor o LAN. Sin requests a internet.

## Setup paso a paso

### Opción A — Open WebUI con knowledge base (más simple)

Open WebUI ya trae todo:

1. Sidebar → "Workspace" → "Knowledge".
2. "+ New Knowledge Base", nombre "reglamento".
3. Configuración (sidebar):
   - Embedder: `nomic-embed-text` (descargar con `ollama pull nomic-embed-text`).
   - Vector DB: ChromaDB local (default).
4. Subir PDFs institucionales.
5. Chunking automático (configurable: tamaño, overlap).
6. Indexación tarda 1-5 min según volumen.
7. En chat, mencionas con `#reglamento` y RAG se activa.

Todo bajo `localhost:3000`. Ningún byte sale.

### Opción B — n8n con tools locales (más control)

Para el agente con RAG local desde n8n:

```
Workflow:
[Input] → [Ollama Embedder node] → [ChromaDB Query node]
        → [Ollama Chat node con context] → [Output]
```

Cada nodo apunta a `localhost:*`. n8n self-hosted en tu servidor.

### Opción C — Dify self-hosted con Ollama

Dify expone configuración de "Model Providers":
- Eliges "Ollama" como provider.
- Configuras URL `http://localhost:11434`.
- Eliges modelo (llama3.1, qwen2.5:72b, etc.).
- En knowledge: configuras embedder local también.

Resultado: el constructor visual de Dify pero todo apuntando a local.

## Embedders locales recomendados

| Embedder | Modelo Ollama | Idiomas | Calidad |
|---|---|---|---|
| **nomic-embed-text** | `ollama pull nomic-embed-text` | inglés ++, español + | bueno |
| **bge-m3** | `ollama pull bge-m3` | multilingüe excelente | muy bueno |
| **multilingual-e5-large** | requires custom | multilingüe | excelente |
| **mxbai-embed-large** | `ollama pull mxbai-embed-large` | inglés excelente | muy bueno |

Para institución mexicana: **bge-m3** es la opción.

## Vector stores locales

| Vector Store | Pros | Cuándo |
|---|---|---|
| **ChromaDB** | simple, embebido, default Open WebUI | < 100k vectores |
| **Qdrant local** | feature-rich, escalable | 100k - 10M vectores |
| **pgvector** | PostgreSQL + vectors | si ya usas Postgres |
| **Weaviate local** | GraphQL, hybrid search | empresarial |

Para 90 % de instituciones: ChromaDB embebido en Open WebUI basta.

## Comparación de calidad RAG: local vs cloud

Mismo reglamento (35 pp), mismas 10 preguntas:

```
LOCAL (Qwen 2.5 72B + bge-m3 + ChromaDB):
  - 9/10 respuestas correctas con cita.
  - Latencia mediana: 12 segundos.
  - Costo: $0 (electricidad).

CLOUD (Claude Sonnet + OpenAI embeddings + Pinecone):
  - 9.5/10 respuestas correctas con cita.
  - Latencia: 3 segundos.
  - Costo: ~$0.05 por consulta.
```

Para instituciones con datos sensibles, las 6 segundos extra de latencia y 0.5 punto de calidad **valen** la soberanía total.

## Anti-patrones

- **Mezclar embedder cloud y modelo local.** Pierdes soberanía y compatibilidad.
- **No actualizar embeddings al cambiar modelo.** Si migras de `nomic` a `bge-m3`, **reindexa todo**.
- **Backup ausente del vector store.** Pérdida = re-indexar todo (puede tardar horas).
- **Olvidar resincronización.** Si actualizas reglamento, re-indexa o tu agente cita versión vieja.
