# 7.3 Open WebUI (interfaz local tipo ChatGPT)

Tu Ollama corre. Tu CLI funciona. Pero tu coordinadora académica no va a abrir terminal. Necesita ChatGPT — pero local. Eso es Open WebUI.

::concepto{titulo="Open WebUI"}
**Open WebUI** (antes Ollama WebUI) es una interfaz web open source que se conecta a Ollama (u otros backends compatibles OpenAI API) y ofrece una experiencia idéntica a ChatGPT corriendo en localhost o en un servidor de tu institución. Características:

- **UI completa** tipo ChatGPT (conversaciones, historial, regenerar, editar).
- **Multi-modelo**: switch entre Llama, Mistral, Qwen, etc., en la misma sesión.
- **Multi-usuario** con autenticación, roles y permisos.
- **Knowledge base** integrado (RAG sin código).
- **Functions** custom (similar a tools).
- **Soporte para imágenes** (modelos vision como llava).
- **Whisper** integrado para audio.
- **Self-hosted** vía Docker en 5 minutos.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Construir un sistema soberano sin UI accesible es como construir un coche eléctrico sin tablero: técnicamente funciona, pero nadie lo va a manejar. Open WebUI es la diferencia entre "corro Ollama por curiosidad" y "mi institución entera usa IA local todos los días".
::/interioriza

## Instalación con Docker

Asumiendo que Ollama ya corre:

```bash
docker run -d \
  -p 3000:8080 \
  --add-host=host.docker.internal:host-gateway \
  -v open-webui:/app/backend/data \
  --name open-webui \
  --restart always \
  ghcr.io/open-webui/open-webui:main
```

Abres `http://localhost:3000`. Crear cuenta admin. Detecta automáticamente Ollama en `host.docker.internal:11434`.

## Lo que ofrece de fábrica

- **Conversaciones** con historial guardado (en SQLite local).
- **Selector de modelo** en cada chat.
- **System prompts** custom por chat.
- **Plantillas** de prompts reutilizables.
- **Compartir** conversación (link público o exportar).
- **Búsqueda en historial.**
- **Tags y carpetas** para organizar.
- **Multi-modal**: subir PDFs, imágenes, audio.
- **Voz**: Whisper para STT, TTS para output.

## Multi-usuario y roles

Open WebUI soporta multiusuario con roles:
- **Admin**: configura modelos, knowledge bases, usuarios, plugins.
- **User**: usa pero no configura.
- **Pending**: cuenta nueva esperando aprobación admin.

Auth: usuario+contraseña local, o vía OAuth (Google, GitHub, OpenID).

Para una institución, esto significa: 50 cuentas con login institucional, todos chateando con el modelo local sin que tú toques nada.

## Knowledge base integrado (RAG sin código)

Open WebUI tiene RAG embebido:
1. Sidebar → "Workspace" → "Knowledge".
2. Crear knowledge base nueva, nombre.
3. Upload PDFs, DOCX, TXT, URL.
4. Open WebUI los chunkea, embedde y indexa automáticamente con `nomic-embed-text` (o el embedder que configures).
5. En chat, mencionas la KB con `#nombre_kb` y el modelo recibe contexto relevante con citas.

Es Custom GPT pero **completamente local**.

## Functions (tools custom)

Plugin system:
1. Sidebar → "Workspace" → "Functions".
2. Code Python que define la función con docstring.
3. Open WebUI lo expone al modelo como tool.

Ejemplo: una function que consulta tu sistema de calificaciones interno y el modelo la invoca cuando relevante.

## Workflow del Asistente Institucional con Open WebUI

```
1. Servidor de la institución (puede ser una NUC o mini PC con GPU)
2. Ollama corre con Llama 3.1 70B + Qwen 2.5 72B + nomic-embed
3. Open WebUI corre en :3000 con login institucional (Google Workspace SSO)
4. Knowledge bases:
   - #reglamento → reglamento académico
   - #manualdocente → manual del docente
   - #calendario → calendario académico
5. Functions custom:
   - get_grade_info(student_id) → consulta sistema de calificaciones interno
   - check_attendance(student_id) → asistencias
6. 50 cuentas para staff. Cada uno usa con login normal.
```

Comparado con ChatGPT Enterprise: 0 dólares en suscripciones, datos nunca salen, control total.

## Costos reales

- **Hardware**: NUC o mini PC con GPU integrada o discreta. Configuración recomendada para 70B con buen rendimiento: $1500-3000 USD.
- **Electricidad**: ~50-100 USD/mes (depende GPU).
- **Mantenimiento TI**: 5-10 horas/mes una vez configurado.
- **Software**: $0 (Ollama y Open WebUI son MIT/BSD).

A 50 usuarios institucionales y volumen normal, ROI vs ChatGPT Enterprise: 4-6 meses.

## Anti-patrones

- **Self-hosted sin backups.** El servidor falla un día, pierdes historial. Configura backups del volumen Docker.
- **Sin HTTPS.** Si exponen al staff por LAN, mínimo TLS interno; nunca passwords planos.
- **Sin actualización.** Open WebUI publica updates frecuentes con security patches. `docker pull` mensual.
- **Sin moderation.** Si lo expones a externos (estudiantes), modelos abiertos sin filtros pueden generar contenido inapropiado. Llama 3.1 viene "alineado" pero no es perfecto.
