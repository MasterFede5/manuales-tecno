# 7.2 Ollama: descarga y ejecución

Ollama es la herramienta que convierte "correr un LLM local" de proyecto de fin de semana a comando de 60 segundos. Si vas a probar IA local, empezar por aquí no admite debate.

::concepto{titulo="Ollama"}
**Ollama** es un runtime ligero que descarga, ejecuta y sirve modelos de lenguaje abiertos en tu máquina. Provee:

- **CLI simple** (`ollama pull`, `ollama run`, `ollama list`).
- **API REST local** compatible con OpenAI API (en `localhost:11434`).
- **Library de modelos** curada (Llama, Mistral, Qwen, DeepSeek, Phi, Gemma, embeddings).
- **Soporte multi-OS** (macOS, Linux, Windows desde 2024).
- **GPU automática** si la tienes (CUDA/Metal/ROCm).

Lanzado en 2023, dominante en 2025 para consumo local de modelos abiertos.
::/concepto

::interioriza{titulo="¿Por qué te importa?"}
Antes de Ollama, correr un modelo local requería: descargar pesos, configurar Python, instalar PyTorch con CUDA correcto, escribir código de inferencia. Total: 3-5 horas mínimo. Hoy: `curl https://ollama.ai/install.sh | sh && ollama run llama3.1`. Total: 5 minutos. Esa diferencia transforma quién puede correr IA local — y por eso te importa: tu equipo de TI sin doctorado puede operar.
::/interioriza

## Instalación (Windows / macOS / Linux)

### Linux / macOS
```bash
curl -fsSL https://ollama.ai/install.sh | sh
```

### Windows
Descargar instalador desde [ollama.com/download](https://ollama.com/download).

### Docker (recomendado para servidor)
```bash
docker run -d \
  --gpus all \
  -v ollama:/root/.ollama \
  -p 11434:11434 \
  --name ollama \
  ollama/ollama
```

Tras instalación, el daemon corre en `localhost:11434`.

## Comandos esenciales

```bash
# Listar modelos disponibles para descargar
ollama list

# Descargar un modelo (~4.7 GB para llama3.1 8B Q4)
ollama pull llama3.1

# Ejecutar conversación interactiva
ollama run llama3.1

# Ejecutar con prompt único
ollama run llama3.1 "Resume el reglamento institucional en 3 puntos"

# Ejecutar con archivo
cat reglamento.md | ollama run llama3.1 "Resume este documento"

# Modelo + parámetros custom
ollama run llama3.1:70b --temperature 0.3 "..."

# Borrar un modelo (libera disco)
ollama rm llama3.1
```

## API REST (compatible OpenAI)

Ollama expone API tipo OpenAI en `http://localhost:11434`:

```python
# Ejemplo conceptual con OpenAI Python SDK apuntando a Ollama
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:11434/v1",
    api_key="ollama"  # cualquier string; Ollama lo ignora
)

response = client.chat.completions.create(
    model="llama3.1",
    messages=[{"role": "user", "content": "Hola"}]
)
print(response.choices[0].message.content)
```

Esto significa que **puedes usar el mismo código que escribiste para OpenAI/Anthropic** apuntando a localhost. Cero migración.

## Modelfile: customización

Ollama soporta `Modelfile` (similar a Dockerfile) para crear variantes customizadas:

```dockerfile
# Modelfile.asistente
FROM llama3.1:8b

SYSTEM """
Eres el Asistente Institucional Albatros. Respondes en español
mexicano profesional. Tono cálido pero formal. No inventas datos
sobre el reglamento.
"""

PARAMETER temperature 0.3
PARAMETER top_p 0.9
PARAMETER num_ctx 8192
```

Build:
```bash
ollama create asistente -f Modelfile.asistente
ollama run asistente
```

Tu modelo "asistente" ahora arranca con system prompt embebido.

## Cómo se compara con cloud (en práctica)

```
PROMPT: "Resume en 3 puntos el reglamento de admisiones (texto pegado de 5000 palabras)"

LLAMA 3.1 8B local (CPU): 25 segundos, calidad 7/10.
LLAMA 3.1 70B local (GPU): 8 segundos, calidad 8.5/10.
QWEN 2.5 72B local (GPU): 9 segundos, calidad 9/10 español.
CLAUDE SONNET 4.6 cloud: 3 segundos, calidad 9.5/10.
CLAUDE OPUS 4.7 cloud: 4 segundos, calidad 10/10.
```

Para 80 % de tareas institucionales, Llama 3.1 70B o Qwen 2.5 72B es **suficiente**. Para razonamiento crítico, cloud sigue ganando.

## Anti-patrones

- **Empezar con modelo gigante** (70B+) sin tener GPU. Inferencia tarda 60+ segundos en CPU.
- **No medir RAM disponible.** Modelos requieren al menos 1.2× su tamaño en RAM.
- **Olvidar quotas de disco.** Acumular modelos sin borrar llena el disco rápido.
- **Confiar en calidad sin medir.** Probar con tu golden set antes de migrar producción.
- **No actualizar.** Modelos abiertos mejoran cada 2-3 meses; tu Llama 3.1 hoy puede ser obsoleto en 6 meses.
